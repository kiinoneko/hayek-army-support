# actor_id 政治アクター台帳仕様 v0.1

## 0. 目的

本仕様は、後方支援軍において、政治行政上の意思決定・議決・答弁・監査・補助金交付・委託契約・指定管理・同意人事・審議会・外郭団体運営・公費支出に関与する政治行政アクターを、個人名・役職・関与内容・日付・`source_id` 付きで管理するための `actor_id` 仕様である。

この仕様の目的は、個人攻撃ではなく、公的権限・公費・公的影響に対する説明責任を明確化することである。

## 1. actor_id の定義

`actor_id` は、政治行政上の意思決定、公的権限、公費支出、公的発言、公的地位に関与する個人又は機関を管理するIDである。

`actor_id` は、人物・機関を単なる役職名だけで処理せず、どの公的立場の誰が、いつ、どの案件に、どのように関与したかを後から検証できる形で管理するために用いる。

## 2. 個人名明記原則

- 公的意思決定に関与した者は、役職名だけでなく、原則として個人名を明記する。
- 首長、議員、副首長、教育長、監査委員、教育委員、農業委員、審議会委員、議会同意人事対象者、外郭団体役員、指定管理者の責任者、補助金・委託契約に関与する公的責任者は、`source_id` 付きで個人名を記録する。
- 役人についても、公式資料、答弁、決裁、説明、委員会資料、議事録、契約資料等で職務上明記されている場合は、個人名、役職、関与内容を記録する。
- これは個人攻撃ではなく、公的職務・公的権限・説明責任の記録である。

## 3. actor_id 命名規則

`actor_id` は必ず `ACTOR_` で始める。

形式例:

- `ACTOR_0001`
- `ACTOR_TAKETOYO_001`
- `ACTOR_MIHAMA_001`
- `ACTOR_DIET_001`
- `ACTOR_PREF_AICHI_001`
- `ACTOR_COUNCIL_001`
- `ACTOR_APPOINTMENT_001`

推奨:

- 地域、機関、案件種別が明確な場合は、`ACTOR_地域又は機関_連番` とする。
- 同意人事など案件種別が重要な場合は、`ACTOR_APPOINTMENT_連番` のように種別を含める。
- 同姓同名、役職交代、兼職がある場合は、任期・所属・関与案件・`source_id` で識別する。

## 4. actor_id 台帳の必須項目

| 項目 | 内容 | 必須 |
|---|---|---|
| actor_id | 政治行政アクターID | 必須 |
| 氏名 | 個人名 | 原則必須 |
| 役職 | 首長、議員、職員、委員等 | 必須 |
| 所属 | 自治体、議会、委員会、団体等 | 必須 |
| 任期・在任期間 | 任期又は確認時点 | 任意 |
| 関与案件 | case_id又はpolicy_id | 必須 |
| 関与区分 | 発言、答弁、議決、同意、監査、決裁、契約、補助金交付等 | 必須 |
| 関与日 | 日付 | 必須 |
| source_id | 根拠資料 | 必須 |
| quote_id | 発言がある場合 | 任意 |
| policy_id | 政策案件 | 任意 |
| gap_id | 未確認事項 | 任意 |
| warning_id | 注意事項 | 任意 |
| 備考 | 注意点 | 任意 |

## 5. 関与区分

`actor_id` では、少なくとも以下の関与区分を扱う。

- 発言
- 答弁
- 質問
- 議決
- 賛否
- 同意人事
- 任命
- 監査
- 決裁
- 説明
- 契約
- 補助金交付
- 委託
- 指定管理
- 審議会
- 外郭団体運営
- 公費支出

## 6. 議会同意人事の記録項目

議会同意人事に関する `actor_id` では、可能な限り以下を記録する。

- 対象者氏名
- 任命権者
- 同意した議会
- 議決日
- 任期
- 職務内容
- 報酬・非常勤報酬の有無
- 関係団体・兼職
- 議会での説明内容
- 賛否
- `source_id`
- `gap_id`

未確認の項目は、空欄で確定させず、`gap_id` として管理する。

## 7. 境界線

- 公的職務、公的権限、公費、公的意思決定に関わる範囲に限定する。
- 家族、私生活、住所、連絡先、病歴、思想信条、宗教、その他公的職務と無関係な個人情報は扱わない。
- 未確認情報、伝聞、噂に基づいて個人責任を断定しない。
- `source_id` 又は `quote_id` がない場合は正式記録にしない。
- 個人名が未確認の場合は `gap_id` として管理する。

## 8. WARNING / STOP

| ID | 内容 |
|---|---|
| `WARN_ACTOR` | 関与者個人名・役職・所属が未確認 |
| `WARN_ROLE` | 役職又は関与区分が未確認 |
| `WARN_APPOINTMENT` | 同意人事の任期・議決日・同意状況が未確認 |
| `WARN_QUOTE` | 発言原文未確認 |
| `WARN_CONTEXT` | 発言前後文脈未確認 |
| `STOP_PRIVACY` | 公的職務と無関係な個人情報混入 |
| `STOP_PUBLIC` | 外部公開判断前 |
| `USER_DECISION_PUBLIC` | 外部発信するか |
| `USER_DECISION_LEGAL` | 法務官へ引き渡すか |

## 9. 禁止事項

- 公的職務と無関係な個人情報を扱わない。
- 家族、住所、連絡先、病歴、思想信条を扱わない。
- 伝聞で個人責任を断定しない。
- `source_id` なしで `actor_id` を正式化しない。
- `quote_id` なしで発言を正式言質化しない。
- `actor_id` を根拠に違法、不正、癒着、汚職を断定しない。
- 外部発信へ自動進行しない。
- 情報公開請求へ自動進行しない。
- 法的断定へ自動進行しない。
- 政策評価断定へ自動進行しない。

## 10. 運用メモ

- `actor_id` は、`source_id`、`quote_id`、`policy_id`、`gap_id`、`warning_id` と紐づけて管理する。
- 役職名のみでしか確認できない場合は、仮記録にとどめ、個人名の未確認を `gap_id` として残す。
- 発言を扱う場合は、発言者の `actor_id`、発言原文の `quote_id`、根拠資料の `source_id` を分けて管理する。
- 同一人物が複数の役職又は案件に関与する場合は、同一 `actor_id` に紐づけつつ、関与案件・関与区分・関与日を分けて記録する。
