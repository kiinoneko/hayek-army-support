# 後方支援軍Runbook接続図 v0.1

## 1. 位置づけ

この文書は、後方支援軍で作成済みのRunbook、テンプレート、ID台帳、および今後作成予定の原文資料相互検証・統合検証会議を、案件処理の流れに沿って接続するための索引である。

詳細な分析項目、判定基準、記入例は各Runbook本文へ委ねる。本書では、どの場面でどのRunbookを使うか、どこで原文検証を行うか、どこでハイエク軍・前線部隊を参加させるかを判断するための運用導線だけを整理する。

## 2. 全体処理フロー

```text
case_id起票
→ source_id正式化
→ 原文資料相互検証
→ gap_id / warning_id / stop_id 判定
→ 公金支出類型判定
→ 専門Runbookへ接続
→ 統合検証会議
→ ID台帳へ記録
→ 公開前チェック
→ レポート・外部発信・情報公開請求候補へ接続
```

基本順序は、先に案件と根拠資料を固め、次に原文確認と停止条件を確認し、その後に支出類型と専門Runbookへ進む流れとする。重要案件、対外発信予定案件、解釈が割れる案件は、公開前チェックの前に統合検証会議へ接続する。

## 3. 最初に使うRunbook

新しい案件を扱うときは、最初に `05_tools/runbook_案件起票初動分類_v0.1.md` を使う。

この段階では、案件の入口を作り、親 `case_id`、子 `case_id`、初動分類、想定される支出類型、必要な根拠資料、未確認事項を整理する。類型が不明な場合は断定せず、後続の公金支出類型判定へ接続する。

## 4. source_id根拠資料管理への接続

根拠資料を扱う段階では、`05_tools/runbook_source_id根拠資料管理_v0.1.md` に接続する。

ここでは、資料の所在、資料名、取得日、公開元、URL、保存区分、参照対象を整理し、案件で使う根拠資料に `source_id` を正式に付与する。ただし、`source_id` があるだけでは原文確認済みとは扱わない。`source_id` は根拠資料を識別するIDであり、引用・数値・発言の正確性を保証するものではない。

## 5. 原文資料相互検証への接続候補

原文、引用、数値、発言を確定する前に、今後作成予定の `05_tools/runbook_原文資料相互検証_v0.1.md` へ接続する。

この段階では、次を確認する。

- `source_id` があるだけでは原文確認済みと扱わない。
- 原文、ページ、日時、発言者、資料名、金額、表、引用箇所を確認する。
- 要約資料、議会だより、動画切り取り、SNS投稿だけで重要断定しない。
- 原文未確認の場合は `gap_id` または `stop_id` へ接続する。
- この段階は今後作成予定の `runbook_原文資料相互検証_v0.1.md` に接続する。

未作成の段階では、既存Runbookとして扱わず、接続候補として参照する。原文未確認のまま専門Runbookへ進む場合は、確認状態を明示し、必要に応じて停止する。

## 6. gap_id / warning_id / stop_id 判定への接続

未確認、注意、停止条件を整理する段階では、`05_tools/runbook_gap_warning_stop判定仕様_v0.1.md` に接続する。

原文未確認、資料不足、類型不明、数値不一致、引用箇所不明、公開判断の不確実性がある場合は、`gap_id`、`warning_id`、`stop_id` のいずれかで状態を明示する。`stop_id` が出た場合は、専門Runbookまたは統合検証会議へ進む前に停止条件を解消する。

## 7. 公金支出類型判定への接続

支出類型が不明、または補助金・委託契約・指定管理・調達手続が混同されている場合は、`05_tools/runbook_公金支出類型判定_v0.1.md` に接続する。

この文書内では支出類型を詳細分析しない。類型が不明なまま専門Runbookへ進まず、まず公金支出類型判定Runbookで扱う入口を決める。

## 8. 専門Runbookへの接続

公金支出類型判定後、類型に応じて次の専門Runbookへ接続する。

| 類型・確認対象 | 接続先 |
|---|---|
| 補助金、交付決定、精算、返還条件 | `05_tools/runbook_補助金支出確認_v0.1.md` |
| 委託契約、仕様書、成果物、検収 | `05_tools/runbook_委託契約確認_v0.1.md` |
| 公の施設、指定管理料、利用料金、モニタリング | `05_tools/runbook_指定管理確認_v0.1.md` |
| 入札、随意契約、プロポーザル、落札率 | `05_tools/runbook_調達手続確認_v0.1.md` |

専門Runbookへ進む前に、`source_id`、原文確認状態、`gap_id / warning_id / stop_id`、支出類型を確認する。

## 9. 統合検証会議への接続候補

重要案件、対外発信予定案件、情報公開請求候補、解釈が割れる案件、または後方支援軍だけで公開判断すべきでない案件は、今後作成予定の `05_tools/runbook_統合検証会議_v0.1.md` へ接続する。

この段階では、次を確認する。

- 後方支援軍、ハイエク軍、前線部隊が参加する検証ゲートである。
- 後方支援軍は原文、数値、引用、`source_id` を確認する。
- ハイエク軍は自由、財産権、納税者負担、制度効果、権力集中を評価する。
- 前線部隊はレポート化、PDF化、外部発信、情報公開請求に使えるか確認する。
- 反証役は読み違い、別解釈、矛盾、断定過剰を確認する。
- この段階は今後作成予定の `runbook_統合検証会議_v0.1.md` に接続する。

未作成の段階では、既存Runbookとして扱わず、接続候補として参照する。

## 10. ID台帳への接続

案件、根拠資料、未確認事項、警告、停止条件、判断、変更履歴を台帳化する場合は、`04_templates/template_後方支援軍ID台帳_v0.1.md` に接続する。

台帳化の対象には、少なくとも `case_id`、`source_id`、`gap_id`、`warning_id`、`stop_id`、判断記録、公開前確認状態を含める。ID台帳は、案件処理の追跡性を確保するための記録先であり、詳細分析本文の代替ではない。

## 11. 公開前チェックへの接続

公開、PDF化、外部発信、断定表現、情報公開請求候補への接続前には、`05_tools/runbook_公開前チェック実行_v0.1.md` に接続する。

公開前チェックを通過するまで、外部発信・PDF化・断定表現を行わない。個人情報、黒塗り前資料、非公開資料、secret、APIキー、PDF原本、画像が混入していないかも確認する。

## 12. 親case_idと子case_idの使い分け

親 `case_id` は、案件全体を束ねる単位として使う。複数年度、複数支出、複数契約、複数施設、複数論点にまたがる場合でも、同一テーマで追跡する必要があるときは親 `case_id` に集約する。

子 `case_id` は、個別の支出、契約、施設、年度、発言、資料、情報公開請求候補を分けて管理する単位として使う。専門Runbookへ接続する際は、子 `case_id` ごとに支出類型と確認状態を明示する。

親子関係が不明な場合は、案件起票初動分類Runbookへ戻り、無理に統合または分割しない。

## 13. 後方支援軍・ハイエク軍・前線部隊の役割分担

| 役割 | 主な確認対象 | 接続場面 |
|---|---|---|
| 後方支援軍 | 原文、数値、引用、資料名、ページ、`source_id`、ID台帳 | 起票、資料管理、原文検証、公開前チェック |
| ハイエク軍 | 自由、財産権、納税者負担、制度効果、権力集中 | 統合検証会議、論点評価 |
| 前線部隊 | レポート化、PDF化、外部発信、情報公開請求への使用可能性 | 統合検証会議、公開前チェック後の接続 |
| 反証役 | 読み違い、別解釈、矛盾、断定過剰 | 原文資料相互検証、統合検証会議 |

## 15. 作業停止条件

次に該当する場合は `stop_id` を付与し、停止条件を解消するまで次工程へ進まない。

- 類型不明のまま専門Runbookへ進めようとしている。
- `source_id` 未確定のまま分析を確定しようとしている。
- 原文未確認のまま引用・数値・発言を確定しようとしている。
- `gap_id / warning_id / stop_id` 判定を省略している。
- 補助金、委託契約、指定管理、調達手続を混同している。
- 統合検証会議が必要な重要案件で、後方支援軍だけで公開判断しようとしている。
- 公開前チェック前に外部発信・PDF化・断定表現を行おうとしている。
- 個人情報、黒塗り前資料、非公開資料、secretの混入疑いがある。
- PDF原本や画像をGitHubへ追加しようとしている。
- 旧本文と新本文が混在している。
- 変更対象外ファイルが差分に入っている。
- `README.md`、`archive`、`04_templates`、`05_tools` 既存Runbookが目的外に変更されている。

## 16. 禁止事項

- 既存Runbook本文を長くコピーして本書へ混入させない。
- 補助金、委託契約、指定管理、調達手続を本書内で詳細分析しない。
- 類型不明の案件を断定しない。
- 原文未確認の引用・数値・発言を確定しない。
- `stop_id` が出た案件を未解消のまま専門Runbookまたは統合検証会議へ進めない。
- 公開前チェック前に外部発信、PDF化、断定表現を行わない。
- PDF原本、画像、個人情報、secret、APIキーを追加しない。
- README.md、archive、04_templates、05_tools既存Runbookを目的外に変更しない。
- 旧本文と新本文を併記しない。

## 17. PR前確認条件

PR前に次を確認する。

- 変更ファイルは `00_index/index_後方支援軍Runbook接続図_v0.1.md` の1個のみである。
- 新規追加なので差分は追加行のみであり、既存本文の削除を含まない。
- `README.md` を変更していない。
- `archive` を変更していない。
- `04_templates` を変更していない。
- `05_tools` 既存Runbookを変更していない。
- PDFや画像を追加していない。
- secretや個人情報を追加していない。
- 章番号の重複がない。
- Markdown表が壊れていない。
