# 後方支援軍 運用ダッシュボード仕様 v0.1

## 1. 運用ダッシュボードの定義

後方支援軍 運用ダッシュボードは、後方支援軍が扱う全案件について、案件の現在状態、停止理由、判断待ち、次工程、引渡先、公開可否を一目で確認するための司令盤である。

このダッシュボードは、案件処理ログ、判断ログ台帳、source_id台帳、gap_id台帳、warning_stop仕様、incident対応記録、change_id記録の代替ではない。全体状況を俯瞰し、どの案件が進行中、停止中、判断待ち、引渡待ち、公開可否確認中であるかを一覧管理するために使用する。

## 2. 使用目的

- 全案件の現在状態を一覧化する。
- STOP案件、USER_DECISION待ち案件、gap_id未解消案件、warning_id残存案件を即時把握する。
- source_id未整備、PDF本文未確認、該当ページ未確認など、根拠資料上の未完了事項を把握する。
- actor_idに関する確認状況を管理し、公的職務、公的権限、公費、公的影響に関する説明責任の範囲を整理する。
- 中核軍、法務官、前線部隊、資料化担当、ユーザーへの引渡待ち案件を管理する。
- インシデント、PR、change_id、マージ状況、差し戻し状況を俯瞰する。
- GitHub公開、外部発信、資料公開区分の確認状態を管理する。
- 次工程、主担当、必要なsource_id又はgap_id、USER_DECISIONの有無を整理する。

|---|---|

| STOP案件一覧 | stop_idにより停止している案件 |
| USER_DECISION待ち一覧 | ユーザー判断待ちの案件 |
| gap_id未解消一覧 | 未確認事項が残る案件 |
| warning_id一覧 | 注意事項が残る案件 |

| actor_id要確認一覧 | 政治行政アクターの個人名・役職・関与内容確認が必要な案件 |

| インシデント一覧 | 誤公開、誤断定、個人情報混入等の対応状況 |
| 変更管理一覧 | PR、change_id、マージ状況、差し戻し状況 |
| 公開可否確認一覧 | GitHub公開、外部発信、資料公開区分の確認状況 |

actor_idの取扱いは、以下を必ず遵守する。

- actor_idは個人攻撃ではなく、公的職務・公的権限・公費・公的影響の説明責任を管理するためのIDである。
- 公的職務と無関係な家族、住所、連絡先、病歴、思想信条、私生活は扱わない。
- actor_idを根拠に個人責任、違法、不正、癒着、汚職を断定しない。

## 17. 更新ルール

- 運用ダッシュボードはcase_log、判断ログ台帳、source_id台帳、gap_id台帳、warning_stop仕様、incident対応記録、change_id記録を参照する。
- ダッシュボード単独で事実認定しない。
- ダッシュボードは一覧表示であり、正式根拠はsource_id台帳と原資料に戻る。
- STOP案件は解除条件が満たされるまで次工程へ進めない。
- USER_DECISION待ち案件はユーザー判断なしに進めない。
- 更新日は必ず記録する。
- CLOSED案件も必要に応じて履歴として残す。

## 18. 禁止事項

- ダッシュボードを正式根拠として扱わない。
- source_idなしの事実を正式化しない。
- gap_idを根拠として扱わない。
- STOP案件を自動で進めない。
- USER_DECISION待ち案件を自動で進めない。
- 外部発信へ自動進行しない。
- 情報公開請求へ自動進行しない。
- 法的断定へ自動進行しない。
- 政策評価断定へ自動進行しない。
- actor_idを根拠に個人責任、違法、不正、癒着、汚職を断定しない。
- PDF原本、個人情報、黒塗り前資料をGitHubに追加しない。
- archive配下を変更しない。
