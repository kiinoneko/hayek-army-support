# 各軍成果物常設ビュー v0.1

## 1. 位置づけ

この索引は、後方支援軍、ハイエク軍、前線部隊、法務官・慎重確認、猫さん最終判定の成果物を常設で見るためのダッシュボードである。

この文書は成果物本文ではなく、成果物の所在、状態、関連ID、公開範囲、次工程を管理する索引である。実案件の詳細本文、個人情報、PDF原本、画像、secret、API key、黒塗り前情報は記載しない。成果物の詳細は、各Runbook、ID台帳、decision_id、公開前チェックへ接続する。

## 2. 対象範囲

対象範囲は、各軍・部門が作成または確認する成果物のうち、所在、確認状態、関連ID、公開範囲、次工程を一覧管理する必要があるものとする。

- 後方支援軍の調査・検証・ID管理・公開前チェック関連成果物
- ハイエク軍の制度評価・自由秩序評価・負担評価・裁量評価関連成果物
- 前線部隊の公開案・説明文・広報PR本文・公開用PDF候補・SNS投稿案
- 法務官・慎重確認のAI法務官確認、実弁護士確認要否、法務リスク、公開停止条件関連成果物
- 猫さん最終判定のdecision_id、公開範囲、保留判断、残存リスク関連成果物

## 3. 成果物を常設表示する目的

成果物を常設表示する目的は、成果物本文を複製せずに、各軍の成果物、確認状態、関連ID、公開範囲、次工程を俯瞰できる状態を維持することである。

- 成果物の所在と責任部門を確認する。
- case_id、source_id、gap_id、warning_id、stop_id、decision_id、request_idの接続を確認する。
- 公開可否、保留、停止、追加確認の状態を確認する。
- 事実記述、制度評価、価値判断、政策提案を混同しないように管理する。
- 公開前チェック、法務確認、猫さん最終判定への接続を明確にする。

## 4. 各軍・部門の役割

| 軍・部門 | 主な役割 | 主な成果物 | 公開可否への関与 |
|---|---|---|---|
| 後方支援軍 | 原資料確認、ID接続、相互検証、差分・警告・停止管理 | source_id、原文資料相互検証、gap_id、warning_id、stop_id、ID台帳、公開前チェック | 事実根拠と停止条件を提示する |
| ハイエク軍 | 制度評価、自由秩序評価、納税者負担評価、行政裁量評価、利益団体化評価 | 評価メモ、論点整理、制度評価案 | 事実根拠に基づく評価状態を提示する |
| 前線部隊 | 公開案、説明文、広報PR本文、公開用PDF候補、SNS投稿案の作成 | 外部発信用の候補成果物 | 表現リスクと公開範囲を整理する |
| 法務官・慎重確認 | AI法務官確認、実弁護士確認要否、法務リスク、公開停止条件の確認 | 法務確認メモ、停止条件整理 | stop_idや法務リスクにより公開可否へ影響する |
| 猫さん最終判定 | 最終判定、保留判断、decision_id、公開範囲、残存リスクの確認 | decision_id、最終判定ログ | 最終的な公開可否と次工程を決定する |

## 12. ID接続ルール

各成果物は、可能な限り次のIDへ接続する。

| ID | 接続対象 | 表示ルール |
|---|---|---|
| case_id | 案件単位 | すべての成果物で可能な限り表示する |
| source_id | 原資料、数値、直接引用、事実根拠 | 事実記述と評価の根拠として表示する |
| gap_id | 原資料間の不一致、欠落、差分 | 未解消の場合は確認状態に反映する |
| warning_id | 注意喚起、表現リスク、公開前警告 | 公開範囲と次工程に反映する |
| stop_id | 公開停止条件、法務停止、未解除リスク | 未解除の場合は公開可として表示しない |
| decision_id | 猫さん最終判定、保留判断、公開範囲決定 | 最終判定欄へ接続する |
| request_id | 情報公開請求、追加資料請求 | 原資料不足や次工程へ接続する |

## 13. 公開範囲の表示ルール

公開範囲は、次の区分で表示する。

この索引における公開範囲は、成果物の用途区分又は使用候補を示すものであり、公開可否の最終判定ではない。公開可否は、stop_id、法務確認、猫さん最終判定、decision_id、公開前チェックの状態を確認した上で判断する。公開範囲が「公開用PDF」又は「外部発信」であっても、最終判定が未判定又は保留の場合は公開可として扱わない。

| 公開範囲 | 内容 | 表示上の注意 |
|---|---|---|
| 内部メモ | 内部検討、調査、確認、法務確認用 | 外部公開可と誤認させない |
| 要約 | 原資料や成果物の要約 | 原文と要約の区別を明示する |
| 数値 | 公開可能な数値、集計値 | source_idに接続する |
| 直接引用 | 公開可能な短い引用 | source_idと引用条件に接続する |
| 公開用PDF | 公開候補PDFまたは公開用PDF | PDF原本や黒塗り前情報は貼り付けない |
| 外部発信 | SNS、広報PR、Web公開文 | 法務確認と猫さん最終判定に接続する |

stop_id未解除、法務確認未了、猫さん最終判定未了の成果物は、公開可として表示しない。

## 14. 更新ルール

- 成果物本文をこの索引へ貼り付けず、所在、状態、関連ID、次工程のみを更新する。
- 新しい成果物を追加する場合は、関連case_idと可能な限りsource_id、gap_id、warning_id、stop_id、decision_id、request_idを接続する。
- 確認状態、公開範囲、最終判定、次工程を同時に見直す。
- stop_idが発生した場合は、該当成果物を公開可として表示しない。
- 法務確認が必要な成果物は、AI法務官確認と実弁護士確認要否を分けて記録する。
- 猫さん最終判定が更新された場合は、decision_id、公開範囲、残存リスク、次工程を更新する。
- 事実記述、制度評価、価値判断、政策提案は別項目として扱い、混同しない。

## 15. 禁止事項

- 実案件の詳細本文を記載しない。
- 個人情報を記載しない。
- PDF原本、画像、黒塗り前情報を貼り付けない。
- secret、API key、認証情報、非公開トークンを記載しない。
- 成果物本文を全文転載しない。
- stop_id未解除、法務確認未了、猫さん最終判定未了の成果物を公開可として表示しない。
- 事実記述と制度評価・価値判断・政策提案を混同しない。
- 公開範囲が未確定の成果物を外部発信可能として扱わない。
