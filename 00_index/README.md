# 00_index — 案件・索引管理

## 1. そのフォルダの目的

案件、資料、作業状況、成果物の対応関係を一元管理し、原資料から上申書まで追跡できる入口を作る。

## 2. 保存する資料の種類

- 案件台帳、資料索引、進捗表、更新履歴
- 案件ID、対象機関、対象期間、担当、状態、関連ファイルへの相対リンク
- 未確認事項一覧、情報公開請求候補一覧
- 納税者負担の集計索引（補助金、助成、給付、減免、委託、指定管理、外郭団体支援、公費投入、施設維持費、行政サービス拡充を含む）
- 後方支援軍の正式指示プロンプト、部隊総覧、運用方針

## 3. Codexが行ってよい作業

- 台帳の新規作成、追記、並べ替え、重複候補の表示
- ファイル間リンク、メタデータ、進捗状態の整備
- 欠落項目、出典切れ、更新漏れの機械的な検出
- 事実、引用、出典、推定、評価、未確認事項、情報公開請求候補を分けた索引の生成

## 4. Codexが行ってはいけない作業

- 案件の重要度、違法性、妥当性、政治的評価、制度設計を最終判断すること
- 根拠なく案件を完了扱いにすること、未確認事項を削除すること
- 原資料や報告書の内容を、索引上の都合で書き換えること
- ハイエク軍またはユーザーの承認を代行すること

## 5. ハイエク軍への接続方法

案件IDをキーに、`01_sources`の原資料、`02_processed`の作業資料、`03_reports`の報告書を相対リンクで結ぶ。ハイエク軍への引き渡し時は、案件の状態、最新報告書、主要出典、未確認事項、情報公開請求候補をこのフォルダの台帳から参照できるようにする。

## 6. 運用上の注意

- 台帳は判断結果ではなく、探索と進捗管理のための記録とする。
- 推定値と確認済み金額を同じ列で混在させず、推定には根拠、計算方法、仮定、不確実性を付す。
- 納税者負担は区分間の重複計上、年度、支出主体、受益主体を確認する。
- ファイルを移動・改名した場合は、関連リンクと更新履歴も更新する。

## 7. 主要文書

- [後方支援軍 正式指示プロンプト](後方支援軍_指示プロンプト.md)：マイGPT「後方支援軍」に貼り付けて使用する正式な指示、常設中枢、必要時召集の専門与力および標準出力形式を定める。
- [後方支援軍 基本ドクトリン](../06_doctrine/後方支援軍_基本ドクトリン.md)：情報加工段階、責任境界、GitHub運用、ハイエク軍への接続および禁止事項を定める。
- [石田三成 総大将指示プロンプト](石田三成_総大将指示プロンプト.md)：マイGPT「石田三成GPT」用に、後方支援軍総大将としての任務、指揮下の与力、案件処理、Codex軍令および上申形式を定める。
- [石田三成 兵站行政ドクトリン](../06_doctrine/石田三成_兵站行政ドクトリン.md)：資料・台帳・進捗・Codex軍令・PR確認および上申経路に対する石田三成の統制方法を定める。

## 8. 実戦投入用台帳

- [原資料測量台帳](原資料測量台帳.md)：原資料の所在、出典、取得日、保存場所、加工先、上申先および確度を管理し、原資料から加工物・上申資料までの復帰線を維持する。

台帳は原資料単位で記録し、不明な項目は推測で補わず「未確認」とする。政策評価や最終判断には使用しない。
