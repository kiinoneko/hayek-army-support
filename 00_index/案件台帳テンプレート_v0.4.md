# 案件台帳テンプレート v0.4（実案件運用補足仕様）

## 1. v0.4の位置づけ

- `案件台帳テンプレート_v0.3.md` は案件台帳の基本テンプレートである。
- v0.4は案件台帳を実案件で運用するための補足仕様である。
- v0.4は、案件状態、信頼度、内容確度、gap優先度、公開前チェック、金額照合、引渡条件を標準化する。
- v0.4は既存の v0.1、v0.2、v0.3 を上書きしない。
- IDは証拠そのものではなく、証拠・判断・処理を追跡する索引である。

## 2. `case_id` 命名ルール

| 形式 | 用途 | 例 |
|---|---|---|
| `case_YYYYMMDD_連番` | 汎用案件 | `case_20260625_001` |
| `case_自治体_分野_YYYYMMDD_連番` | 自治体政策案件 | `case_TAKETOYO_WELFARE_20260625_001` |
| `case_自治体_団体_YYYYMMDD_連番` | 外郭団体・社協案件 | `case_TAKETOYO_SHAKYO_20260625_001` |
| `case_自治体_QUOTE_YYYYMMDD_連番` | 議会発言確認案件 | `case_TAKETOYO_QUOTE_20260625_001` |
| `case_自治体_CONTRACT_YYYYMMDD_連番` | 委託契約確認案件 | `case_TAKETOYO_CONTRACT_20260625_001` |
| `case_自治体_SUBSIDY_YYYYMMDD_連番` | 補助金確認案件 | `case_TAKETOYO_SUBSIDY_20260625_001` |
| `case_自治体_REQUEST_YYYYMMDD_連番` | 情報公開請求案件 | `case_TAKETOYO_REQUEST_20260625_001` |

- 個人名、住所、電話番号、メールアドレス、未加工資料名を `case_id` に入れない。
- 長すぎる `case_id` を避ける。
- 案件名の詳細は `case_name` に記録する。
- 一度発行した `case_id` は別案件に再利用しない。

## 3. 案件状態遷移ルール

| 現在状態 | 次に進める状態 | 条件 |
|---|---|---|
| 未着手 | 調査中 | 案件目的、対象範囲、ownerが設定済み |
| 調査中 | 根拠資料確認中 | 資料候補がある |
| 根拠資料確認中 | 数値整合性確認中 | `source_id`候補がある |
| 根拠資料確認中 | 未確認事項あり | 資料不足又は本文未確認がある |
| 未確認事項あり | 情報公開請求案件 | `gap_id`が公開資料で解消できない |
| 未確認事項あり | ユーザー判断待ち | `request_id`化又は保留判断が必要 |
| warningあり | 分析完了 | warningの影響範囲が明記されている |
| stop中 | ユーザー判断待ち | `decision_id`が必要 |
| stop中 | 根拠資料確認中 | 停止解除に必要な資料確認へ戻す |
| 請求文案作成済み | 提出待ち | `decision_id`がある |
| 提出待ち | 提出済み | ユーザーが提出を承認した |
| 提出済み | 回答待ち | 受付又は提出事実を記録済み |
| 回答待ち | 回答受領 | 回答又は開示資料を受領 |
| 回答受領 | 根拠資料確認中 | 開示資料を `source_id` 化する |
| 分析完了 | 引渡済み | `handoff_id`が作成済み |
| 分析完了 | 終了 | remaining_gaps / remaining_warnings を記録済み |
| 終了 | 再調査 | 新資料、矛盾、追加判断が発生 |

## 4. 状態変更時の `change_id` 条件

| 状態変更 | change_id要否 | 理由 |
|---|---|---|
| 未着手 → 調査中 | 任意 | 初期作業開始 |
| 調査中 → 根拠資料確認中 | 任意 | 資料探索段階 |
| 根拠資料確認中 → 分析完了 | 必須 | 判断結果に影響する |
| 未確認事項あり → 情報公開請求案件 | 必須 | 外部アクション候補化 |
| stop中 → 再開 | 必須 | 停止解除の理由が必要 |
| 請求文案作成済み → 提出待ち | 必須 | 文案確定に近い |
| 提出待ち → 提出済み | 必須 | 外部アクション実行 |
| 回答受領 → 根拠資料確認中 | 必須 | 新資料追加 |
| 分析完了 → 引渡済み | 必須 | 責任線の移動 |
| 分析完了 → 終了 | 必須 | 終了理由の記録 |
| 終了 → 再調査 | 必須 | 案件再開理由の記録 |

- 重要な状態変更は `change_id` で記録する。
- `stop_id`解除、外部提出、引渡、終了、再調査は必ず `change_id` を残す。
- `change_id`なしに重要変更を処理済みにしない。

## 5. `source_id` 信頼度コード表

| 信頼度 | 情報源 | 扱い |
|---|---|---|
| S | 原資料、一次資料、開示資料、契約書、議事録、決算書、予算書 | 正式根拠候補 |
| A | 公式概要、公式説明資料、公式広報、公式サイト | 根拠候補。ただし原資料確認優先 |
| B | 公的機関資料、統計資料、制度資料 | 補助根拠 |
| C | 報道、専門家記事、団体資料 | 参考情報 |
| D | SNS、ブログ、動画、演説、公開発信 | quote_id候補又は補助情報 |
| E | AI回答、未出典メモ、伝聞 | 事実化禁止 |

## 6. `source_id` 内容確度コード表

| 内容確度 | 意味 | 扱い |
|---|---|---|
| A | 本文確認済み、該当ページ又は該当欄特定済み | 正式根拠候補 |
| B | 資料存在確認済み、本文確認一部のみ | 仮根拠。正式化前に再確認 |
| C | URL又は資料名のみ確認、本文未確認 | 正式根拠化不可 |
| D | 要約、二次資料、引用経由のみ | 一次資料確認が必要 |
| E | 真偽未確認、取得未了 | 事実化禁止 |

- 信頼度Sでも内容確度Cなら正式根拠化しない。
- 内容確度Aには該当ページ、該当欄、確認日を記録する。
- PDF存在確認とPDF本文確認を混同しない。

## 7. `gap_id` 優先度分類

| 優先度 | 意味 | 対応 |
|---|---|---|
| A | 正式判断、外部発信、情報公開請求、評価断定に必須 | 優先解消又はSTOP |
| B | 精度向上に重要 | 調査継続 |
| C | 参考確認 | 保留可 |
| HOLD | ユーザー判断待ち | `decision_id`待ち |
| CLOSED | 解消済み | 解消根拠を記録 |

## 8. `gap_id` 解消方法分類

| 解消方法 | 内容 |
|---|---|
| SOURCE確認 | 既存公開資料を確認する |
| WEB探索 | 公式サイト、議会、WAM、入札情報等を再探索する |
| UPLOAD待ち | ユーザーによる資料アップロードを待つ |
| REQUEST化 | 情報公開請求候補にする |
| LEGAL確認 | 法務官へ確認する |
| USER_DECISION | ユーザー判断へ戻す |
| HOLD | 保留 |
| CLOSED | 解消済み |
| UNRESOLVABLE | 解消不能として記録 |

## 9. `warning_id` 判定チェックリスト

- [ ] 年度が異なる
- [ ] 会計区分が異なる
- [ ] 予算額と決算額が混在している
- [ ] 補助金と委託料が混在している
- [ ] 公費と制度収入が混在している
- [ ] 比較対象の条件が異なる
- [ ] 発言の前後文脈が不足している
- [ ] 要約と原文が混在している
- [ ] 数値の分母・分子が不明
- [ ] PDF本文未確認
- [ ] 原資料ではなく概要資料のみ
- [ ] 外部発信時に誤認リスクがある

## 10. `stop_id` 判定チェックリスト

- [ ] source_idがない
- [ ] PDF本文未確認で正式化しようとしている
- [ ] 数値の根拠がない
- [ ] 分母・分子・算出式が不明
- [ ] quote_idの原文未確認
- [ ] actor_idの職務上関与が未確認
- [ ] 個人情報が含まれる
- [ ] 黒塗り前資料が含まれる
- [ ] APIキー、パスワード、トークンが含まれる
- [ ] 情報公開請求提出判断前
- [ ] 外部発信判断前
- [ ] 法的断定前
- [ ] 政策評価断定前
- [ ] archive配下変更前
- [ ] GitHub公開判断前

## 13. 議会発言確認手順

| 手順 | 内容 | 必要ID |
|---|---|---|
| 1 | 公式議事録、会議録、議会録画を確認する | `source_id` |
| 2 | 発言者の氏名、役職、所属を確認する | `actor_id` |
| 3 | 発言日、会議名、議題を確認する | `source_id` |
| 4 | 発言原文を確認する | `quote_id` |
| 5 | 前後文脈を確認する | `warning_id` |
| 6 | 関連政策・事業へ接続する | `policy_id` |
| 7 | 発言と議決行動、予算、決算の整合確認候補を作る | `gap_id` |

- 原文未確認の発言は正式 `quote_id` 化しない。
- 切り取りで意味を変えない。
- 矛盾候補を直ちに虚偽、不正、欺瞞と断定しない。

## 17. 禁止事項

- source_idなしで確認済み事実にしない。
- 内容確度C以下の資料を正式根拠にしない。
- gap_idを根拠として扱わない。
- warning_idを無視して外部発信しない。
- stop_idをAI判断だけで解除しない。
- decision_idなしに情報公開請求、外部発信、評価断定、引渡確定を行わない。
- 予算額、決算額、契約額、支払実績を混同しない。
- 補助金、委託料、負担金、交付金、制度収入を混同しない。
- 原文未確認の発言をquote_id化しない。
- actor_idだけで個人責任、違法、不正、癒着、汚職を断定しない。
- PDF原本、個人情報、黒塗り前資料、APIキーをGitHubに追加しない。
- archive配下をUSER_DECISIONなしに変更しない。
