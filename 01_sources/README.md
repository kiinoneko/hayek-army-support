# 01_sources — 原資料保管

## 1. そのフォルダの目的

分析前の原資料と取得記録を改変せず保存し、後続成果物の事実・引用・出典を検証可能にする。

## 2. 保存する資料の種類

- 法令、条例、規則、予算書、決算書、事業評価、契約・入札資料
- 補助金、助成、給付、減免、委託、指定管理、外郭団体支援に関する公表資料（すべて納税者負担として整理する）
- 議事録、会見資料、統計、監査資料、法人資料、公開ウェブページの保存物
- URL、資料名、作成主体、公開日、取得日、対象期間、ページ等の取得記録
- 情報公開請求で取得した文書および開示・不開示等の通知

## 3. Codexが行ってよい作業

- ユーザーの指示に基づく資料の保存、命名、分類、重複候補の表示
- 出典メタデータ、取得記録、チェックサム、目録の作成
- OCRや文字抽出用の複製を`02_processed`に作成すること
- 資料不足や情報公開請求候補を、未確認事項として別途提示すること

## 4. Codexが行ってはいけない作業

- 原資料の本文、画像、日付、作成主体等を改変または上書きすること
- 出典不明の資料を確認済みの公的資料として扱うこと
- 欠落部分を推測で補完すること、推定を原資料に書き込むこと
- 違法性、妥当性、政治的評価、制度設計を最終判断すること

## 5. ハイエク軍への接続方法

各原資料を`00_index`の案件IDに紐づけ、抽出・整形物は`02_processed`へ渡す。`03_reports`からは資料名、ページ、URLまたは識別子、取得日を通じてこのフォルダの原資料へ遡れるようにする。

## 6. 運用上の注意

- 原資料は読み取り専用の証拠層として扱い、加工物と混在させない。
- 引用に使う資料は、引用箇所と前後の文脈を確認できる形で保存する。
- 個人情報、機密情報、著作権、利用規約、保存期限に注意し、必要最小限を扱う。
- リンク切れに備え、許容される範囲で取得日時と保存物または恒久識別子を残す。
