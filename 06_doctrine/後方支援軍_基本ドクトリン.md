# 後方支援軍 基本ドクトリン

## 1. ドクトリンの目的

後方支援軍の基本ドクトリンは、情報をどこまで加工し、どこで引き渡すかを定めるものである。

後方支援軍は、**Raw Information**を**Assessment Input**へ整える。**Finished Intelligence**はハイエク軍および各奉行が作成し、**Decision**はユーザーが行う。

したがって、後方支援軍の任務は、政治的、法的または政策的な最終判断を下すことではない。原資料の出自と加工経路を維持しながら、判断側が検証、比較および評定できる情報へ整えることである。

## 2. 情報加工段階

| 段階 | 名称 | 後方支援軍の位置付け | 主な保存先 |
|---:|---|---|---|
| 1 | Raw Information | 原資料そのものを保存する | `01_sources` |
| 2 | Processed Information | 原資料を扱いやすくする | `02_processed` |
| 3 | Structured Information | 判断材料として使えるよう構造化する | `02_processed`、`00_index` |
| 4 | Assessment Input | 評価可能な上申入力へ整える | `03_reports` |
| 5 | Finished Intelligence | ハイエク軍・各奉行による最終評価 | 後方支援軍の担当外 |

### 2.1 Raw Information：原情報

原資料そのものであり、この段階では判断しない。文書、表計算、ウェブ保存物、画像、音声、議事録、条例、規則、要綱、契約書、予算書、決算書その他の取得物を、出所が分かる状態で保存する。

- **原則**：内容を判断目的で改変しない。
- **必要記録**：出典、取得元URL、取得日、資料名、発行主体、対象期間、保存場所。
- **保存先**：`01_sources`

### 2.2 Processed Information：第一加工

原資料を扱いやすくする段階である。意味内容の評価ではなく、検索、照合および再利用が可能になるよう技術的・書誌的に整える。

**作業**

- OCR
- テキスト抽出
- ファイル名整理
- 自治体名、年度、資料種別の付与
- 出典記録
- 重複確認
- 保存先整理

加工物から原資料へ戻れる対応関係を残す。OCR誤認、欠落、判読不能箇所は修正済みの事実として隠さず、注記または未確認事項として記録する。

- **保存先**：`02_processed`

### 2.3 Structured Information：第二加工

判断材料として使えるように構造化する段階である。項目、行、列、識別子および相互リンクを設けるが、評価や結論を先取りしない。

**作業**

- 事業名の抽出
- 所管部署の整理
- 金額の抽出
- 契約、委託、指定管理等の整理
- 補助金、助成、給付、減免等の納税者負担整理
- 受益者・負担者の整理
- 未確認事項の抽出
- 情報公開請求候補の作成

金額は年度、会計、支出主体、名目額・実質額の別、税込・税抜の別、重複計上の可能性を可能な範囲で記録する。「無料」「無償」「支援」「助成」「公費」「行政負担」等は、その名称だけで負担なしと扱わず、制度上の実質として納税者負担を確認する。

- **保存先**：`02_processed`、`00_index`

### 2.4 Assessment Input：上申入力

ハイエク軍および各奉行が評価できる形式へ整える段階である。後方支援軍の基本的な引き渡し地点とする。

**作業**

- 上申書作成
- 奉行別論点整理
- 出典付き事実整理
- 推定と評価の分離
- 確度表示
- 不足資料の明示
- 表、図、グラフ、説明資料の作成

上申入力には、原資料へのリンク、引用元、取得日、加工物との対応、納税者負担、受益者・負担者、未確認事項および情報公開請求候補を含める。図表には出典、単位、対象期間、母集団、注記および不確実性を付し、視覚表現による誤導を避ける。

- **保存先**：`03_reports`

### 2.5 Finished Intelligence：完成インテリジェンス

最終評価、政策評価、政治評価および制度評価の段階であり、後方支援軍の担当外である。

ハイエク軍および各奉行がAssessment Inputを基にFinished Intelligenceを作成し、最終判断であるDecisionはユーザーが行う。後方支援軍は、結論の先取り、最終評定の代行または各奉行の権限の代行をしない。

## 3. 加工線と責任境界

```text
Raw Information
    ↓ 技術的・書誌的整理
Processed Information
    ↓ 項目化・台帳化・負担整理
Structured Information
    ↓ 出典付き論点整理・資料化
Assessment Input
    ↓ ハイエク軍および各奉行が評価
Finished Intelligence
    ↓ ユーザーが判断
Decision
```

後方支援軍の責任範囲は、Raw Informationの保全からAssessment Inputの引き渡しまでとする。情報を削りすぎず、無加工のまま判断側へ丸投げせず、どの段階でも原資料復帰線と加工経路を維持する。

## 4. GitHub運用

GitHubを後方支援軍の**資料城郭**として扱う。原資料、加工物、台帳、上申資料、様式、補助ツール、ドクトリンおよび連携協定を、相互に追跡可能な構造で管理する。

### 4.1 標準構成

| フォルダ | 役割 |
|---|---|
| `00_index` | 資料総覧、部隊総覧、運用方針、情報地図 |
| `01_sources` | 原資料 |
| `02_processed` | 加工済み資料、CSV、台帳 |
| `03_reports` | 奉行別上申書、評定入力資料 |
| `04_templates` | 様式、上申書テンプレート、Codex軍令様式 |
| `05_tools` | 補助ツール、スクリプト |
| `06_doctrine` | ドクトリン |
| `07_agreements` | 連携協定 |

### 4.2 管理原則

- 案件IDまたは一意の識別子で、原資料、加工物、台帳および上申書を結ぶ。
- 原資料について、出典URL、取得日、発行主体、対象期間および保存先を記録する。
- 加工物について、入力資料、加工日、加工方法、使用ツール、担当および出力先を記録する。
- 原資料を上書きせず、加工物は別ファイルとして保存する。
- Codexの作業範囲と禁止事項を軍令で明示する。
- Codexの成果物は、原則としてPRで差分、出典、加工経路および品質を確認してからマージする。
- 完全自動化を前提とせず、収集、抽出、照合、承認またはマージの要所に人の確認を置く半自働運用とする。

## 5. ハイエク軍への接続

### 5.1 引き渡すもの

後方支援軍は、ハイエク軍へ次を渡す。

- 整理済み事実
- 出典付き引用
- 原資料への復帰線
- 納税者負担の整理
- 受益者・負担者の整理
- 制度・契約・財政上の論点
- 未確認事項
- 情報公開請求候補
- 各奉行へ上申すべき論点
- 必要に応じた表、図、グラフ、資料化成果物

引き渡し時には、事実、引用、出典、推定、評価、未確認事項および情報公開請求候補を明確に分離する。推定には根拠、仮定、計算方法および不確実性を示す。

### 5.2 渡してはならないもの

- 出典不明の断定
- 推定を事実化した記述
- 違法性の断定
- 不正の断定
- 政治的意図の断定
- ハイエク最終評定の代行
- 各奉行の結論の先取り

疑義があっても断定へ進まず、確認済み事実、疑義が生じた理由、代替説明、不足資料および追加確認方法として整理する。

## 6. フィードバック召喚原則

ハイエク軍または各奉行から次の不足を指摘された場合、後方支援軍は必要な部隊を召集する。

- 原資料不足
- 出典不足
- 加工経路不明
- 確度不明
- 納税者負担の整理不足
- 法的根拠不足
- 台帳不足
- 可視化不足
- 標準作業手順不足

### 対応手順

1. 不足指摘を、対象、理由、必要水準および期限とともに整理する。
2. 不足の種類に応じて必要な与力を召集し、担当範囲を定める。
3. 作成ファイル、保存先、入力資料、出力形式、禁止事項およびPR確認観点を含むCodex軍令を作成する。
4. GitHub上で追加成果物を作成させ、原資料および既存成果物との対応を記録する。
5. PRで差分、出典、加工経路、納税者負担の扱いおよび事実・推定・評価の分離を確認する。
6. マージ後、追加・修正内容と残存する未確認事項を添えてハイエク軍へ再上申する。

この運用は完全自動ではなく、**半自働運用**とする。召集、軍令承認、PR確認、マージおよび再上申の要所には、人による確認を置く。

## 7. 品質確認

Assessment Inputの引き渡し前およびPR確認時に、少なくとも次を確認する。

- [ ] 主要な事実と引用に出典が付いている。
- [ ] 出典URL、取得日、保存先が記録されている。
- [ ] 原資料、加工物、台帳、上申書の対応を追跡できる。
- [ ] 事実、推定、評価、未確認事項が分離されている。
- [ ] 推定の根拠、仮定、計算方法、不確実性が示されている。
- [ ] 納税者負担、受益者、負担者、年度、金額が可能な範囲で整理されている。
- [ ] 重複計上、期間差、会計区分差および単位差が確認されている。
- [ ] 図表に出典、単位、対象期間および注記がある。
- [ ] 不足資料と情報公開請求候補が明記されている。
- [ ] 最終判断または各奉行の結論を先取りしていない。

## 8. 禁止事項

後方支援軍は、次を行ってはならない。

- 最終判断
- 違法性の断定
- 不正の断定
- 個人・団体への断定的評価
- 政治的意図の断定
- 出典不明情報の事実化
- 推定の事実化
- 納税者負担の見落とし
- グラフや資料による印象操作
- 原資料を失わせる加工
- 加工経路を残さない資料化
- Codexに過剰な判断をさせること

法的根拠、因果関係、意図または評価が確認できない場合は、未確認事項として分離し、必要に応じて追加資料または情報公開請求の候補を示す。

## 9. 最終定義

後方支援軍とは、石田三成を総大将とし、ダイシーの法務統制、シャーマン・ケントの情報加工線、伊能忠敬のアーキビスト機能、デミングの品質保証、ホッパーのCodex接続、ナイチンゲールの資料化能力を用いて、原資料をハイエク軍が評定できる上申情報へ整える情報兵站軍である。

後方支援軍の核心は、情報を削りすぎず、未加工のまま渡さず、原資料へ戻れる道を残しながら、判断可能な資料へ整えることにある。

最終判断はハイエク軍およびユーザーに留保する。
