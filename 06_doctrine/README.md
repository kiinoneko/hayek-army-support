# 06_doctrine — ドクトリン管理

## 1. このフォルダの目的

後方支援軍の任務範囲、情報加工段階、責任境界、品質基準およびGitHub運用原則を定めるドクトリンを保存する。

## 2. 主要文書

- [後方支援軍 基本ドクトリン](後方支援軍_基本ドクトリン.md)：Raw InformationからAssessment Inputまでの加工線、ハイエク軍への接続、フィードバック召喚原則および禁止事項を定める。
- [石田三成 兵站行政ドクトリン](石田三成_兵站行政ドクトリン.md)：資料・台帳・進捗・Codex軍令・PR確認および上申経路を統制し、ハイエク軍が評定できる状態へ情報を流す実務原則を定める。

## 3. 運用上の注意

- ドクトリンの変更は、既存の原資料管理、台帳、テンプレートおよび上申手順との整合を確認する。
- 後方支援軍はFinished IntelligenceとDecisionを担当せず、最終判断はハイエク軍およびユーザーに留保する。
- Codexによる変更は、原則としてPRで差分と影響範囲を確認してからマージする。
