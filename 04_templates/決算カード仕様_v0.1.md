# 決算カード仕様 v0.1

## 1. 目的

本仕様書は、後方支援軍が、自治体、政府機関、外郭団体、社会福祉協議会、指定管理者、補助金団体、委託先団体、公益法人、行政事業等の決算・予算・財務資料を、`source_id` 付きで整理し、判断前の標準整理票として「決算カード」化するための仕様を定める。

決算カードは評価文ではなく、判断前の標準整理票である。根拠、数値、分母、分子、算出式、`source_id`、`actor_id`、`gap_id`、`warning_id`、`stop_id`、原文URL、QRコード欄を明示し、読み手が原資料へ戻れる構造にする。

## 2. 基本原則

- `source_id` なしの数値は正式カードに載せない。
- PDF存在確認とPDF本文確認を混同しない。
- 予算値、決算値、契約額、支払実績を混同しない。
- 補助金、委託料、交付金、制度収入を分離する。
- 比率には分母、分子、算出式を明記する。
- 人件費率だけで効率性評価を断定しない。
- FTE又は常勤換算が未確認の場合、1人当たり人件費を正式指標にしない。
- 未確認事項は `gap_id` で管理する。
- 注意事項は `warning_id` で管理する。
- 停止事項は `stop_id` で管理する。
- 政治行政アクターが関与する場合は `actor_id` を記録する。
- 原文URLが公開可能な場合はQRコード欄を設ける。
- 評価候補と評価断定を分離する。
- 情報公開請求、法的断定、外部発信、政策評価断定へ自動進行しない。

## 3. 対象範囲

本仕様の対象範囲は以下のとおりとする。

- 国の省庁、庁、委員会、審議会
- 都道府県、市町村、広域連合、一部事務組合
- 国会、地方議会、議会事務局
- 独立行政法人、地方独立行政法人
- 社会福祉協議会、外郭団体、第三セクター
- 補助金交付先団体
- 委託契約先団体
- 指定管理者
- 公益法人、一般社団法人、NPO法人等の公的関与団体
- 行政事業、政策事業、補助事業、委託事業

## 4. カード種別

| 種別 | 内容 | 使用条件 |
|---|---|---|
| 仮決算カード | 資料不足を明示した仮整理 | WARNING付きで使用 |
| 正式決算カード | `source_id`、本文、該当ページ、該当欄確認済み | 正式成果物 |
| 予算カード | 予算資料を整理 | 決算と混同禁止 |
| 比較カード | 複数団体・複数年度比較 | 比較可能性確認後 |
| 事業カード | 特定事業・委託・補助事業整理 | 事業単位で使用 |

## 5. 正式カード化条件

正式カード化するには、以下をすべて確認する。

- 使用する主要数値に `source_id` がある。
- 資料本文を確認している。
- 該当ページと該当欄を記録している。
- 年度又は対象期間が確認されている。
- 予算値、決算値、契約額、支払実績を分離している。
- 比率には分母、分子、算出式を明記している。
- 補助金、委託料、交付金、制度収入を分離している。
- 人件費の対象範囲を明記している。
- FTE未確認の場合、1人当たり人件費を正式指標化しない。
- `gap_id`、`warning_id`、`stop_id` を表示している。
- 関与者がいる場合は `actor_id` を表示している。
- 原文URL又は資料URLがある場合はURL欄とQRコード欄を設けている。
- 評価候補と断定不可事項を分離している。

## 6. QRコード欄の扱い

- 原文URLが公開可能かつ安定している場合、QRコード添付を推奨する。
- QRコードには対応する `source_id` を必ず併記する。
- QRコードは公式資料、公式PDF、公式議事録、公式公開ページを原則対象とする。
- ログイン必須URL、一時共有URL、個人情報を含むURL、非公開資料URLはQR化しない。
- QRコードだけでなく、資料名、公開主体、取得日又は確認日、`source_id` を併記する。
- QRコードは検証導線であり、根拠そのものではない。
- URL切れに備え、`source_id` 台帳で保存先、取得日、本文確認状態を管理する。

## 7. actor_id欄の扱い

- 公的意思決定、議決、答弁、監査、補助金交付、委託契約、指定管理、同意人事、外郭団体運営、公費支出に関与した者は、可能な限り `actor_id` で記録する。
- 個人名、役職、所属、関与内容、関与日、`source_id` を記録する。
- `actor_id` は個人攻撃ではなく、公的権限・公費・公的影響に対する説明責任の記録である。
- 家族、住所、連絡先、病歴、思想信条、私生活など、公的職務と無関係な個人情報は扱わない。
- 未確認の場合は `gap_id` として管理し、個人責任を断定しない。

## 16. WARNING / STOP / USER_DECISION

### WARNING

| warning_id | 内容 | 対応 |
|---|---|---|
| WARN_SRC | `source_id` が未付与又は不十分 | 正式カード化前に出典を確定する |
| WARN_PAGE | 該当ページ未確認 | ページ番号又は該当箇所を確認する |
| WARN_FIELD | 該当欄未確認 | 欄名、表名、注記を確認する |
| WARN_DENOM | 分母・分子・算出式が不明 | 比率を正式指標にしない |
| WARN_FTE | FTE又は常勤換算未確認 | 1人当たり人件費を正式指標にしない |
| WARN_DIFF | 合計、単位、年度等に差異がある | 差異理由を確認する |
| WARN_SCOPE | 対象範囲が不明又は混在 | 団体、事業、会計、期間を分離する |
| WARN_COMPARE | 比較可能性に注意が必要 | 比較条件を明示する |
| WARN_ACTOR | 関与者情報が未確認又は不完全 | `gap_id` として管理し、個人責任を断定しない |

### STOP

| stop_id | 内容 | 停止条件 |
|---|---|---|
| STOP_DOC | 資料本文を確認できていない | 正式カード化を停止する |
| STOP_SOURCE | 主要数値に `source_id` がない | 正式カード化を停止する |
| STOP_VALUE | 数値の根拠、単位、年度、会計区分が不明 | 数値利用を停止する |
| STOP_SCOPE | 対象範囲が混在し分離できない | 比較又は評価候補化を停止する |
| STOP_PRIVACY | 個人情報、非公開資料、黒塗り前資料に該当する | GitHub追加、QR化、外部発信を停止する |

## 17. 禁止事項

- `source_id` なしの数値を正式カードに載せない。
- PDF本文未確認の数値を正式カードに載せない。
- 予算値と決算値を混同しない。
