# 公開前チェックリスト v0.1

## 1. 公開前チェックリストの位置づけ

公開前チェックリストは、外部に出る直前の最終関所である。

公開前チェックは、GitHub公開、PDF化、カード化、QRコード添付、Google Drive共有、外部発信、情報公開請求提出前に行う。

AIは公開前チェックを補助できるが、公開判断を代替しない。

公開範囲、外部発信、情報公開請求提出、法的断定、政策評価断定は、必要に応じて `decision_id` を要求する。

- `stop_id` がある場合は公開しない。
- `gap_id` が残る場合は断定しない。
- `warning_id` がある場合は公開文に影響範囲を明記する。

## 2. 公開対象区分

| 公開対象 | 内容 | 主な確認 |
|---|---|---|
| GitHub公開 | Markdown、テンプレート、仕様書、README等 | secret、個人情報、PDF原本、archive変更 |
| PDF化 | カード、報告書、資料化成果物 | source_id、未確認事項、断定表現 |
| QRコード添付 | 原文URL、資料URL、公開ページURL | 公開可能URLか、secret入りURLでないか |
| Google Drive共有 | PDF原本、開示資料、作業資料 | 共有範囲、黒塗り前資料、個人情報 |
| 情報公開請求提出 | 請求文案、照会文案 | decision_id、request_id、請求範囲 |
| 外部発信 | SNS、ブログ、配布資料、提出資料 | 誤断定、名誉毀損、原文確認 |
| 内部共有 | 後方支援軍内、別AI、法務官、中核軍 | handoff_id、stop_id、warning_id |

## 3. 公開可否の基本判定

| 判定 | 意味 | 処理 |
|---|---|---|
| 公開可 | 必要確認済み。公開してよい | 公開又は提出へ進む |
| 条件付き公開可 | 注意事項付きで公開可 | warning_id、注記、未確認事項を明記 |
| 保留 | 判断又は追加確認が必要 | USER_DECISION又はsource確認 |
| 公開不可 | STOP条件あり | 公開しない |
| 差し戻し | 内容修正が必要 | 修正後に再チェック |

- [ ] `.env` 実ファイルを含まない

- [ ] service account JSONを含まない
- [ ] Cookie又はsession情報を含まない

- secretをチャット、PR本文、GitHub、PDF、QRコードに含めない。

## 17. USER_DECISIONが必要な場面

- 外部発信する
- 情報公開請求を提出する
- 請求文案を提出用最終案にする
- 政策評価を断定調にする
- 法務官へ引き渡す
- 中核軍へ引き渡す
- 前線部隊へ引き渡す
- STOPを解除する
- GitHubへ公開する
- archive配下を変更する
- 黒塗り後資料を公開可能と判断する
- インシデント対応を終了扱いにする

## 19. WARNING / STOP

| ID | 条件 | 対応 |
|---|---|---|
| `WARN_PUBLIC_SCOPE` | 公開範囲が広い | USER_DECISION確認 |
| `WARN_QR_URL` | QRコードURLの公開可否注意 | URL確認 |
| `WARN_GAP_REMAIN` | gap_idが残る | 断定回避 |
| `WARN_QUOTE_CONTEXT` | 発言文脈注意 | 文脈明記 |
| `STOP_PUBLIC` | 公開可否未判断 | 公開停止 |
| `STOP_SECRET` | secret混入 | 公開停止、incident_id発行 |
| `STOP_PRIVACY` | 個人情報混入 | 公開停止 |
| `STOP_REDACTION` | 黒塗り前資料混入 | 公開停止 |
| `STOP_SOURCE` | source_idなし | 正式化停止 |
| `STOP_QUOTE` | 発言原文未確認 | quote_id正式化停止 |
| `STOP_LEGAL` | 法的断定リスク | 法務官確認 |
| `STOP_DECISION` | USER_DECISION不足 | 判断取得まで停止 |

## 20. 禁止事項

- 公開前チェックなしに外部公開しない
- source_idなしに確認済み事実として公開しない
- gap_idを隠して断定しない
- warning_idを無視して公開しない
- stop_idがある状態で公開しない
- decision_idなしにUSER_DECISION事項を進めない
- PDF原本をGitHubに追加しない
- 個人情報をGitHubに追加しない
- 黒塗り前資料を公開しない
- APIキー、パスワード、トークンを公開しない
- secret入りURLをQRコード化しない
- 発言原文未確認のままquote_id化しない
- 違法、不正、癒着、汚職を根拠なく断定しない
- AI判断だけで外部発信しない
- archive配下をUSER_DECISIONなしに変更しない
