# 後方支援軍ID台帳テンプレート v0.1

## 1. 使い方

このテンプレートは、後方支援軍で扱う案件ごとに `case_id` / `source_id` / `gap_id` / `warning_id` / `stop_id` / `decision_id` / `request_id` を追跡するための台帳です。

1. 案件を開始したら、まず `case_id` 台帳に1行を追加する。
2. 参照・取得した資料は、資料単位で `source_id` 台帳に追加する。
3. 不足、注意、停止に該当する事項があれば、該当するID台帳に追加する。
4. 判断を行った場合は、`decision_id` 台帳に判断経緯を残す。
5. 情報公開請求の候補がある場合は、`request_id` 台帳に追加する。
6. ID同士の関係は、`ID間接続表` に整理する。
7. 公開前チェックへ引き渡す事項は、`公開前チェック引渡表` にまとめる。

## 11. 記入ルール

- IDは案件内で重複させない。
- IDの形式は、原則として `種別-YYYY-連番3桁` とする。
- 日付は `YYYY-MM-DD` 形式で記入する。
- 状態は、原則として次の分類から選ぶ。
  - 未着手
  - 確認中
  - 追加資料待ち
  - 情報公開請求候補
  - 注意付き使用可
  - 停止中
  - 解除済み
  - 完了
  - 取下げ
  - 公開前再確認必要
- 複数IDを記入する場合は、カンマ区切りで記入する。
- 根拠資料が未確認の場合は、状態または備考に未確認であることを明記する。
- 公開前チェックに引き渡す必要がある事項は、該当台帳の `公開前チェック引渡要否` を `要` とし、公開前チェック引渡表にも追加する。
- source_id台帳の確認状態は、未確認 / 確認中 / 確認済み / 該当なし / 要再確認 など、後続工程で判別できる値で記入する。
- source_id台帳の公開使用可は、内部メモ、要約、数値、直接引用、公開用PDFの使用範囲ごとに、未判断 / 可 / 条件付き可 / 不可 / 要再確認 などで分けて記入する。
- source_id台帳では、確認済み事実と推測・未確認事項を必ず分けて記録し、推測を確認済み事実欄に記入しない。

## 12. 禁止事項

- PDF原本、画像、個人情報、secret、APIキーをこの台帳に貼り付けない。
- 既存Runbook本文をコピーして混入させない。
- 確認していない資料を確認済みとして扱わない。
- 推測を事実として記入しない。
- 解除条件を満たしていない `gap_id` / `warning_id` / `stop_id` を解除済みにしない。
- 公開前再確認が必要な事項を、引渡表に記入せずに公開工程へ渡さない。
