# 引渡テンプレート v0.1

## 1. 引渡テンプレートの定義

本テンプレートは、後方支援軍が、収集・資料管理・分析・資料化・統制確認を終えた案件について、中核軍、ダイシー型法務官、前線部隊、ナイチンゲール型資料化担当官、ユーザーへ引き渡すための標準テンプレートである。

引渡にあたっては、結論を急がず、根拠、未確認事項、WARNING、STOP、USER_DECISION、source_id、gap_id、actor_id、quote_id、policy_idを整理し、次の判断者が誤認しない状態で受け取れるようにする。

後方支援軍は、事実整理、根拠整理、未確認事項の明示、統制確認までを担当し、最終判断、法的断定、政策評価断定、外部発信判断を代替しない。

## 2. 使用目的

本テンプレートの使用目的は以下のとおりである。

- 案件の確認済み事実、算出事項、推定、評価候補、断定不可事項を分離する。
- source_id、gap_id、warning_id、stop_id、actor_id、quote_id、policy_idを次工程へ引き継ぐ。
- PDF本文確認状態、該当ページ、該当欄、原文URL、QRコード欄を明確にする。
- 情報公開請求、法務確認、資料化、議会対応、外部発信、ユーザー判断へ進む前に、未確認事項と禁止条件を明示する。
- 次の判断者が、根拠の有無、未確認事項、危険表現、停止条件、ユーザー判断事項を誤認しないようにする。

## 3. 引渡先の種類

| 引渡先 | 目的 |
| --- | --- |
| ハイエク中核軍 | 自由秩序、公共選択、財政制約、制度評価の観点で評価候補を検討する |
| ダイシー型法務官 | 法の支配、条例、要綱、契約、同意人事、情報公開請求、行政手続上の論点を確認する |
| 前線部隊 | 請願、陳情、質問案、公開質問、議会対応、住民向け説明へ展開する |
| ナイチンゲール型資料化担当官 | A4カード、決算カード、比較表、概要資料へ整える |
| ユーザー | 情報公開請求、外部発信、法務官確認、政策評価断定、追加調査などの最終判断を行う |

## 4. 引渡前チェック

引渡前に、以下を確認する。

- [ ] case_idがあるか
- [ ] case_nameがあるか
- [ ] 対象団体・事業・人物が明確か
- [ ] 年度又は対象期間が明確か
- [ ] source_idが整理されているか
- [ ] PDF本文確認状態が明記されているか
- [ ] 該当ページ・該当欄が明記されているか
- [ ] 事実、算出事項、推定、評価候補、断定不可事項が分離されているか
- [ ] 数値の分母・分子・算出式が明記されているか
- [ ] actor_idが必要な場合に整理されているか
- [ ] quote_idが必要な場合に発言原文・日付・媒体・文脈が確認されているか
- [ ] gap_idが整理されているか
- [ ] warning_id / stop_idが整理されているか
- [ ] USER_DECISION事項が明記されているか

## 5. 引渡禁止条件

以下に該当する場合は、引渡を禁止し、差戻し又はUSER_DECISIONへ回す。

- source_idなしの数値を正式根拠として扱っている
- PDF存在確認のみで本文確認済みとしている
- 予算値、決算値、契約額、支払実績を混同している
- 補助金、委託料、交付金、制度収入を混同している
- 分母不明の比率を正式指標としている
- quote_idなしで発言を正式言質化している
- actor_idを根拠に個人責任、違法、不正、癒着、汚職を断定している
- gap_idを根拠の代替として扱っている
- WARNING / STOP / USER_DECISIONを省略している
- 情報公開請求、法的断定、外部発信、政策評価断定へ自動進行している
- PDF原本、個人情報、黒塗り前資料、APIキーが混入している

## 15. USER_DECISION事項

USER_DECISION事項は、後方支援軍が自動で進めてはならない最終判断を整理する欄である。推奨や誘導ではなく、判断事項、選択肢、関連根拠、未確認事項を分離して記載する。

| decision_id | 判断事項 | 選択肢 | 推奨ではなく整理 | 関連source_id / gap_id |
| --- | --- | --- | --- | --- |
|  |  |  |  |  |

## 16. 禁止事項

- 後方支援軍は最終判断を代替しない
- 情報公開請求へ自動進行しない
- 法的断定へ自動進行しない
- 外部発信へ自動進行しない
- 政策評価断定へ自動進行しない
- 個人責任を断定しない
- source_idなしの事実を正式化しない
- gap_idを根拠として扱わない
- PDF原本、個人情報、黒塗り前資料をGitHubに追加しない
- archive配下を変更しない
