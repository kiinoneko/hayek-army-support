# 02_processed — 抽出・整形資料

## 1. そのフォルダの目的

`01_sources`の原資料を、検証、比較、集計、報告書作成に使える作業資料へ変換する。

## 2. 保存する資料の種類

- OCR結果、文字起こし、表抽出、正規化データ、比較表、時系列表
- 納税者負担台帳、主体・制度・契約・資金経路の整理表
- 事実、引用、出典、推定、評価、未確認事項、情報公開請求候補を分離した分析メモ
- 変換手順、計算式、仮定、検証結果、原資料との対応表

## 3. Codexが行ってよい作業

- OCR、抽出、整形、項目統一、集計、比較、重複候補の表示
- 補助金、助成、給付、減免、委託、指定管理、外郭団体支援、公費投入、施設維持費、行政サービス拡充を納税者負担として台帳化すること
- 推定値の計算と感度整理（推定である旨、根拠、方法、仮定、不確実性を明記する場合に限る）
- 未確認事項と情報公開請求候補の抽出、報告書用の論点整理

## 4. Codexが行ってはいけない作業

- 原資料にない内容を事実または引用として生成すること
- 推定と確認済み事実を混同し、推定を断定すること
- 都合の悪いデータ、反対方向の解釈、不確実性を削除すること
- 違法性、妥当性、政治的評価、制度設計を最終判断すること

## 5. ハイエク軍への接続方法

すべての作業資料に`00_index`の案件IDと`01_sources`の出典参照を付ける。`04_templates`に沿って整理した内容を`03_reports`へ渡し、ハイエク軍が報告書から根拠と計算過程へ遡れるようにする。

## 6. 運用上の注意

- 情報区分ごとに見出し、列、ファイルを分け、引用は原文と要約を区別する。
- 単位、年度、名目・実質、税込・税抜、予算・決算、支出・減収を明記する。
- 納税者負担の合計では、国・自治体間移転、再委託、複数制度による重複を点検する。
- 加工日時、使用ツール、変換手順を残し、可能な限り再現可能にする。

## 7. 実戦投入用台帳

- [納税者負担台帳](納税者負担台帳.csv)：補助金、助成、給付、減免、委託、指定管理、外郭団体支援、公費投入、施設維持費、行政サービス拡充等を、事実・推定・未確認事項に分けて整理する。

### 負担区分

`補助金`、`助成`、`給付`、`減免`、`委託`、`指定管理`、`外郭団体支援`、`公費投入`、`施設維持費`、`行政サービス拡充`、`その他`のいずれかを記載する。

- 「無料」「無償」「支援」「助成」「公費」「行政負担」等の表現は、その名称だけで判断せず、制度上の実質として納税者負担を確認する。
- 金額が不明な場合は空欄または「未確認」とし、推定額を確認済み金額として記載しない。
- 受益者と負担者を可能な範囲で分け、推定は根拠、方法、仮定、不確実性とともに推定欄へ記載する。
- 台帳には最終評価を記載しない。
