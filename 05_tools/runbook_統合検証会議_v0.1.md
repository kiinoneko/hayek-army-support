# Runbook: 統合検証会議 v0.1

## 1. 位置づけ

統合検証会議は、後方支援軍、ハイエク軍、前線部隊、必要に応じて法務官又は外部確認役を接続し、公開前、重要案件、外部発信用成果物を横断確認するための上位ゲートである。

このRunbookは、個別Runbookの代替ではない。後方支援軍の原文資料相互検証Runbook、ID台帳テンプレート、公開前チェックRunbookで整理された確認状態、判定状態、引渡状態を会議で統合し、次工程へ進める条件を明確にする。

統合検証会議では、後方支援軍だけで重要案件を最終判断しない。原文確認、`source_id`確認状態、`gap_id` / `warning_id` / `stop_id`、ID台帳、公開前チェック、制度評価、公開表現リスクを分けて確認する。

`source_id`が存在するだけでは、原文確認済み、引用確認済み、数値確認済み、公開使用可とは扱わない。公開使用可は、内部メモ、要約、数値、直接引用、公開用PDF、外部発信で分けて判断する。

このRunbookは、現時点では文書上の運用統制であり、コード、CI、GitHub Actions、入力フォーム、台帳バリデーションによる技術的強制ではない。したがって、`stop_id`未解除のまま記録上「公開可」と記入できてしまう可能性は残る。この技術的歯止めは別工程で整備する。技術的強制が未整備の間は、統合検証会議、ID台帳、公開前チェック、最終人間判定で二重確認する。

## 2. 適用対象

このRunbookは、次の案件又は成果物に適用する。

- 公開前の成果物
- 重要案件として扱う案件
- 外部発信用文章、投稿案、説明資料
- 公開用PDF化を検討する資料
- 情報公開請求へ接続する可能性がある案件
- 法務官引渡又は慎重確認が必要な案件
- 金額、発言者名、団体名、制度評価、納税者負担を含む成果物
- `gap_id` / `warning_id` / `stop_id` が残っている案件
- 原文資料相互検証結果を複数部門で確認する必要がある案件

通常の内部メモ、作業途中の草稿、公開予定のない軽微な整理は、必要に応じて簡易確認で扱う。ただし、途中で外部発信、公開用PDF化、法務官引渡、情報公開請求へ進める可能性が出た場合は、このRunbookへ接続する。

## 3. 参加部門と役割

| 参加部門 | 主な役割 | 会議で確認する観点 |
| --- | --- | --- |
| 後方支援軍 | 根拠資料、原文確認、ID台帳、公開前チェックの準備を行う。 | `source_id`確認状態、原文確認結果、数値確認、引用確認、`gap_id` / `warning_id` / `stop_id`、台帳整合性を確認する。 |
| ハイエク軍 | 制度、自由秩序、納税者負担の評価を行う。 | 自由、財産権、法の支配、分散知、行政裁量、利益団体化、制度効果を確認する。 |
| 前線部隊 | 公開可能性と表現リスクを確認する。 | 読み手、公開目的、表現の強さ、断定度、炎上リスク、誤読リスク、情報公開請求への接続可能性を確認する。 |
| 法務官 | 法的リスク又は法的評価が必要な場合に確認する。 | 違法、不正、虚偽、利益供与、名誉毀損、個人情報、非公開情報、黒塗り前情報の扱いを確認する。 |
| 外部確認役 | 必要に応じて第三者視点で確認する。 | 読み手の誤読可能性、根拠不足、説明不足、公開時の受け止めを確認する。 |

参加部門の不在がある場合は、その観点を確認済みとして扱わない。不在部門の確認が必要な場合は、判定を保留し、追加確認後再審査へ回す。

## 4. 最終責任者と判定権限

統合検証会議に参加する各AI部門は、確認、評価、警告、停止提案を行う補助機関であり、公開、外部発信、公開用PDF化、個人名又は団体名を伴う評価の最終責任者ではない。

最終判定は、人間責任者である猫さんが確定する。猫さんが最終判定を行う場合でも、未確認事項、注意事項、停止条件、法務官確認の要否、公開使用可の範囲をID台帳又は`decision_id`に記録する。

AI部門の合議だけで、実名の個人、団体、部署、事業者に対する外部公開評価を完結させない。名誉毀損、信用毀損、個人情報、非公開情報、黒塗り前情報、法的評価が関係する場合は、猫さんの最終確認に加え、必要に応じて実弁護士又は外部専門家への確認を検討する。

## 5. 会議を開く条件

次のいずれかに該当する場合は、統合検証会議を開く。

- 外部発信を予定している。
- 公開用PDF化を予定している。
- 重要断定、金額確定、発言者名を伴う批判を含む。
- 違法、不正、虚偽、利益供与、隠蔽、改ざんなどの表現が候補に出ている。
- 個人名、団体名、部署名、事業者名を伴う批判又は評価を含む。
- 納税者負担、財産権、行政裁量、利益団体化、制度効果を扱う。
- `gap_id` / `warning_id` / `stop_id` が未処理である。
- 原文未確認、引用未確認、数値未確認、反証未確認が残っている。
- 情報公開請求へ接続する可能性がある。
- 法務官又は外部確認役への引渡が必要になる可能性がある。

## 6. 判定が割れた場合の優先順位

部門間で判定が割れた場合は、次の優先順位で扱う。

| 優先 | 状態 | 扱い |
| --- | --- | --- |
| 1 | `stop_id`継続、個人情報、secret、API key、黒塗り前情報、非公開情報の疑い | 公開不可又は停止継続を暫定採用する。 |
| 2 | 法務官確認又は実弁護士確認が必要 | 公開可にせず、法務確認又は慎重確認へ回す。 |
| 3 | 原文未確認、引用未確認、数値未確認、反証未確認 | 重要断定、金額確定、発言者名を伴う批判、公開用PDF化、外部発信に進めない。 |
| 4 | `warning_id`が残る | 条件付き公開可、表現修正、注意書き、内部使用のみを検討する。 |
| 5 | 全部門で確認済み、停止条件なし | 用途別に公開可を検討できる。 |

判定が割れた場合は、原則として最も保守的な判定を暫定採用する。保守的判定を変更する場合は、猫さんが理由を確認し、`decision_id`に判断理由、残存リスク、公開範囲、責任者を記録する。ただし、未確認事項を確認済みに変更することはできない。

## 7. 事前準備資料

会議前に、後方支援軍は次の資料を準備する。

| 資料 | 必須状態 | 備考 |
| --- | --- | --- |
| 対象成果物 | 最新版が特定されている。 | 草稿、公開案、PDF化候補、外部発信案を混同しない。 |
| `case_id` | 対象案件と成果物が接続されている。 | 未起票の場合は会議で最終判定しない。 |
| `source_id`一覧 | 資料名、取得日、確認状態が整理されている。 | ID登録済みと原文確認済みを分ける。 |
| 原文資料相互検証結果 | 原文、ページ、引用、数値、反証の確認状態が分かる。 | 未確認項目は未確認として残す。 |
| `gap_id`一覧 | 未確認事項と解消方法が整理されている。 | 解消済み、未処理、情報公開請求候補を分ける。 |
| `warning_id`一覧 | 注意付き使用の理由が整理されている。 | 公開時に明示する注意点を分ける。 |
| `stop_id`一覧 | 停止理由と再開条件が整理されている。 | 解除判断を会議で明示する。 |
| ID台帳 | `case_id`、`source_id`、`quote_id`、`actor_id`、`policy_id`等の接続が確認できる。 | 台帳未整備の場合は公開判定へ進めない。 |
| 公開前チェック結果 | 公開範囲、公開手段、PDF化、外部発信、法務官引渡の確認状態が分かる。 | 公開可否ではなく確認状態として提出する。 |

PDF、画像、個人情報、secret、API key、黒塗り前情報そのものはGitHubへ追加しない。実案件の記録は、このRunbookへ追記しない。

## 8. source_id確認状態の確認

会議では、各`source_id`について次の状態を分けて確認する。

| 確認状態 | 確認内容 | 会議での扱い |
| --- | --- | --- |
| source_id登録済み | 資料候補又は根拠資料として`source_id`を登録した。 | これだけでは根拠確定、引用可能、公開使用可にしない。 |
| URL確認済み | URL、保存場所、資料名、公開主体、取得日を確認した。 | 本文、数値、引用の確認済みとは分ける。 |
| 本文確認済み | 対象本文を原文で確認した。 | 要約、概略説明の根拠候補にできる。 |
| ページ確認済み | ページ、表、見出し、発言箇所を特定した。 | 数値、引用、発言確認の前提にできる。 |
| 引用確認済み | 引用文と原文の一致、省略、前後文脈を確認した。 | 直接引用の候補にできる。 |
| 数値確認済み | 金額、件数、年度、単位、算式を確認した。 | 数値を成果物へ反映する候補にできる。 |
| 別資料照合済み | 別資料、原資料、要約資料、最新版との照合を確認した。 | 公開使用可、断定度、注意付き使用の判断材料にする。 |
| 反証確認済み | 反証、訂正版、別解釈、前後文脈を確認した。 | 断定度と公開使用可の判断材料にする。 |
| 公開使用可 | 必要な確認が完了し、停止条件がない。 | 用途別に公開可否を判断する。 |

原文未確認、引用未確認、数値未確認、反証未確認がある場合は、重要断定、金額確定、発言者名を伴う批判、公開用PDF化、外部発信に進めない。

## 9. 原文資料相互検証結果の確認

原文資料相互検証結果は、次の順に確認する。

1. 対象成果物の主張、数値、引用、評価が一覧化されているか確認する。
2. 各主張に`source_id`が接続されているか確認する。
3. `source_id`ごとの原文、ページ、表、発言箇所、取得日、版を確認する。
4. 直接引用と要約が混同されていないか確認する。
5. 金額、年度、単位、税込・税抜、対象期間、算式を確認する。
6. 要約資料、議会だより、SNS投稿、動画切り取りを原文の代替として扱っていないか確認する。
7. 反証、訂正版、差替版、前後文脈、別資料照合の確認状態を確認する。
8. 未確認項目を`gap_id`、注意事項を`warning_id`、停止条件を`stop_id`へ接続しているか確認する。

確認できない項目がある場合は、未確認として記録する。会議中に推測で確認済みに変更しない。

## 10. gap_id / warning_id / stop_id の確認

会議では、`gap_id` / `warning_id` / `stop_id` を次のように扱う。

| ID | 会議で確認すること | 判定の方向 |
| --- | --- | --- |
| `gap_id` | 未確認事項、解消方法、解消期限、情報公開請求への接続要否を確認する。 | 解消済み、追加確認、情報公開請求へ接続、未確認のまま内部使用に限定のいずれかを明示する。 |
| `warning_id` | 注意付き使用の理由、公開時に明示すべき注意、断定度の調整を確認する。 | 注意付き使用、表現修正、内部使用のみ、再確認のいずれかを明示する。 |
| `stop_id` | 停止理由、再開条件、解除の可否、継続理由を確認する。 | 解除、条件付き解除、停止継続、公開不可のいずれかを明示する。 |

`gap_id` / `warning_id` / `stop_id` が未処理の場合は、会議で解除、注意付き使用、停止継続を明確にする。未処理のまま公開可とはしない。

## 11. ID台帳との照合

会議では、対象成果物とID台帳を照合する。

- `case_id`が対象案件と成果物を正しく接続しているか。
- `source_id`が事実、金額、年度、引用の根拠資料に接続しているか。
- `quote_id`が直接引用の範囲、原文、ページ、発言者と接続しているか。
- `actor_id`が個人、団体、部署、役職、発言者を混同していないか。
- `policy_id`が制度、施策、事業、契約を取り違えていないか。
- `gap_id` / `warning_id` / `stop_id`が成果物本文、公開前チェック、会議判定に接続しているか。
- `request_id`が情報公開請求へ接続する未確認事項を示しているか。
- `decision_id`が外部発信、公開用PDF化、法務官引渡、評価断定の判断に接続しているか。

台帳と成果物が不一致の場合は、公開判定に進めず、台帳修正又は成果物修正へ戻す。

## 12. ハイエク軍による制度・自由秩序・納税者負担評価

ハイエク軍は、次の観点を確認する。

| 観点 | 確認内容 |
| --- | --- |
| 自由 | 個人、事業者、地域社会の選択可能性を不必要に狭めていないか。 |
| 財産権 | 課税、補助、規制、契約が財産権に与える影響を確認したか。 |
| 法の支配 | 裁量、例外、非公開運用、恣意的運用の可能性を確認したか。 |
| 分散知 | 現場、住民、事業者が持つ情報を制度が活用又は阻害していないか。 |
| 納税者負担 | 金額、負担主体、費用対効果、将来負担を確認したか。 |
| 行政裁量 | 裁量の範囲、基準、透明性、説明責任を確認したか。 |
| 利益団体化 | 特定団体、受益者、委託先、補助対象への偏りを確認したか。 |
| 制度効果 | 意図した効果、逆効果、副作用、代替手段を確認したか。 |

ハイエク軍の評価は、法的断定又は事実認定の代替ではない。制度評価、価値判断、政策提案は、原文確認済みの事実と分けて記録する。

## 13. 前線部隊による公開可能性・表現リスク確認

前線部隊は、次の観点を確認する。

前線部隊による確認は、猫さんが対象成果物、公開予定の有無、想定読者、公開手段、重点確認観点を指定することで開始する。

- 想定する読み手が明確か。
- 公開目的が説明可能か。
- 表現の強さが確認状態に見合っているか。
- 断定度を下げる必要がないか。
- 炎上リスク、誤読リスク、切り取りリスクがないか。
- 発言者名、団体名、部署名を伴う批判が過度に強くないか。
- 成果物内で、後方支援軍が確認した事実記述と、ハイエク軍による制度評価・価値判断・政策提案が見出し単位で分けられているか。
- 未確認事項、注意事項、停止条件を隠していないか。
- 情報公開請求への接続可能性があるか。
- 公開用PDF、SNS投稿、説明文、広報PR本文で同じ断定度を使ってよいか。

公開目的が不明確な場合、又は表現リスクが確認状態を上回る場合は、条件付き公開可、内部使用のみ可、追加確認後再審査、情報公開請求へ接続のいずれかを検討する。

## 14. 法務官又は慎重確認が必要な条件

このRunbookにおける「法務官」は、原則として内部AI役割又は法務リスク確認役を指す。実際の弁護士資格を有する外部専門家を意味する場合は、「実弁護士」又は「外部弁護士」と明記する。AI法務官による確認は、実弁護士の法的助言を代替しない。

次のいずれかに該当する場合は、法務官接続又は慎重確認を検討する。

- 違法、不正、虚偽、利益供与、癒着、汚職、隠蔽、改ざんの表現を使う可能性がある。
- 個人名を伴う批判又は責任追及がある。
- 名誉毀損、信用毀損、業務妨害、プライバシー侵害のリスクがある。
- 個人情報、非公開情報、黒塗り前情報が関係する。
- 情報公開請求で取得した資料の公開範囲が不明である。
- 契約、補助金、指定管理、利益供与、利益相反に関する法的評価が必要である。
- 外部発信後に相手方から法的反応が想定される。
- 原文確認済みであっても、評価表現が強く、社会的評価を低下させる可能性がある。

法務官確認が必要な場合は、公開可否を会議だけで最終判断しない。法務官引渡資料では、事実、推定、評価、未確認事項、注意事項、停止条件を分けて記録する。

## 15. 判定区分

会議の判定区分は、次のいずれかとする。

`stop_id`が未解除の場合は、「公開可」と判定しない。法務官確認又は実弁護士確認が必要な状態では、「公開可」ではなく「追加確認後再審査」又は「stop_id継続」とする。

| 判定区分 | 意味 | 次工程 |
| --- | --- | --- |
| 公開可 | 必要な確認が完了し、停止条件がなく、用途別に公開可能と判断した。 | 公開前チェックへ進む。 |
| 条件付き公開可 | 表現修正、注意書き、用途制限、追加明記を条件に公開可能と判断した。 | 条件反映後、公開前チェックへ進む。 |
| 内部使用のみ可 | 内部メモ又は検討資料としては使えるが、外部発信、公開用PDF、直接引用には使わない。 | 内部保管又は追加確認へ戻す。 |
| 追加確認後再審査 | 原文、引用、数値、反証、台帳、法務確認の不足がある。 | 追加確認後、再度会議へ戻す。 |
| 情報公開請求へ接続 | 公開資料だけでは確認できず、請求で未確認事項を解消する必要がある。 | `request_id`を作成し、情報公開請求Runbookへ接続する。 |
| stop_id継続 | 停止条件が残っており、解除できない。 | 公開、PDF化、外部発信を停止する。 |
| 公開不可 | 公開に不適切な情報、重大な未確認、法的又は倫理的リスクがある。 | 公開しない。必要に応じて内部記録又は法務官へ引き渡す。 |

判定は用途別に分ける。内部メモ可であっても、直接引用可、公開用PDF可、外部発信可とは限らない。

## 16. 会議結果の記録

会議結果には、少なくとも次を記録する。

| 記録項目 | 記録内容 |
| --- | --- |
| 会議ID | 会議を識別するID。 |
| 対象`case_id` | 対象案件のID。 |
| 対象成果物 | ファイル名、版、作成日、公開予定の有無。 |
| 参加部門 | 後方支援軍、ハイエク軍、前線部隊、法務官、外部確認役の参加有無。 |
| 確認した`source_id` | 各`source_id`の確認状態。 |
| 未確認事項 | 残存する`gap_id`と扱い。 |
| 注意事項 | 残存する`warning_id`と公開時の扱い。 |
| 停止条件 | `stop_id`の解除、条件付き解除、継続、公開不可の判断。 |
| 用途別判定 | 内部メモ、要約、数値、直接引用、公開用PDF、外部発信の可否。 |
| 引渡状態 | 公開前チェック、追加確認、情報公開請求、法務官引渡への接続先。 |
| 最終判定者 | 猫さんによる最終判定又は保留判断を必ず記録する。 |
| `decision_id` | 統合検証会議の対象案件では必ず作成又は更新し、判断理由、残存リスク、公開範囲、責任者を記録する。 |

統合検証会議の対象となった案件では、猫さんによる最終判定又は保留判断を必須とする。AI部門の判定、会議結果、公開前チェック結果だけで最終判定済みとして扱わない。

会議結果は、確認状態、判定状態、引渡状態を分けて記録する。議論中の推測、未確認の断定、実案件の詳細、個人情報、secret、黒塗り前情報は記録しない。

## 17. 公開前チェックへの接続

会議で公開可又は条件付き公開可となった場合でも、公開前チェックを省略しない。

公開前チェックへ接続する際は、次を引き渡す。

- 用途別の公開可否
- 解除済み又は継続中の`gap_id` / `warning_id` / `stop_id`
- 公開時に明示する注意事項
- 表現修正条件
- PDF化の可否
- 外部発信の可否
- 情報公開請求へ接続する未確認事項
- 法務官確認の要否
- 事実記述と制度評価・価値判断・政策提案の見出し分離状態
- 猫さんによる最終判定又は保留判断
- 作成又は更新済みの`decision_id`

条件付き公開可の場合は、条件反映前に公開前チェックを完了扱いにしない。公開前チェックで新たな停止条件が見つかった場合は、会議判定にかかわらず停止する。

## 18. 禁止事項

統合検証会議では、次を禁止する。

- `source_id`があるだけで、原文確認済み、引用確認済み、数値確認済み、公開使用可として扱うこと。
- 原文未確認、引用未確認、数値未確認、反証未確認のまま、重要断定、金額確定、発言者名を伴う批判、公開用PDF化、外部発信へ進めること。
- `gap_id` / `warning_id` / `stop_id`を未処理のまま公開可にすること。
- 後方支援軍だけで重要案件の最終判断を行うこと。
- 要約資料、議会だより、SNS投稿、動画切り取りを原文の代替として扱うこと。
- 法的評価が必要な表現を、制度評価又は一般的感想として処理すること。
- 個人情報、secret、API key、黒塗り前情報、非公開情報をGitHubへ追加すること。
- 実案件の記録をこのRunbookへ追記すること。
- 公開前チェック、法務官確認、情報公開請求Runbookを省略すること。
- Markdown表を壊した状態でPRへ進めること。

## 19. PR前確認

このRunbook又は会議記録に関するPR前には、次を確認する。

- 新規ファイル、変更ファイル、削除ファイルの範囲が意図どおりである。
- README.md、archive、00_index、01_sources、04_templates、既存05_toolsファイルを誤って変更していない。
- PDF、画像、個人情報、secret、API keyを追加していない。
- 実案件の記録を追加していない。
- 章番号が重複していない。
- Markdown表が壊れていない。
- `stop_id`未解除のまま公開可としていない。
- 部門間判定が割れた場合の優先順位を記録している。
- 猫さんによる最終判定又は保留判断を必須項目として記録している。
- 統合検証会議の対象案件について、`decision_id`を必ず作成又は更新する運用になっている。
- 成果物内で、事実記述と制度評価・価値判断・政策提案を見出し単位で分ける運用になっている。
- 法務官がAI役割か、実弁護士確認かを混同していない。
- `source_id`確認状態、原文資料相互検証結果、`gap_id` / `warning_id` / `stop_id`、ID台帳、公開前チェックへの接続が本文内で分けて記載されている。
- 断定調ではなく、確認状態、判定状態、引渡状態を分ける運用文書になっている。
