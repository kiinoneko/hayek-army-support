# 議会発言確認Runbook v0.1

## 1. Runbookの位置づけ

議会発言確認Runbookは、政治行政アクターの発言を根拠資料に基づいて確認し、`quote_id`として整理するための作業手順書である。

発言確認は、批判や評価の前に、原文、発言者、媒体、日時、文脈を確定する作業である。

`quote_id`は発言そのものではなく、発言記録を追跡する索引である。`quote_id`は必ず`source_id`と接続する。`actor_id`は必ず発言者又は関係者を明示する。

公的職務に関係する政治行政アクターの氏名、役職、発言は、責任線を明確にするため記録対象とする。ただし、公的職務と無関係な私的情報、住所、電話番号、家族情報、健康情報等は扱わない。

発言原文未確認のまま、言質化、矛盾断定、虚偽断定、不正断定をしてはならない。

## 2. 対象

本Runbookは、以下の発言、記録、媒体、資料を対象とする。

- 本会議発言
- 委員会発言
- 一般質問
- 代表質問
- 議案質疑
- 討論
- 採決理由
- 議会だより掲載文
- 議事録
- 会議録
- 議会録画
- 首長会見
- 公式サイト掲載文
- 公式SNS投稿
- X投稿
- Facebook投稿
- Instagram投稿
- YouTube動画
- note、ブログ、後援会サイト
- 選挙公約
- マニフェスト
- チラシ、広報紙
- 報道コメント
- 審議会、協議会、検討会の発言
- 行政職員による公式説明
- 議会同意人事、監査委員、教育長等の公的職務上の発言

## 3. 使用する主なID

| ID | 用途 |
|---|---|
| `case_id` | 発言確認案件を管理する |
| `actor_id` | 発言者、関係者、責任線を管理する |
| `quote_id` | 発言、答弁、公約、投稿、説明を管理する |
| `source_id` | 議事録、動画、投稿、会見録等の根拠資料を管理する |
| `policy_id` | 発言に関連する政策、事業、議案、予算、契約を管理する |
| `gap_id` | 原文未確認、文脈未確認、動画未確認等を管理する |
| `warning_id` | 切り取り注意、文脈注意、要約注意を管理する |
| `stop_id` | 発言正式化、外部発信、断定評価の停止を管理する |
| `decision_id` | 外部発信、評価断定、法務官引渡の判断を管理する |
| `change_id` | 発言記録、要約、評価の修正履歴を管理する |
| `incident_id` | 誤引用、人物誤認、誤公開等を管理する |

## 4. 発言確認の基本原則

- 発言は「誰が」「いつ」「どこで」「何を」「どの文脈で」言ったかを確認する。
- 原文、要約、解釈、評価を分ける。
- 議事録、会議録、公式動画、本人投稿、公式資料を優先する。
- 動画だけの場合は、発言時刻、発言者、前後文脈を記録する。
- SNS投稿は投稿日時、URL、投稿者、削除・編集の可能性を記録する。
- 発言の一部だけを切り取って意味を変えない。
- 発言原文が確認できない場合は`quote_id`を正式化しない。
- 発言に基づく政策評価、矛盾指摘、外部発信は必要に応じて`decision_id`を要求する。

## 5. actor_id確認手順

| 確認項目 | 内容 |
|---|---|
| 氏名 | 公的資料、議員名簿、職員名簿、公式サイトで確認 |
| 役職 | 町長、副町長、議員、委員、教育長、監査委員等 |
| 所属 | 会派、部署、委員会、団体 |
| 任期・時点 | 発言時点の役職を確認 |
| 関連政策 | 発言が関連する政策、事業、議案 |
| 公的職務性 | 公的職務に関係する発言か確認 |

政治行政アクターは、影響力と責任線を持つため、氏名と役職を明記する。同姓同名、役職変更、会派変更、委員会変更に注意する。

`actor_id`だけで個人責任、故意、違法、不正を断定しない。公的職務に関係しない私的情報は記録しない。

## 6. source_id確認手順

| 資料種別 | source_id化の注意 |
|---|---|
| 議事録 | 会議名、開催日、ページ、発言者を記録 |
| 会議録 | 会議名、出席者、発言箇所を記録 |
| 議会録画 | URL、発言時刻、発言者、前後文脈を記録 |
| SNS投稿 | 投稿URL、投稿日時、投稿者、スクリーンショット有無を記録 |
| 公式サイト | ページ名、公開元、取得日を記録 |
| ブログ・note | URL、投稿者、投稿日、編集可能性を記録 |
| 報道記事 | 媒体名、公開日、引用範囲を記録 |
| 選挙公約 | 発行主体、発行日、対象選挙を記録 |

`source_id`なしに`quote_id`を正式化しない。SNSやブログは削除・編集可能性があるため`warning_id`を検討する。

報道記事は本人原文ではない場合があるため、可能な限り一次資料を優先する。動画は本文確認ではなく視聴確認として記録する。

## 7. quote_id作成手順

| 手順 | 内容 |
|---|---|
| 1 | `actor_id`を確認する |
| 2 | `source_id`を確認する |
| 3 | 発言日時を確認する |
| 4 | 発言場所又は媒体を確認する |
| 5 | 原文又は要旨を分けて記録する |
| 6 | 前後文脈を確認する |
| 7 | 関連`policy_id`を接続する |
| 8 | 文脈注意があれば`warning_id`を付与する |
| 9 | 原文未確認なら`gap_id`又は`stop_id`を付与する |
| 10 | `quote_id`を発行する |

`quote_id`は発言を追跡する索引であり、発言そのものではない。`quote_id`は必ず`source_id`と`actor_id`に接続する。原文、要約、評価を同じ欄に混在させない。

## 9. 動画・音声確認手順

| 確認項目 | 内容 |
|---|---|
| 動画URL | 公開可能URLか確認 |
| 媒体 | 議会録画、YouTube、会見動画等 |
| 発言時刻 | 開始時刻、終了時刻を記録 |
| 発言者 | 画面表示、議事進行、字幕等で確認 |
| 前後文脈 | 質問、答弁、再質問を確認 |
| 文字起こし | 自動字幕は誤認リスクあり |
| 保存状態 | 削除、公開期限、差し替え可能性を記録 |

自動字幕だけで原文確定しない。聞き取り不明箇所は推定で補わず`gap_id`化する。

動画URLをQRコード化する場合は公開可能URLに限定する。非公開、限定公開、期限付きURLは外部資料に載せない。

## 10. SNS・ブログ確認手順

| 確認項目 | 内容 |
|---|---|
| 投稿者 | 本人又は公式アカウントか確認 |
| 投稿日時 | 投稿日時、編集日を確認 |
| 投稿URL | 公開可能URLか確認 |
| 投稿本文 | 原文と要約を分ける |
| 添付画像 | 画像内文字、個人情報、著作権に注意 |
| 削除可能性 | スクリーンショット保存可否を検討 |
| 引用・リポスト | 本人発言と他者発言を分ける |

SNS投稿は削除・編集される可能性があるため`warning_id`を検討する。他者投稿の引用、リポスト、いいねを本人発言と混同しない。

スクリーンショットには個人情報やsecretが写っていないか確認する。公的職務に関係しない私的投稿を不用意に扱わない。

## 11. 議事録・会議録確認手順

| 確認項目 | 内容 |
|---|---|
| 会議名 | 本会議、委員会、協議会等 |
| 開催日 | 年月日 |
| 発言者 | 氏名、役職 |
| 発言区分 | 質問、答弁、討論、説明、質疑 |
| 該当ページ | ページ番号、段落 |
| 議案番号 | 関連する場合 |
| 関連政策 | `policy_id`と接続 |
| 議決結果 | 必要に応じて確認 |

## 12. 発言分類

| 分類 | 内容 |
|---|---|
| 事実主張 | 事実、数値、出来事についての発言 |
| 意見 | 評価、考え、見解 |
| 約束 | 公約、実施方針、期限付き発言 |
| 弁明 | 批判や指摘への説明 |
| 方針表明 | 政策方向、行政方針 |
| 質問 | 議員質問、確認質問 |
| 答弁 | 首長、職員、執行部回答 |
| 討論 | 賛成討論、反対討論 |
| 議決行動 | 賛否、退席、棄権 |
| 説明 | 行政説明、資料説明 |
| 投稿 | SNS、ブログ、note等 |
| 報道コメント | 報道を通じた発言 |

発言分類は評価ではなく整理である。一つの発言が複数分類に該当する場合は分けて記録する。事実主張には`source_id`確認を優先する。

## 13. policy_idとの接続

| 発言内容 | 接続するpolicy_id例 |
|---|---|
| 予算に関する発言 | 予算、決算、事業別予算 |
| 補助金に関する発言 | 補助金、交付金、負担金 |
| 委託契約に関する発言 | 委託事業、契約、仕様書 |
| 公共施設に関する発言 | 施設管理、指定管理、建設事業 |
| 福祉に関する発言 | 社協、地域福祉、介護、障害福祉 |
| 教育に関する発言 | 教育委員会、学校、給食、ICT |
| 防災に関する発言 | 防災計画、避難所、消防、BCP |
| 行政改革に関する発言 | 事務事業評価、外郭団体、補助金見直し |

`policy_id`は発言と政策・事業を接続する索引である。`policy_id`だけで政策効果、失敗、不正を断定しない。発言と実際の予算、議決、契約、成果が一致するかは別途`source_id`で確認する。

## 14. 矛盾・過去発言照合

| 照合対象 | 確認内容 |
|---|---|
| 過去発言 | 以前の`quote_id`と比較 |
| 議決行動 | 賛否、退席、棄権と発言の関係 |
| 予算行動 | 予算案、決算認定、修正案との関係 |
| 公約 | 選挙公約、広報紙、マニフェストとの関係 |
| 現在方針 | 最新発言、施政方針、事業計画との関係 |

矛盾候補は直ちに虚偽、欺瞞、不正と断定しない。年度、制度変更、立場変更、情勢変化を確認する。

矛盾候補は`warning_id`又は`gap_id`として扱う。外部発信する場合は`USER_DECISION`を必要とする。

## 15. gap_id / warning_id / stop_id 判定

| 状態 | 発行ID | 対応 |
|---|---|---|
| 発言原文が未確認 | `GAP_` 又は `STOP_` | `quote_id`正式化停止 |
| 発言者が未確認 | `GAP_` 又は `STOP_` | `actor_id`確認 |
| 動画の聞き取り不明 | `GAP_` | 再確認又は保留 |
| SNS投稿が削除済み | `WARN_` 又は `GAP_` | 取得状態を記録 |
| 前後文脈が不明 | `WARN_` | 文脈注意を明記 |
| 要約と原文が混在 | `WARN_` | 欄を分ける |
| `source_id`なし | `STOP_` | `quote_id`正式化停止 |
| 法的断定が必要 | `STOP_` | 法務官確認 |
| 外部発信予定 | `DEC_` | `USER_DECISION` |

## 16. 外部発信前チェック

公開前チェックリストを使用し、以下を確認する。

- `source_id`があるか確認する。
- `quote_id`が`source_id`と`actor_id`に接続しているか確認する。
- 原文、要約、評価を分けているか確認する。
- `gap_id`を隠していないか確認する。
- `warning_id`を反映しているか確認する。
- `stop_id`がある場合は公開しない。
- 違法、不正、癒着、汚職、虚偽を根拠なく断定しない。
- 個人攻撃になっていないか確認する。
- 公的職務に関係しない私的情報を含めない。
- 外部発信には必要に応じて`decision_id`を取得する。

## 18. 禁止事項

- `source_id`なしに`quote_id`を正式化しない。
- 発言原文未確認のまま言質化しない。
- 要約を原文として扱わない。
- 動画の自動字幕だけで原文確定しない。
- 発言者未確認のまま`actor_id`を確定しない。
- 同姓同名を確認せず同一人物扱いしない。
- SNSの引用、リポスト、いいねを本人発言と混同しない。
- 文脈を無視して切り取らない。
- 矛盾候補を直ちに虚偽、欺瞞、不正と断定しない。
- `actor_id`だけで個人責任、故意、違法、不正を断定しない。
- `policy_id`だけで政策失敗、不正、癒着を断定しない。
- 公的職務と無関係な私的情報を扱わない。
- 個人情報、黒塗り前資料、APIキー、secretをGitHubに追加しない。
- `stop_id`がある状態で外部発信しない。
- `decision_id`なしに外部発信、法務官引渡、評価断定へ進まない。

## 19. PR条件

- 新規ファイル追加のみ。
- 既存ファイルを変更しない。
- 既存ファイルを削除しない。
- `archive`配下を変更しない。
- PDF原本を追加しない。
- 個人情報を追加しない。
- 黒塗り前資料を追加しない。
- APIキー、パスワード、トークンを追加しない。
- `main`へ直接コミットしない。
- Pull Requestは作成するが、自動マージはしない。
