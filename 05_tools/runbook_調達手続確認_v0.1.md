# Runbook: 調達手続確認 v0.1

## 1. 位置づけ

このRunbookは、後方支援軍が国、都道府県、市町村、外郭団体、第三セクター等の調達手続を確認するときの標準手順である。
確認対象は、契約に至る手続、公開資料、根拠資料、結果資料の整合確認に限定する。

このRunbookでは、次を分離して記録する。

- 事実: 公開資料または取得資料で確認できる事項
- 推定: 事実から考えられるが、未確定の事項
- 評価: 手続上の競争性、透明性、説明責任に関する整理
- 未確認事項: 追加資料がないと判断できない事項
- 情報公開請求候補: 確認のために取得候補とする資料

1者応札、1者見積、随意契約、落札率の高さ、仕様条件の限定性だけで、不正、癒着、違法、利益供与、談合と断定してはならない。

## 2. 対象範囲

対象に含める調達手続は次のとおりである。

- 一般競争入札
- 指名競争入札
- 随意契約
- プロポーザル
- 企画提案
- 見積合わせ
- 1者応札
- 1者見積
- 入札不調
- 辞退
- 再公告
- 契約変更

対象に含める確認項目は次のとおりである。

- 契約名
- 発注者
- 契約相手方
- 契約方法
- 公告日
- 入札日または選定日
- 契約日
- 契約金額
- 予定価格
- 最低制限価格
- 落札率
- 参加者数
- 辞退者数
- 1者応札または1者見積の有無
- 随意契約理由
- 参加資格
- 仕様条件
- 質問回答
- 評価基準
- 評価結果
- 契約変更の有無
- 手続の公開状況

対象外とするものは次のとおりである。

- 補助金の政策目的、成果、妥当性の中身評価
- 委託契約の事業内容、成果物、実績の中身評価
- 指定管理の運営評価、施設評価、利用者評価の中身評価
- 契約相手方または関係者の属性だけに基づく評価
- 非公開資料、個人情報、secret、APIキー、PDF原本、画像の保管

補助金、委託契約、指定管理の中身評価が必要な場合は、別Runbookまたは別case_idへ分離する。

## 3. 調達手続の基本区分

| 区分 | 主な確認対象 | 確認の要点 |
| --- | --- | --- |
| 一般競争入札 | 公告、仕様、参加資格、入札結果 | 広く参加機会が示され、参加資格と仕様条件が説明可能か確認する。 |
| 指名競争入札 | 指名業者一覧、指名理由、入札結果 | 指名範囲、辞退状況、競争性が説明可能か確認する。 |
| 随意契約 | 随意契約理由書、見積書、契約結果 | 随意契約理由、相手方選定理由、価格確認の手続を確認する。 |
| プロポーザル・企画提案 | 実施要領、評価基準、審査結果 | 価格点と技術点を分け、審査基準と評価結果の対応を確認する。 |
| 見積合わせ | 見積依頼、見積書、選定結果 | 見積先、見積数、辞退、価格比較の記録を確認する。 |
| 1者応札・1者見積 | 公告、参加状況、辞退状況、理由資料 | 1者である事実と、その背景確認を分けて整理する。 |
| 不調・辞退・再公告 | 不調理由、辞退届、再公告資料 | 不調の理由、条件変更、再公告時の変更点を確認する。 |
| 契約変更 | 変更契約資料、変更理由、金額変更 | 変更理由、変更範囲、増減額、当初契約との関係を確認する。 |

## 4. 確認する資料

確認資料は、資料名、入手元、取得日、source_id、公開URLまたは保管場所、確認できた事項を記録する。

最低限、次の資料を確認候補に含める。

| 資料 | 主な確認事項 |
| --- | --- |
| 入札公告 | 契約名、発注者、公告日、参加資格、入札日、手続の公開状況 |
| 入札説明書 | 手続、提出書類、スケジュール、評価または落札決定方法 |
| 仕様書 | 業務内容、仕様条件、履行場所、履行期間、限定条件 |
| 設計書 | 積算前提、数量、内訳、仕様との対応 |
| 予定価格資料 | 予定価格、設定根拠、税抜税込の別、公開範囲 |
| 最低制限価格資料 | 最低制限価格、算定方式、適用対象、公開範囲 |
| 参加資格要件 | 参加資格、地域要件、実績要件、許認可要件 |
| 質問回答書 | 仕様条件の補足、参加者からの疑義、発注者回答 |
| 入札結果 | 入札日、参加者数、辞退者数、入札額、落札者、落札率 |
| 契約結果一覧 | 契約日、契約相手方、契約金額、契約方法 |
| 辞退届または辞退状況 | 辞退者数、辞退時点、辞退理由の有無 |
| 不調理由資料 | 不調の理由、入札額、予定価格との関係、再実施方針 |
| 再公告資料 | 再公告日、条件変更、仕様変更、スケジュール変更 |
| 指名業者一覧 | 指名先、指名数、指名理由、辞退状況 |
| 見積書 | 見積者、見積額、見積日、見積条件 |
| 随意契約理由書 | 随意契約理由、相手方選定理由、根拠規定 |
| 契約書 | 契約名、契約日、契約金額、履行期間、相手方 |
| 変更契約資料 | 変更日、変更理由、変更金額、変更内容 |
| プロポーザル実施要領 | 参加資格、提出書類、審査方法、選定日 |
| 評価基準 | 評価項目、配点、価格点、技術点、失格基準 |
| 審査結果 | 評価結果、順位、選定者、参加者数 |
| 選定理由 | 優先交渉権者または契約候補者の選定理由 |
| 審査委員会資料 | 委員構成、審査経過、議事概要、利益相反確認の有無 |
| 議会資料 | 契約議案、説明資料、質疑、承認状況 |
| 予算書 | 予算科目、予算額、事業名、財源 |
| 決算書 | 決算額、支出済額、予算との差異 |
| 主要施策成果報告書 | 事業実施状況、支出概要、成果説明 |

## 5. 一般競争入札の確認観点

一般競争入札では、参加機会の公開性と落札決定までの手続を確認する。

- 入札公告に、契約名、発注者、公告日、入札日、契約方法が明記されているか。
- 参加資格が、業務内容、法令要件、安全性、専門性、履行能力と対応しているか。
- 仕様条件が、特定事業者だけを過度に有利にする内容として断定できるかではなく、説明可能な業務上の理由があるかを確認する。
- 質問回答により、仕様条件や参加資格が補足または修正されていないか。
- 入札結果に、参加者数、辞退者数、入札額、落札者、落札率が記録されているか。
- 予定価格、最低制限価格、契約金額、支払額、決算額を混同していないか。
- 手続の公開状況として、公告、仕様書、質問回答、入札結果の公開有無を整理する。

## 6. 指名競争入札の確認観点

指名競争入札では、指名の範囲、辞退状況、競争性を確認する。

- 指名業者一覧が確認できるか。
- 指名数、参加者数、辞退者数を分けて記録しているか。
- 指名理由が、登録業種、履行実績、地域性、専門性、緊急性等と対応しているか。
- 辞退届または辞退状況が確認できるか。
- 指名後に仕様条件、予定価格、入札日等が変更されていないか。
- 1者入札となった場合、指名数、辞退理由、再入札または再公告の検討有無を確認する。
- 指名競争入札である事実だけで、競争性欠如や不正を断定しない。

## 7. 随意契約の確認観点

随意契約では、契約方法の根拠、相手方選定理由、価格確認の手続を確認する。

- 随意契約理由書が確認できるか。
- 根拠規定、契約方法、契約相手方、契約日、契約金額が確認できるか。
- 随意契約理由が、専門性、継続性、緊急性、互換性、権利関係、少額随契等のどれに該当するか整理する。
- 1者見積の場合、1者である理由、他者見積を求めなかった理由、価格妥当性確認の有無を確認する。
- 複数見積の場合、見積合わせの記録、見積書、辞退状況を確認する。
- 随意契約である事実だけで、不正、癒着、違法、利益供与と断定しない。

## 8. プロポーザル・企画提案の確認観点

プロポーザル・企画提案では、価格競争ではなく総合評価である場合があるため、価格点と技術点を分けて確認する。

- プロポーザル実施要領または企画提案募集要領が確認できるか。
- 参加資格、提出書類、審査日または選定日、契約候補者の決定方法を確認する。
- 評価基準に、評価項目、配点、価格点、技術点、失格基準が明記されているか。
- 審査結果に、参加者数、評価結果、順位、選定理由が記録されているか。
- 審査委員会資料に、委員構成、審査経過、利益相反確認の有無が記録されているか。
- 価格が最低ではない者が選定された場合でも、評価基準に基づく総合評価であれば、その対応関係を確認する。
- 評価基準や審査結果が未確認のまま、妥当性、不正、恣意性を断定しない。

## 9. 見積合わせの確認観点

見積合わせでは、見積先、見積条件、見積結果、選定理由を確認する。

- 見積依頼先または見積取得先が確認できるか。
- 見積書に、見積者、見積額、見積日、見積条件が記載されているか。
- 仕様条件、数量、履行期間等が見積者間で同一条件になっているか。
- 辞退がある場合、辞退者数、辞退時点、辞退理由の有無を整理する。
- 最低価格者以外を選定した場合、選定理由、仕様適合性、納期、資格要件を確認する。
- 見積合わせの結果だけでなく、契約結果一覧または契約書と照合する。

## 10. 1者応札・1者見積の確認観点

1者応札・1者見積では、1者である事実、背景、発注者の対応を分けて整理する。

- 1者応札または1者見積の有無を明記する。
- 公告日から入札日または見積期限までの期間が確認できるか。
- 参加資格、仕様条件、履行期間、地域要件、実績要件が過度に限定的かを確認する。
- ただし、施設特性、専門性、安全性、継続性、法令要件による合理的理由がないかを必ず確認する。
- 辞退者数、辞退理由、質問回答の内容を確認する。
- 再公告、仕様変更、参加資格緩和、見積先追加等の検討有無を確認する。
- 1者である事実だけで、不正、癒着、違法、利益供与と断定しない。

## 11. 不調・辞退・再公告の確認観点

不調、辞退、再公告では、発生理由と条件変更の有無を確認する。

- 入札不調の理由資料が確認できるか。
- 不調が、参加者なし、全者辞退、予定価格超過、最低制限価格未満、資格不備等のどれに該当するか整理する。
- 辞退届または辞退状況により、辞退者数、辞退時点、辞退理由の有無を確認する。
- 再公告資料により、公告日、入札日、仕様条件、参加資格、予定価格、履行期間の変更点を確認する。
- 再公告後の参加者数、落札率、契約相手方を当初手続と比較する。
- 不調や辞退がある事実だけで、発注条件の不当性や談合を断定しない。

## 12. 予定価格・最低制限価格・落札率の確認観点

予定価格、最低制限価格、落札率は、価格情報の種類を区分して確認する。

- 予定価格、最低制限価格、入札額、契約額、支払額、決算額を別項目として記録する。
- 税込、税抜、単年度、複数年度、総額、単価契約の別を確認する。
- 落札率を計算する場合は、原則として落札額を予定価格で除した割合として扱い、計算式を記録する。
- 最低制限価格がある場合、入札額が最低制限価格未満で失格となっていないか確認する。
- 予定価格資料または入札結果で予定価格が公開されていない場合、未確認事項または情報公開請求候補に分離する。
- 落札率が高いだけで、談合、不正、癒着、違法と断定しない。

## 13. 参加資格・仕様条件の確認観点

参加資格・仕様条件では、限定性と合理性を分けて確認する。

- 参加資格に、地域要件、登録業種、実績要件、資格者配置、許認可、財務要件が含まれるか確認する。
- 仕様条件に、機器指定、既存設備との互換性、保守継続性、専門資格、安全基準、法令要件が含まれるか確認する。
- 限定的に見える条件がある場合、施設特性、専門性、安全性、継続性、法令要件の根拠を確認する。
- 質問回答により、参加資格や仕様条件が緩和、補足、修正されていないか確認する。
- 仕様条件の限定性を評価する場合は、事実、推定、評価、未確認事項を分けて記録する。

## 14. 契約変更の確認観点

契約変更では、当初契約と変更契約の差分、変更理由、金額影響を確認する。

- 契約変更の有無を確認する。
- 変更契約資料に、変更日、変更理由、変更内容、変更金額、履行期間変更が記載されているか。
- 当初契約書、仕様書、設計書、変更契約資料を照合する。
- 増額、減額、工期延長、仕様変更、数量変更のいずれかを区分する。
- 変更理由が、現場条件、法令対応、物価変動、数量精算、発注者都合、受注者都合等のどれに該当するか整理する。
- 変更後の契約金額と支払額、決算額を混同しない。

## 15. 競争性・透明性・説明責任の整理

確認結果は、競争性、透明性、説明責任の3軸で整理する。

| 軸 | 確認する内容 | 記録方法 |
| --- | --- | --- |
| 競争性 | 参加者数、辞退者数、1者応札、1者見積、参加資格、仕様条件 | 事実と背景確認を分離して記録する。 |
| 透明性 | 公告、入札説明書、仕様書、質問回答、入札結果、評価結果の公開状況 | 公開済み、未公開、未確認を区分する。 |
| 説明責任 | 随意契約理由、選定理由、評価基準、契約変更理由 | 根拠資料の有無と説明の対応を記録する。 |

評価を書く場合は、断定表現を避け、資料で確認できる範囲を明示する。

## 16. source_idとの接続

各資料にはsource_idを付与し、確認事項と対応させる。

- 1資料につき1つのsource_idを原則とする。
- 同一資料の版違い、再公告版、変更契約版は別source_idとする。
- source_idには、資料名、発行者、発行日または取得日、URLまたは保管場所を紐づける。
- PDF原本、画像、個人情報、secret、APIキーはGitHubへ追加しない。
- 黒塗り前資料または非公開資料の混入疑いがある場合はstop_idを発行する。

## 17. case_idとの接続

調達手続の確認単位にはcase_idを付与する。

- 原則として、1契約または1調達手続につき1つのcase_idを付与する。
- 再公告は、当初公告と同一case_id内で枝番管理するか、必要に応じて別case_idに分離する。
- 契約変更は、当初契約のcase_idに紐づけ、変更回数と変更日を記録する。
- 補助金、委託契約、指定管理の中身評価は、調達手続確認のcase_idと混在させない。

## 18. gap_id / warning_id / stop_id の発行条件

確認の不足、注意、作業停止条件は、gap_id、warning_id、stop_idで分離する。

### gap_id

次の場合はgap_idを発行する。

- 入札公告、仕様書、入札結果、契約書のいずれかが未取得である。
- 予定価格または最低制限価格の公開有無が未確認である。
- 参加者数、辞退者数、1者応札または1者見積の有無が未確認である。
- 随意契約理由書、評価基準、審査結果、変更契約資料の該当有無が未確認である。
- 質問回答書、再公告資料、不調理由資料の有無が未確認である。

### warning_id

次の場合はwarning_idを発行する。

- 1者応札、1者見積、随意契約、落札率の高さなど、誤って断定につながりやすい事実がある。
- 参加資格または仕様条件が限定的に見えるが、合理的理由の確認が未完了である。
- プロポーザルで価格点と技術点の内訳が未確認である。
- 再公告時に条件変更があるが、変更理由が未確認である。
- 契約変更に増額があるが、変更理由、変更範囲、支払額との関係が未確認である。

### stop_id

次の場合はstop_idを発行し、公開前またはPR前の作業を停止する。

- 入札公告、仕様書、入札結果、随意契約理由、評価結果が未確認のまま不正・癒着・違法と断定しようとしている。
- 予定価格、契約額、支払額、決算額の区分が未確認のまま重要評価を行う。
- 1者応札、1者見積、随意契約だけで重要断定を行おうとしている。
- 評価基準や審査結果が未確認のままプロポーザルの妥当性を断定しようとしている。
- 個人情報、黒塗り前資料、非公開資料、secretの混入疑いがある。
- PDF原本や画像をGitHubへ追加しようとしている。
- 要約資料だけで重要断定を行おうとしている。
- 旧本文と新本文が混在している。
- 変更対象外ファイルが差分に入っている。
- README.md、archive、04_templatesが目的外に変更されている。

## 19. 情報公開請求候補への接続

未確認事項は、情報公開請求候補として整理する。

請求候補にする資料は、次のように記録する。

| 項目 | 記録内容 |
| --- | --- |
| 請求候補資料名 | 例: 予定価格調書、随意契約理由書、審査委員会資料 |
| 確認したい事項 | 例: 予定価格、選定理由、評価結果、契約変更理由 |
| 関連source_id | 既に確認した資料のsource_id |
| 関連case_id | 対象の調達手続のcase_id |
| 請求理由 | 手続確認に必要な理由を簡潔に記載する。 |
| 注意事項 | 個人情報、非公開情報、黒塗り必要箇所の可能性を記載する。 |

情報公開請求候補は、断定の補強ではなく、未確認事項を確認するために作成する。

## 20. 公開前チェックへの引き渡し

公開前チェックへ引き渡す前に、次を確認する。

- 事実、推定、評価、未確認事項、情報公開請求候補が分離されている。
- 1者応札、1者見積、随意契約、落札率の高さだけで重要断定をしていない。
- 予定価格、最低制限価格、契約額、支払額、決算額を混同していない。
- プロポーザルの価格点と技術点を分けて確認している。
- 参加資格や仕様条件について、施設特性、専門性、安全性、継続性、法令要件を確認している。
- source_idとcase_idの対応が記録されている。
- gap_id、warning_id、stop_idの発行状況が整理されている。
- PDF原本、画像、個人情報、secret、APIキーが含まれていない。

## 21. 禁止事項

このRunbookで禁止する事項は次のとおりである。

- 1者応札、1者見積、随意契約だけで、不正、癒着、違法、利益供与と断定すること。
- 落札率が高いだけで、談合と断定すること。
- 予定価格、最低制限価格、契約額、支払額、決算額を混同すること。
- 評価基準や審査結果が未確認のまま、プロポーザルの妥当性を断定すること。
- 仕様条件が限定的に見えるだけで、合理的理由の確認を省略すること。
- 要約資料だけで重要断定を行うこと。
- 補助金、委託契約、指定管理の中身評価をこのRunbookのcase_idに混入させること。
- PDF原本、画像、個人情報、secret、APIキーをGitHubへ追加すること。
- 既存Runbook本文をコピーして混入させること。
- 過去の壊れ差分を再利用すること。
- 旧本文と新本文を併記すること。

- README.mdを変更していない。
- archiveを変更していない。
- 04_templatesを変更していない。
- 他の05_toolsファイルを変更していない。
- PDFや画像を追加していない。
- secretや個人情報を追加していない。
- 章番号の重複がない。
- Markdown表が壊れていない。
- mainへ直接コミットしていない。
- 旧本文と新本文が混在していない。
