# Runbook: 補助金支出確認 v0.1

## 1. 位置づけ

このRunbookは、後方支援軍が補助金支出を確認するときの標準手順を定める。国、都道府県、市町村、外郭団体、社会福祉協議会、観光協会、商工会、商工会議所、任意団体、実行委員会等に関する補助金支出について、交付根拠、交付目的、財源内訳、交付額、補助率、対象経費、実績報告、精算、成果、返還条件、団体自己財源、行政との関係性を、公開資料と入手資料に基づき分離して確認する。

補助金交付の存在だけで、不正、癒着、違法、利益供与と断定しない。事実、推定、評価、未確認事項、情報公開請求候補を明確に分ける。

## 2. 対象範囲

対象に含める支出は次のとおりとする。

- 国から自治体への国庫補助金
- 国から自治体への国庫負担金
- 国から自治体への国庫交付金
- 国から団体への直接補助
- 国から都道府県を経由して市町村・団体へ流れる補助
- 都道府県から市町村・団体への補助
- 市町村から団体への補助
- 国費・県費・市町村費・団体自己財源が混在する事業
- 間接補助
- 再交付
- 協議会・実行委員会経由の支出
- 社会福祉協議会、観光協会、商工会、商工会議所、任意団体への補助金

対象外は、委託契約、指定管理、随意契約、入札の詳細確認とする。これらが同一事業に併存する場合でも、このRunbookでは補助金支出の確認に限定し、契約手続の適否判断は別Runbookまたは別案件に分離する。

## 3. 補助金支出の基本区分

補助金支出を確認するときは、最初に支出の性質を区分する。

| 区分 | 主な確認内容 | 注意点 |
| --- | --- | --- |
| 国庫補助金 | 所管省庁、制度名、補助要綱、補助率、交付決定、実績報告、精算 | 国費の交付先と最終支出先を分ける |
| 国庫負担金 | 法令上の負担根拠、負担割合、自治体負担、実績報告 | 裁量的補助金と混同しない |
| 国庫交付金 | 交付金制度名、交付要綱、交付対象事業、自治体での配分 | 地方交付税と混同しない |
| 都道府県補助金 | 県要綱、県費負担、補助対象、市町村・団体への流れ | 国費が含まれるか確認する |
| 市町村補助金 | 市町村要綱、予算科目、交付先、対象経費、成果 | 一般財源充当の有無を区分する |
| 間接補助・再交付 | 一次交付先、再交付先、再交付要綱、精算責任 | どの主体が審査・精算したか確認する |
| 協議会・実行委員会経由 | 構成団体、事務局、収支、補助金の流れ | 行政と団体の役割を分ける |

地方交付税、地方債、一般財源は、特定団体への補助金と同列に扱わない。地方交付税や一般財源は財源区分として整理し、交付先団体への補助金そのものとは分けて記録する。

## 4. 国費・県費・市町村費・自己財源の区分

財源は、少なくとも次の区分で整理する。

| 財源区分 | 確認する資料 | 記録する事項 |
| --- | --- | --- |
| 国費 | 所管省庁資料、補助要綱、交付要綱、交付要領、公募要領、採択結果、交付決定 | 所管省庁、補助制度名、国費額、補助率、国費の受入科目、間接補助・再交付の有無 |
| 県費 | 都道府県資料、県予算書、県決算書、県補助金一覧 | 県費額、県補助率、県の交付先、国費を含むか |
| 市町村費 | 自治体予算書、自治体決算書、主要施策成果報告書、補助金一覧 | 市町村費額、一般財源充当額、歳入歳出科目、交付先 |
| 団体自己財源 | 団体の収支予算書、収支決算書、事業報告書、決算書または計算書類 | 会費、寄附金、事業収入、繰越金、自己負担額、公費依存度 |

国費を扱う場合は、所管省庁、補助制度名、補助要綱、交付要綱、交付要領、公募要領、採択結果、交付決定、補助率、国費・県費・市町村費・自己財源の財源内訳、間接補助の有無、再交付の有無、自治体予算・決算での受入科目、団体側の収入科目、実績報告、精算、返還条件、成果指標を確認する。

## 5. 確認する資料

確認資料は、入手済み、未入手、情報公開請求候補に分けて管理する。最低限、次の資料を確認対象に含める。

- 補助金交付要綱
- 補助要綱
- 交付要領
- 公募要領
- 採択結果
- 補助金交付申請書
- 事業計画書
- 収支予算書
- 交付決定通知
- 変更承認資料
- 実績報告書
- 収支決算書
- 精算書
- 返還命令または返還記録
- 所管省庁資料
- 都道府県資料
- 自治体予算書
- 自治体決算書
- 主要施策成果報告書
- 補助金一覧
- 団体の事業報告書
- 団体の決算書または計算書類

資料ごとに、資料名、作成主体、作成年月日、公開URLまたは入手経路、対象年度、対応するsource_idを記録する。

## 6. 交付要綱・補助要綱・交付要領の確認観点

交付根拠は次の順で確認する。

1. 制度の根拠となる法令、条例、規則、要綱、要領を特定する。
2. 交付主体、交付対象、対象事業、対象経費、補助率、上限額、申請手続、変更承認、実績報告、精算、返還条件を抜き出す。
3. 年度ごとに要綱改正がないか確認する。
4. 国、県、市町村の要綱が重なる場合は、上位制度と自治体内手続を分けて記録する。
5. 要綱が未確認の場合は、重要評価を保留し、情報公開請求候補に回す。

## 7. 交付目的の確認観点

交付目的は、要綱、交付申請書、事業計画書、予算説明資料、主要施策成果報告書を照合して確認する。

- 制度目的と個別事業目的が一致しているか。
- 交付先団体の活動目的と補助目的が整合しているか。
- 目的が抽象的な場合、具体的な実施事業、対象者、地域、成果物を確認する。
- 目的外使用の疑いは、対象経費、実績報告、精算資料で裏取りするまで断定しない。

## 8. 交付対象団体の確認観点

交付対象団体について、次を確認する。

- 団体名、法人格、代表者、所在地、設立目的、所管部署。
- 社会福祉協議会、観光協会、商工会、商工会議所、任意団体、協議会、実行委員会などの種別。
- 要綱上の交付対象に該当する根拠。
- 構成団体、会員、事務局、会計責任者、監査体制。
- 行政職員や首長、議員、外郭団体職員との役職兼務・関係性。ただし公開資料に基づく範囲で整理し、個人攻撃にしない。

## 9. 交付額・補助率・対象経費の確認観点

交付額、補助率、対象経費は、申請、交付決定、変更承認、実績報告、精算の各段階で比較する。

| 項目 | 確認観点 |
| --- | --- |
| 交付額 | 当初申請額、交付決定額、変更後額、確定額、返還額の差異 |
| 補助率 | 要綱上の補助率、実際の補助率、上限額、定額補助か割合補助か |
| 対象経費 | 人件費、謝金、旅費、委託費、備品費、印刷費、会場費、広報費などの対象可否 |
| 対象外経費 | 飲食、交際、政治活動、宗教活動、目的外支出などの除外規定 |
| 自己負担 | 団体自己財源、受益者負担、寄附、会費、事業収入の充当状況 |

補助金の確認観点には、交付要綱、交付目的、交付対象団体、交付額、補助率、対象経費、交付決定、実績報告、精算、返還条件、成果指標、団体の自己財源、行政との役職兼務・関係性を必ず含める。

## 10. 財源内訳の確認観点

財源内訳は、自治体側と団体側の両方で確認する。

- 自治体歳入で国費・県費を受け入れているか。
- 自治体歳出でどの款項目節から補助金を支出しているか。
- 団体側の収入科目に国費、県費、市町村費、補助金、負担金、会費、寄附金、事業収入がどのように計上されているか。
- 国費、県費、市町村費、団体自己財源の金額が一致するか。
- 間接補助や再交付がある場合、各段階の交付額、控除額、事務費、返還額を追跡できるか。

内訳が未確認のまま、財源評価を断定しない。

## 11. 交付決定・実績報告・精算の確認観点

補助金は、少なくとも次の流れで確認する。

1. 交付申請書と事業計画書を確認する。
2. 交付決定通知で交付額、条件、対象期間、返還条件を確認する。
3. 変更承認資料で事業内容、交付額、対象経費の変更を確認する。
4. 実績報告書で実施内容、成果、支出額を確認する。
5. 収支決算書と精算書で確定額、余剰額、返還額を確認する。
6. 返還命令または返還記録がある場合は、理由、金額、日付を確認する。

交付決定、実績報告、精算資料が未確認の場合は、重要評価を保留する。

## 12. 成果指標・成果物の確認観点

成果は、制度目的、事業計画、実績報告、主要施策成果報告書、団体事業報告書を照合して確認する。

- 成果指標が設定されているか。
- 成果指標の実績値が報告されているか。
- 成果物、開催記録、参加者数、配布物、報告書、ウェブ掲載、アンケート結果などが確認できるか。
- 成果が未確認の場合は「成果物未確認」と記録し、「成果なし」と断定しない。
- 定量指標がない場合は、定性的成果と未確認事項を分ける。

## 13. 団体の自己財源・公費依存度の確認観点

団体の自己財源は、団体の決算書または計算書類、収支決算書、事業報告書で確認する。

- 会費、寄附金、事業収入、繰越金、積立金、受益者負担の有無。
- 補助金収入の総収入に占める割合。
- 特定自治体または特定補助制度への依存度。
- 自己財源が補助対象事業に充当されているか。
- 公費依存度が高い場合でも、それだけで直ちに不正と断定しない。

## 14. 行政との役職兼務・関係性の確認観点

行政との関係性は、公開資料で確認できる範囲に限定する。

- 団体役員に行政職員、首長、議員、外郭団体関係者が含まれるか。
- 事務局が自治体庁舎内または所管課内に置かれているか。
- 自治体が団体運営にどの程度関与しているか。
- 補助金審査、交付決定、実績確認、精算確認の担当部署と団体側役職が重なっていないか。
- 関係性は「公開資料で確認できる事実」と「評価」を分けて記録する。

## 15. 予算・決算・団体資料との照合

自治体資料と団体資料を次の観点で照合する。

| 照合対象 | 確認する差異 |
| --- | --- |
| 自治体予算書と交付決定 | 予算額、補助制度名、所管課、歳出科目 |
| 自治体決算書と精算書 | 支出済額、確定額、不用額、返還額 |
| 主要施策成果報告書と実績報告書 | 実施内容、成果指標、参加者数、成果物 |
| 補助金一覧と団体決算書 | 自治体支出額と団体収入額の一致 |
| 団体収支決算書と事業報告書 | 事業内容、対象経費、自己財源の整合 |

差異がある場合は、単純な年度ずれ、税込税抜、千円単位表示、複数事業合算、返還処理の有無を確認する。

## 16. source_idとの接続

すべての根拠資料にsource_idを付与する。

- 1資料につき1つ以上のsource_idを付ける。
- 同一PDFや同一ページ内で複数箇所を使う場合は、ページ、章、表番号、行、項目名を記録する。
- 要約資料だけでなく、可能な限り一次資料をsource_idに接続する。
- source_idには、資料名、発行主体、公開URLまたは入手経路、確認日、対象年度を含める。

## 17. case_idとの接続

補助金支出確認は、対象事業または対象団体ごとにcase_idへ接続する。

- 同一団体に複数年度・複数制度の補助がある場合は、年度別または制度別に枝番を付ける。
- 国費、県費、市町村費が同一事業に混在する場合は、1つのcase_id内で財源別に整理する。
- 委託契約、指定管理、入札等の別論点が出た場合は、補助金case_idとは分離する。
- case_idには、対象年度、交付主体、交付先、補助制度名、確認段階を記録する。

## 18. gap_id / warning_id / stop_id の発行条件

確認結果に応じて、gap_id、warning_id、stop_idを発行する。

| ID | 発行条件 | 対応 |
| --- | --- | --- |
| gap_id | 要綱、交付決定、実績報告、精算、財源内訳、団体資料の一部が未入手 | 未確認事項として記録し、情報公開請求候補へ接続する |
| warning_id | 金額差異、目的と支出内容の不整合、成果指標未設定、自己財源不明、関係性の説明不足がある | 追加照合を行い、断定表現を避ける |
| stop_id | 重大な未確認、非公開情報混入、対象外ファイル変更、根拠なし断定の危険がある | 作業または公開を停止し、差分と根拠資料を点検する |

stop_idは、少なくとも次の場合に発行する。

- 根拠資料が未確認のまま不正・癒着・違法と断定しようとしている。
- 交付要綱、交付決定、実績報告、精算資料が未確認のまま重要評価を行う。
- 国費、県費、市町村費、自己財源の内訳が未確認のまま財源評価を断定する。
- 個人情報、黒塗り前資料、非公開資料、secretの混入疑いがある。
- PDF原本や画像をGitHubへ追加しようとしている。
- 要約資料だけで重要断定を行おうとしている。
- 旧本文と新本文が混在している。
- 変更対象外ファイルが差分に入っている。
- README.md、archive、04_templatesが目的外に変更されている。

## 19. 情報公開請求候補への接続

次の資料が未入手の場合は、情報公開請求候補に接続する。

- 補助金交付申請書
- 事業計画書
- 収支予算書
- 交付決定通知
- 変更承認資料
- 実績報告書
- 収支決算書
- 精算書
- 返還命令または返還記録
- 審査資料、採択理由、採択結果の詳細
- 国費、県費、市町村費、自己財源の内訳が分かる資料

請求候補では、請求先、対象年度、補助制度名、交付先団体名、必要資料、公開済み資料との差分を記録する。

## 20. 公開前チェックへの引き渡し

公開前チェックへ引き渡す前に、次を確認する。

- 事実、推定、評価、未確認事項、情報公開請求候補が分離されている。
- 不正、癒着、違法、利益供与などの断定表現に根拠資料が接続されている。
- 成果物未確認を成果なしと書いていない。
- 公費依存度が高いだけで不正と書いていない。
- 行政との関係性が公開資料に基づく範囲に限定されている。
- 個人情報、黒塗り前資料、非公開資料、secret、APIキーが混入していない。
- PDF原本、画像、添付ファイルをGitHubへ追加していない。

## 21. 禁止事項

このRunbookで禁止する事項は次のとおりとする。

- 補助金交付だけで不正、癒着、違法、利益供与と断定すること。
- 成果物未確認のまま成果なしと断定すること。
- 公費依存度が高いだけで直ちに不正と断定すること。
- 国庫補助金、国庫負担金、国庫交付金、地方交付税、地方債、一般財源を混同すること。
- 地方交付税や一般財源を、特定団体への補助金と同列に扱うこと。
- 行政との関係性を個人攻撃として記述すること。
- 事実、推定、評価、未確認事項、情報公開請求候補を混在させること。
- 委託契約、指定管理、随意契約、入札の詳細確認をこのRunbook内で扱うこと。
- PDF原本、画像、個人情報、secret、APIキーを追加すること。
- 既存Runbook本文をコピーして混入させること。
- 旧本文と新本文を併記すること。

## 22. PR前確認条件

PR前に次を確認する。

- 変更ファイルは `05_tools/runbook_補助金支出確認_v0.1.md` の1個のみである。
- 新規追加の差分であり、既存本文の削除がない。
- `README.md` を変更していない。
- `archive` を変更していない。
- `04_templates` を変更していない。
- 他の `05_tools` ファイルを変更していない。
- PDFや画像を追加していない。
- secretや個人情報を追加していない。
- 章番号の重複がない。
- Markdown表が壊れていない。
- mainへ直接コミットしていない。
