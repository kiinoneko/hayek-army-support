# 05_tools — 作業支援ツール

## 1. そのフォルダの目的

資料の収集、抽出、整形、台帳生成、検証を再現可能かつ安全に補助し、手作業による転記ミスを減らす。

## 2. 保存する資料の種類

- ダウンロード、OCR、表抽出、形式変換、集計、リンク検査用のスクリプト
- 設定例、依存関係、使用手順、入出力仕様、テスト
- 台帳生成器、重複検査、チェックサム、出典・引用整合性検査
- 実行ログの書式やサンプル（秘密情報・個人情報を含まないもの）

## 3. Codexが行ってよい作業

- ツールの作成、修正、文書化、テスト、静的検査
- 読み取り専用または新規出力を基本とする収集・変換処理の実装
- 補助金、助成、給付、減免、委託、指定管理、外郭団体支援を納税者負担として扱う台帳の生成
- 事実、引用、出典、推定、評価、未確認事項、情報公開請求候補の分離と参照整合性の検査
- 実行前後の差分、エラー、不確実な抽出結果を人間へ提示すること

## 4. Codexが行ってはいけない作業

- 承認なく原資料を上書き・削除する処理、外部へ公開・送信する処理を実行すること
- 認証情報、個人情報、機密情報をコード、設定例、ログへ保存すること
- 抽出結果を無検証で事実認定し、違法性、妥当性、政治的評価、制度設計を最終判断すること
- robots.txt、利用規約、アクセス制限、法令上の制約を回避する実装を行うこと

## 5. ハイエク軍への接続方法

ツールの入力を原則`01_sources`、出力を`02_processed`または`00_index`とし、`03_reports`には直接最終判断を書き込まない。各実行で案件ID、入力、出力、実行日時、バージョン、主要オプションを記録し、ハイエク軍が結果を再検証できるようにする。

## 6. 運用上の注意

- 初回は少量データまたはドライランで確認し、破壊的操作を既定値にしない。
- ネットワーク取得では取得元、取得日時、失敗、再試行、レート制限を記録する。
- OCRや自動分類の誤りを前提に、信頼度の低い結果を未確認事項として扱う。
- 推定を出力する場合は、推定表示、根拠、計算方法、仮定、不確実性を同時に出力する。
