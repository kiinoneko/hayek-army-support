# 公開発信監視 Runbook v0.1

## 1. Runbookの位置づけ

- 公開発信監視Runbookは、政治行政アクターの公開発信を定期的に収集・整理するための作業手順書である。
- 本Runbookは、議会発言確認Runbook、quote_id台帳、actor_id台帳、policy_id台帳、source_id台帳、公開前チェックリストと接続する。
- 監視とは、非公開情報の探索ではなく、公開情報を体系的に確認することである。
- 公開発信の確認は、評価や批判の前に、原文、発信者、媒体、日時、文脈を確定する作業である。
- quote_idは必ずsource_idとactor_idに接続する。
- 公開発信を確認しても、直ちに虚偽、不正、癒着、違法、背信と断定してはならない。
- 外部発信、評価断定、法務官引渡は必要に応じてdecision_idを必要とする。

## 2. 対象

本Runbookの対象は、公開情報の範囲で公的職務、政策、事業、議会活動、行政説明、選挙活動、公的事業への関与に関連する発信を行う以下の主体である。

- 町長、市長、知事等の首長
- 副町長、副市長、副知事等
- 町議会議員、市議会議員、県議会議員、国会議員
- 議長、副議長、委員長、副委員長
- 監査委員
- 教育長、教育委員
- 議会同意人事
- 審議会、協議会、検討会の委員
- 行政職員による公式説明
- 社会福祉協議会、外郭団体、第三セクター等の役員
- 補助金交付先、委託契約先、指定管理者の代表者・担当者
- 政党、政治団体、候補者、後援会
- 公的事業に関与する団体

## 3. 使用する主なID

| ID | 用途 |
|---|---|
| "case_id" | 公開発信監視案件を管理する |
| "actor_id" | 発信者、関係者、責任線を管理する |
| "quote_id" | 発言、投稿、答弁、公約、説明を管理する |
| "source_id" | 発信元URL、議事録、動画、投稿等を管理する |
| "policy_id" | 関連政策、事業、議案、契約、補助金を管理する |
| "gap_id" | 原文未確認、対象者不明、媒体未確認等を管理する |
| "warning_id" | 削除可能性、文脈注意、要約注意を管理する |
| "stop_id" | 正式化停止、外部発信停止、法的断定停止を管理する |
| "decision_id" | 外部発信、評価断定、法務官引渡等を管理する |
| "change_id" | 発信記録、要約、評価の変更履歴を管理する |
| "incident_id" | 誤引用、人物誤認、誤公開等を管理する |

## 4. 公開情報限定の原則

- 対象は公開情報のみとする。
- 非公開アカウント、私信、DM、メール、内部資料、未公開資料は扱わない。
- 公的職務に関係しない私的情報は扱わない。
- 住所、電話番号、家族情報、健康情報、私生活情報等は扱わない。
- 公的職務に関係する氏名、役職、発言、議決、公式説明は記録対象とする。
- 公開情報であっても、個人攻撃、私生活暴露、過度な晒し行為に使ってはならない。
- スクリーンショットを保存・共有する場合は、個人情報、secret、第三者情報に注意する。
- GitHubにはPDF原本、スクリーンショット原本、個人情報、黒塗り前資料を追加しない。

## 5. 監視対象者確認手順

| 手順 | 内容 |
|---|---|
| 1 | 公式サイトで最新の首長、副首長、議員名簿を確認する |
| 2 | 議長、副議長、委員会構成、会派構成を確認する |
| 3 | 監査委員、教育長、審議会委員等の公的役職を確認する |
| 4 | 辞職、補欠選挙、役職変更、会派変更を確認する |
| 5 | actor_idを更新する |
| 6 | 発信時点の役職を記録する |
| 7 | 同姓同名や旧役職による誤認を避ける |

- 監視対象者は毎回公式情報で確認する。
- 古い名簿を根拠に対象者を固定しない。
- 発信時点と現在時点の役職を混同しない。

| 媒体 | 確認内容 |
|---|---|
| 公式サイト | 施政方針、声明、更新情報、資料 |
| 議会サイト | 議事録、会議録、議会録画、議会だより |
| X | 投稿、返信、引用、削除可能性 |
| Facebook | 投稿、コメント、画像、イベント |
| Instagram | 投稿、ストーリー、画像、動画 |
| YouTube | 動画、ライブ配信、概要欄、コメント |
| note | 記事、更新日、著者 |
| ブログ | 記事、投稿日、編集可能性 |
| 後援会サイト | 公約、活動報告、政策説明 |
| 広報紙 | 公式発信、行政説明 |
| 議会だより | 一般質問要約、議員発言、掲載文 |
| 報道記事 | コメント、引用、報道経由発言 |
| 会見録 | 首長会見、記者会見 |
| 審議会資料 | 発言要旨、議事要旨、委員発言 |

## 7. 監視頻度・対象期間

| 監視区分 | 対象期間 | 用途 |
|---|---|---|
| 日次確認 | 直近24時間 | 緊急案件、炎上、議会直前 |
| 週次確認 | 直近1週間 | 定期レポート、公開発信監視 |
| 月次確認 | 直近1か月 | 政策動向整理 |
| 会期中確認 | 議会会期中 | 一般質問、議案、討論、採決 |
| 選挙期確認 | 告示前後、選挙期間 | 公約、候補者発信 |
| 随時確認 | 案件発生時 | 補助金、契約、施設、行政改革等 |

- 監視頻度は案件区分に応じて設定する。
- 直近発言を重視する場合でも、過去発言との照合は必要に応じて行う。
- 最新情報を扱う場合は取得日・確認日を必ず記録する。

## 8. 収集手順

| 手順 | 内容 |
|---|---|
| 1 | 監視対象者一覧を確認する |
| 2 | 対象媒体を列挙する |
| 3 | 公式情報を先に確認する |
| 4 | SNS、動画、ブログ等を確認する |
| 5 | 発言・投稿・動画・資料URLを記録する |
| 6 | 発信日時、発信者、媒体を記録する |
| 7 | 原文、要約、評価を分ける |
| 8 | source_id候補を作成する |
| 9 | quote_id候補を作成する |
| 10 | policy_id候補に接続する |
| 11 | 未確認事項をgap_id化する |
| 12 | 文脈注意をwarning_id化する |

## 9. source_id化手順

| 確認項目 | 内容 |
|---|---|
| 資料名又は投稿名 | ページ名、投稿名、動画名等 |
| 公開元 | 自治体、議会、本人、報道機関等 |
| URL | 公開可能URL |
| 取得日 | 確認した日 |
| 発信日 | 投稿・発言・公開日 |
| 媒体 | X、YouTube、議事録等 |
| 本文確認状態 | 原文確認済、要約のみ、動画確認中等 |
| 保存先 | Google Drive等。GitHubには原本を置かない |
| 注意 | 削除・編集・差替え可能性 |

- source_idなしにquote_idを正式化しない。
- SNS投稿は削除・編集可能性があるためwarning_idを検討する。
- 動画は発言時刻と前後文脈を記録する。
- 報道記事は本人原文ではない場合があるため、一次資料を優先する。

## 10. actor_id接続手順

| 確認項目 | 内容 |
|---|---|
| 氏名 | 公式名簿、本人プロフィール等で確認 |
| 役職 | 発信時点の役職を記録 |
| 所属 | 会派、部署、団体等 |
| 公的職務性 | 公的職務に関係するか確認 |
| 関連政策 | policy_id候補と接続 |
| 注意 | 同姓同名、役職変更、会派変更 |

- 政治行政アクターは、公的影響を持つ範囲で氏名と役職を明記する。
- actor_idは責任線を追跡する索引であり、個人責任や違法性の断定根拠ではない。
- 公的職務と無関係な私的情報は記録しない。

## 11. quote_id化手順

| 手順 | 内容 |
|---|---|
| 1 | 発信者をactor_idで確認する |
| 2 | 発信元をsource_id化する |
| 3 | 発信日時を確認する |
| 4 | 発信媒体を確認する |
| 5 | 原文又は確認部分を記録する |
| 6 | 要約を別欄に記録する |
| 7 | 発言分類を行う |
| 8 | policy_idへ接続する |
| 9 | 文脈注意があればwarning_idを付与する |
| 10 | 原文未確認ならgap_id又はstop_idを付与する |

- quote_idは発言を追跡する索引であり、発言そのものではない。
- quote_idは必ずsource_idとactor_idに接続する。
- 原文、要約、評価を同じ欄に混在させない。

## 12. policy_id接続手順

| 発信内容 | policy_id候補 |
|---|---|
| 予算・決算 | 予算、決算、財政、基金、地方債 |
| 補助金・委託 | 補助金、委託契約、指定管理 |
| 福祉 | 社協、地域福祉、介護、障害福祉 |
| 教育 | 学校、教育委員会、給食、ICT |
| 防災 | 避難所、防災計画、消防、BCP |
| 行政改革 | 事務事業評価、外郭団体、情報公開 |
| 公共施設 | 図書館、体育館、プール、庁舎 |
| 交通 | コミュニティバス、ライドシェア、交通政策 |
| 観光 | 観光協会、イベント、観光船、観光PR |

- policy_idは発信と政策・事業を接続する索引である。
- policy_idだけで政策効果、失敗、不正を断定しない。
- 発言と実際の予算、議決、契約、成果は別途source_idで確認する。

## 13. 削除・編集・差替え対応

| 状態 | 対応 |
|---|---|
| 投稿削除 | 削除確認日、過去確認状態、保存可否を記録 |
| 投稿編集 | 編集前後の確認可否、編集日を記録 |
| 動画削除 | URL、確認日、代替資料を記録 |
| 議事録差替え | 版、更新日、差分確認 |
| 公式資料差替え | 取得日、版、旧版保存先を記録 |
| URL変更 | 旧URL、新URL、取得日を記録 |

- 削除されたことだけで不正や隠蔽と断定しない。
- 削除・編集はwarning_id又はgap_idとして管理する。
- スクリーンショットを扱う場合は公開区分、個人情報、著作権、secretに注意する。
- GitHubにはスクリーンショット原本を不用意に置かない。

## 14. gap_id / warning_id / stop_id 判定

| 状態 | 発行ID | 対応 |
|---|---|---|
| 発信原文が未確認 | "GAP_" 又は "STOP_" | quote_id正式化停止 |
| 発信者が未確認 | "GAP_" 又は "STOP_" | actor_id確認 |
| 発信日時が不明 | "GAP_" | 再確認 |
| 投稿削除・編集可能性 | "WARN_" | 取得日、確認状態を記録 |
| 前後文脈が不明 | "WARN_" | 文脈注意を明記 |
| 要約と原文が混在 | "WARN_" | 欄を分ける |
| source_idなし | "STOP_" | 正式化停止 |
| 法的断定が必要 | "STOP_" | 法務官確認 |
| 外部発信予定 | "DEC_" | USER_DECISION |

## 15. 外部発信前チェック

- 公開前チェックリストを使用する。
- source_idがあるか確認する。
- quote_idがsource_idとactor_idに接続しているか確認する。
- 原文、要約、評価を分けているか確認する。
- gap_idを隠していないか確認する。
- warning_idを反映しているか確認する。
- stop_idがある場合は公開しない。
- 違法、不正、癒着、汚職、虚偽を根拠なく断定しない。
- 個人攻撃になっていないか確認する。
- 公的職務と無関係な私的情報を含めない。
- 外部発信には必要に応じてdecision_idを取得する。

## 17. 禁止事項

- 非公開情報を収集しない。
- 私信、DM、メール、内部資料を扱わない。
- 公的職務と無関係な私的情報を扱わない。
- source_idなしにquote_idを正式化しない。
- 発信原文未確認のまま言質化しない。
- 要約を原文として扱わない。
- SNSの引用、リポスト、いいねを本人発言と混同しない。
- 削除や編集だけで不正、隠蔽、虚偽と断定しない。
- 文脈を無視して切り取らない。
- actor_idだけで個人責任、故意、違法、不正を断定しない。
- policy_idだけで政策失敗、不正、癒着を断定しない。
- 個人情報、黒塗り前資料、APIキー、secretをGitHubに追加しない。
- stop_idがある状態で外部発信しない。
- decision_idなしに外部発信、法務官引渡、評価断定へ進まない。

## 18. PR条件

- 新規ファイル追加のみ。
- 既存ファイルを変更しない。
- 既存ファイルを削除しない。
- archive配下を変更しない。
- PDF原本を追加しない。
- 個人情報を追加しない。
- 黒塗り前資料を追加しない。
- APIキー、パスワード、トークンを追加しない。
- mainへ直接コミットしない。
