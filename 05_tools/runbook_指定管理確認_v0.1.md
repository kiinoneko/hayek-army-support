# Runbook: 指定管理確認 v0.1

## 1. 位置づけ

このRunbookは、後方支援軍が公の施設の指定管理に関する資料を確認し、事実、推定、評価、未確認事項、情報公開請求候補を分離して整理するための標準手順である。

指定管理は、地方公共団体が公の施設の管理を指定管理者に行わせる制度であり、補助金や通常の委託契約とは区別して扱う。指定管理料、利用料金、自主事業収入、修繕費、外部発注費、一般財源を混同せず、資料に基づいて確認する。

このRunbookは、指定管理の募集、選定、協定、運営、収支、モニタリング、成果、再委託、直営・委託・廃止との比較を確認するために用いる。補助金、委託契約、入札制度全体の詳細分析は対象外とし、必要な場合は別Runbookまたは別case_idに分離する。

## 2. 対象範囲

対象に含めるものは次のとおりである。

- 公の施設の指定管理
- 指定管理者の募集
- 指定管理者の選定
- 指定管理協定
- 年度協定
- 指定管理料
- 利用料金制
- 自主事業
- 施設収支
- 事業報告
- モニタリング評価
- 再委託
- 施設利用実績と稼働率
- 修繕、維持管理、備品管理に関する指定管理上の負担区分
- 第三セクター、外郭団体、社会福祉協議会、観光協会、商工会等が指定管理者となる事例

対象外とするものは次のとおりである。

- 補助金制度そのものの詳細確認
- 通常の委託契約の詳細確認
- 入札制度全体の詳細確認
- 指定管理と直接関係しない政策評価
- PDF原本、画像、個人情報、secret、APIキーの収集またはGitHub追加

## 3. 指定管理の基本区分

指定管理を確認するときは、少なくとも次の区分を分けて記録する。

| 区分 | 確認内容 | 注意点 |
| --- | --- | --- |
| 施設 | 公の施設名、所在地、設置条例、所管部署 | 複数施設一括指定の場合は施設ごとに分ける |
| 指定期間 | 開始日、終了日、更新回数 | 年度協定の期間と混同しない |
| 指定管理者 | 法人名、団体名、共同体構成員 | 外郭団体や地域団体であるだけで癒着と断定しない |
| 募集方法 | 公募、非公募、特命的選定 | 非公募の場合は理由を資料で確認する |
| 料金区分 | 指定管理料、利用料金制、自主事業収入 | 収入源を混同しない |
| 支出区分 | 人件費、維持管理費、修繕費、再委託費 | 修繕負担の上限や分担を確認する |
| 評価区分 | モニタリング、事業報告、成果指標 | 未確認のまま成果なしと断定しない |

## 4. 確認する資料

確認資料は、source_idを付与して保存し、出典、取得日、発行主体、対象年度、対象施設を記録する。少なくとも次の資料を確認候補に含める。

- 指定管理者募集要項
- 仕様書
- 業務基準書
- 申請書
- 事業計画書
- 収支計画書
- 選定委員会資料
- 審査基準
- 評価結果
- 選定理由
- 議案資料
- 議決資料
- 基本協定
- 年度協定
- 指定管理料資料
- 利用料金資料
- 事業報告書
- 収支報告書
- モニタリング評価
- 利用実績資料
- 修繕・維持管理資料
- 再委託資料
- 予算書
- 決算書
- 主要施策成果報告書
- 議会資料

資料が要約版のみの場合は、重要断定を避け、原資料または情報公開請求候補に接続する。

## 5. 募集要項の確認観点

募集要項では、次の項目を確認する。

- 公の施設名
- 所管部署
- 指定期間
- 募集方法
- 公募または非公募の別
- 応募資格
- 共同体応募の可否
- 業務範囲
- 管理基準
- 指定管理料の上限または予定額
- 利用料金制の有無
- 自主事業の可否と条件
- 修繕負担の範囲
- 再委託の可否と承認条件
- リスク分担
- 申請書類
- 審査基準
- スケジュール
- 質問回答の公開状況

募集条件が特定団体に過度に限定されているように見える場合でも、直ちに不正、癒着、違法、利益供与と断定せず、条件設定の理由、施設特性、過去運営、法令・条例上の要件を確認する。

## 6. 選定手続の確認観点

選定手続では、次の項目を確認する。

- 応募者数
- 応募団体名または応募者数の公開状況
- 1者応募か複数応募か
- 書類審査、プレゼンテーション、ヒアリングの有無
- 選定日程
- 選定委員会の開催回数
- 利害関係者の除斥または利益相反確認
- 失格、辞退、不選定の有無
- 選定理由
- 評価結果の公開範囲

1者応募だけで直ちに競争性欠如、不正、癒着、違法、利益供与と断定しない。応募者数、募集期間、応募条件、過去の応募状況、質問回答、選定理由を確認したうえで評価する。

## 7. 選定委員会・審査基準の確認観点

選定委員会と審査基準では、次の項目を確認する。

- 委員会の設置根拠
- 委員構成
- 外部委員の有無
- 委員名の公開範囲
- 利益相反確認
- 審査項目
- 配点
- 最低基準点
- 価格評価と非価格評価の比率
- 施設管理能力の評価
- 利用者サービスの評価
- 収支計画の妥当性評価
- 地域連携の評価
- 災害対応、安全管理、個人情報保護の評価
- 評価結果と選定理由の整合性

審査基準が未確認の場合は、評価の妥当性について断定せず、gap_idまたは情報公開請求候補に接続する。

## 8. 指定管理協定・年度協定の確認観点

基本協定と年度協定では、次の項目を確認する。

- 協定当事者
- 指定期間
- 年度協定の対象年度
- 業務範囲
- 指定管理料の額と支払条件
- 利用料金の帰属
- 自主事業の承認条件
- 修繕負担の区分
- 備品、設備、消耗品の管理区分
- 再委託の承認条件
- 損害賠償、保険、不可抗力の扱い
- 個人情報保護、情報セキュリティ、秘密保持
- 事業報告、収支報告、モニタリングの提出条件
- 改善指示、指定取消、協定解除の条件

基本協定と年度協定の内容が異なる場合は、年度、対象施設、改定履歴を確認し、どの条件が当該年度に適用されるかを明示する。

## 9. 指定管理料・利用料金・自主事業収入の確認観点

収入区分は、必ず分けて確認する。

| 収入区分 | 確認観点 | 混同防止 |
| --- | --- | --- |
| 指定管理料 | 予算額、決算額、年度協定額、精算条件 | 委託料や補助金と同一視しない |
| 利用料金 | 利用料金制の有無、料金表、減免、指定管理者への帰属 | 使用料収入と利用料金収入を混同しない |
| 自主事業収入 | 承認条件、収入帰属、施設目的との整合 | 指定管理業務本体の収入と分ける |
| 一般財源 | 自治体負担、繰入、補填 | 指定管理者の収入と混同しない |

指定管理料、利用料金、自主事業収入、修繕費の区分が未確認のまま重要評価を行ってはならない。

## 10. 収支報告・決算との照合

収支報告と自治体決算を照合するときは、次の手順で確認する。

1. 対象年度、対象施設、指定管理者を固定する。
2. 事業報告書と収支報告書の収入・支出科目を確認する。
3. 予算書、決算書、主要施策成果報告書、議会資料の関連額を確認する。
4. 指定管理料、利用料金、自主事業収入、修繕費、再委託費を分けて整理する。
5. 年度協定額と決算額の差異を確認する。
6. 精算、返還、追加支払、繰越、赤字補填の有無を確認する。
7. 不一致がある場合は、科目名、会計区分、税込・税抜、複数施設一括計上の可能性を確認する。

決算書だけ、または事業報告書だけで重要断定を行わない。

## 11. モニタリング評価の確認観点

モニタリング評価では、次の項目を確認する。

- 評価主体
- 評価年度
- 評価対象施設
- 評価項目
- 評価基準
- 評価結果
- 利用者アンケート
- 苦情、事故、改善要望
- 改善指示
- 改善報告
- 次年度協定または次回選定への反映
- 評価結果の公表状況

モニタリング未確認のまま成果なし、管理不全、制度失敗と断定しない。評価結果が非公開または未掲載の場合は、gap_idまたは情報公開請求候補に接続する。

## 12. 事業報告・成果指標の確認観点

事業報告と成果指標では、次の項目を確認する。

- 年間事業内容
- 管理運営業務の実施状況
- 自主事業の実施状況
- 利用者数
- 稼働率
- 利用料金収入
- 収支結果
- 苦情、事故、修繕、トラブル対応
- 成果指標
- 目標値と実績値
- 未達理由
- 改善策

利用者数の減少だけで制度失敗と断定しない。人口動態、休館、工事、災害、感染症、施設老朽化、料金改定、周辺競合、事業内容の変更など、外部要因と施設固有要因を分けて確認する。

## 13. 再委託・外部発注の確認観点

再委託と外部発注では、次の項目を確認する。

- 再委託の可否
- 事前承認の要否
- 再委託先
- 再委託業務の範囲
- 金額
- 契約方法
- 指定管理者の責任範囲
- 丸投げ禁止条項の有無
- 外郭団体、関連会社、構成団体への発注の有無
- 利益相反確認
- 再委託費と修繕費、維持管理費の区分

再委託があるだけで直ちに不正または違法と断定しない。協定、仕様書、承認資料、金額、業務範囲、責任分担を確認する。

## 14. 施設利用実績・稼働率の確認観点

利用実績と稼働率では、次の項目を確認する。

- 利用者数
- 利用件数
- 稼働率
- 開館日数
- 休館日数
- 部屋別、施設別、時間帯別の利用状況
- 減免利用
- 有料利用と無料利用
- 料金改定の影響
- 工事、災害、感染症、臨時休館の影響
- 目標値との比較
- 過年度推移
- 類似施設との比較可能性

稼働率の算定方法が資料間で異なる場合は、分母、分子、対象時間、対象室、休館日の扱いを確認する。

## 15. 直営・委託・廃止との比較観点

直営、通常委託、廃止との比較では、次の項目を確認する。

- 指定管理を選択した理由
- 直営時の人件費、維持管理費、利用者数、サービス水準
- 通常委託との役割分担の違い
- 廃止または統廃合の検討有無
- 公共性、採算性、地域必要性
- 利用者サービスの変化
- 行政負担の変化
- リスク移転の範囲
- 施設老朽化と将来修繕負担
- 指定管理者の創意工夫の成果

比較資料がない場合は、直営より良い、委託より悪い、廃止すべきなどの断定を避け、比較検討資料の有無を情報公開請求候補に接続する。

## 16. source_idとの接続

各資料にはsource_idを付与し、次の項目を記録する。

| 項目 | 記録内容 |
| --- | --- |
| source_id | 一意の資料ID |
| 資料名 | 例: 指定管理者募集要項、基本協定、事業報告書 |
| 発行主体 | 自治体、所管部署、選定委員会、指定管理者等 |
| 取得元 | 公式URL、議会資料、情報公開請求等 |
| 取得日 | YYYY-MM-DD |
| 対象年度 | 年度または指定期間 |
| 対象施設 | 公の施設名 |
| 公開状態 | 公開、非公開、黒塗り、要請中、不明 |
| 注意 | 要約版、抜粋、改定版、複数施設一括等 |

PDF原本や画像はGitHubへ追加しない。必要な場合は、公開URL、資料名、取得日、該当ページまたは項目をテキストで記録する。

## 17. case_idとの接続

case_idには、対象施設、自治体、年度、指定管理者、確認テーマを紐づける。

例:

```text
case_id: sm-<自治体コードまたは略称>-<施設略称>-<年度>-<連番>
```

case_idで扱う範囲は、原則として1施設または1指定管理単位に限定する。複数施設一括指定の場合は、共通協定と施設別実績を分けて記録する。補助金、通常委託、入札制度全体の論点が出た場合は、別case_idまたは別Runbookに分離する。

## 18. gap_id / warning_id / stop_id の発行条件

### gap_id

次の場合はgap_idを発行する。

- 募集要項が未確認
- 仕様書または業務基準書が未確認
- 審査基準または配点が未確認
- 選定理由が未確認
- 基本協定または年度協定が未確認
- 指定管理料、利用料金、自主事業収入の区分が未確認
- 事業報告書または収支報告書が未確認
- モニタリング評価が未確認
- 利用実績または稼働率が未確認
- 再委託資料が未確認
- 直営・委託・廃止との比較資料が未確認

### warning_id

次の場合はwarning_idを発行する。

- 1者応募だが、募集要項、応募条件、選定理由の確認が不十分
- 外郭団体、第三セクター、社会福祉協議会、観光協会、商工会等が指定管理者で、利益相反確認や選定理由が未確認
- 指定管理料と利用料金収入の区分が資料上わかりにくい
- 修繕負担の上限、緊急修繕、資本的修繕の扱いが不明
- 収支報告と決算書の金額が一致しない
- モニタリング評価が要約のみ
- 利用者数または稼働率の算定方法が不明
- 再委託先や金額が非公開または要約のみ
- 施設別実績と複数施設一括実績が混在している

### stop_id

次の場合は作業を停止し、公開前チェックまたは責任者確認へ回す。

- 募集要項、協定、事業報告、収支報告、モニタリングが未確認のまま不正・癒着・違法と断定しようとしている
- 指定管理料、利用料金、自主事業収入、修繕費の区分が未確認のまま重要評価を行う
- 応募者数や選定理由が未確認のまま競争性を断定する
- モニタリング、利用実績、成果指標が未確認のまま成果なしと断定しようとしている
- 個人情報、黒塗り前資料、非公開資料、secretの混入疑いがある
- PDF原本や画像をGitHubへ追加しようとしている
- 要約資料だけで重要断定を行おうとしている
- 旧本文と新本文が混在している
- 変更対象外ファイルが差分に入っている
- README.md、archive、04_templatesが目的外に変更されている

## 19. 情報公開請求候補への接続

公開資料だけで確認できない場合は、次の資料を情報公開請求候補に接続する。

- 募集要項、仕様書、業務基準書の完全版
- 申請書、事業計画書、収支計画書
- 選定委員会資料、審査表、採点表、議事録
- 選定理由の詳細資料
- 基本協定、年度協定、変更協定
- 指定管理料の積算資料
- 利用料金の承認資料
- 自主事業の承認資料
- 事業報告書、収支報告書の詳細版
- モニタリング評価、改善指示、改善報告
- 利用実績、稼働率、利用者アンケート
- 修繕、維持管理、備品管理の資料
- 再委託承認資料、再委託先一覧、金額資料
- 直営・委託・廃止との比較検討資料

情報公開請求候補には、請求目的、対象年度、対象施設、必要な資料名、個人情報を求めない旨を明記する。

## 20. 公開前チェックへの引き渡し

公開前チェックへ引き渡す前に、次の項目を確認する。

- 事実、推定、評価、未確認事項、情報公開請求候補が分離されている
- 不正、癒着、違法、利益供与などの強い表現に根拠資料がある
- 1者応募、外郭団体、地域団体という事実だけで断定していない
- 指定管理料、利用料金、自主事業収入、修繕費、委託費、一般財源を混同していない
- モニタリング、利用実績、成果指標を確認している
- 個人情報、secret、APIキー、黒塗り前資料、非公開資料が含まれていない
- PDF原本や画像を追加していない
- source_idとcase_idが接続されている
- gap_id、warning_id、stop_idの扱いが記録されている

## 21. 禁止事項

次の行為を禁止する。

- 指定管理を補助金または通常の委託契約と混同する
- 補助金、委託契約、入札制度全体の詳細確認をこのRunbookで扱う
- 1者応募だけで不正、癒着、違法、利益供与と断定する
