# Runbook: 公金支出類型判定 v0.1

## 1. 位置づけ

このRunbookは、国、都道府県、市町村、外郭団体、第三セクター、社会福祉協議会、観光協会、商工会、協議会、実行委員会等に関係する公金支出について、初動段階で支出類型を整理し、適切なRunbookまたはcase_idへ接続するための判定手順である。

本Runbookで行うのは、支出が補助金、委託契約、指定管理、調達手続、負担金、交付金、補償金、扶助費、直営事業費、その他支出のいずれに近いかを分類することに限る。補助金、委託契約、指定管理、調達手続それぞれの詳細確認は、接続先Runbookで扱う。

## 2. 対象範囲

対象に含める支出類型は以下とする。

- 補助金、交付金、負担金、分担金
- 国庫補助金、国庫負担金、国庫交付金
- 県補助金、市町村補助金
- 委託料、通常委託
- 指定管理料、利用料金制
- 調達契約、工事請負、物品購入、役務提供
- 随意契約、プロポーザル、入札、見積合わせ
- 扶助費、補償金、報償費
- 交際費、旅費、需用費、備品購入費
- 直営事業費
- 外郭団体、協議会、実行委員会経由の支出
- 上記に当てはまらないその他支出

## 3. 公金支出類型判定の基本原則

1. 予算科目だけで最終分類しない。
2. 支出名だけで最終分類しない。
3. 支出先が外郭団体、社会福祉協議会、観光協会、商工会等であるだけで、補助金、委託、癒着、不正と断定しない。
4. 補助金と委託料を混同しない。
5. 指定管理料と通常の委託料を混同しない。
6. 国庫補助金、国庫負担金、国庫交付金、地方交付税、地方債、一般財源を混同しない。
7. 事実、推定、評価、未確認事項、情報公開請求候補を分離する。
8. 類型不明の場合は断定せず、gap_idまたは情報公開請求候補へ接続する。
9. 複数の性質が混在する場合は、親case_idを作り、支出類型ごとに子case_idへ分離する。

## 5. 予算科目による初動判定

予算科目と歳出節は初動判定の入口として使う。ただし、最終分類は契約、交付決定、協定、仕様書、成果物、精算、返還条件等と突合して行う。

| 予算科目・歳出節の例 | 初動で疑う類型 | 追加確認 |
| --- | --- | --- |
| 負担金補助及び交付金 | 補助金、交付金、負担金、分担金 | 交付決定、補助要綱、負担根拠、精算、返還条件 |
| 委託料 | 委託契約、指定管理、役務提供 | 契約書、仕様書、成果物、検収、再委託、協定の有無 |
| 工事請負費 | 調達契約、工事請負 | 入札結果、契約書、設計書、変更契約、検査調書 |
| 需用費 | 直営事業費、物品・印刷・修繕等の調達 | 見積、請求書、契約方式、納品・検収 |
| 備品購入費 | 物品購入、調達契約 | 仕様、見積、入札、納品、検収 |
| 扶助費 | 扶助費 | 給付根拠、対象者要件、支給決定、個人情報保護 |
| 補償補填及び賠償金 | 補償金、賠償金 | 補償根拠、算定資料、合意書、議決の有無 |
| 報償費 | 報償費、謝金 | 支払対象、依頼内容、源泉徴収、成果確認 |
| 旅費 | 旅費、直営事業費 | 出張命令、用務、行程、復命書 |
| 交際費 | 交際費 | 支出基準、相手方、目的、公開範囲 |

## 6. 支出名・事業名による初動判定

支出名や事業名に含まれる語は、類型判定の手がかりにすぎない。名称と実態がずれる場合があるため、必ず根拠資料を確認する。

| 名称に含まれる語 | 初動で疑う類型 | 確認する資料 |
| --- | --- | --- |
| 補助、助成、奨励 | 補助金 | 交付要綱、交付決定、実績報告、精算書 |
| 交付、給付 | 交付金、扶助費、補助金 | 交付根拠、対象者、使途制限、返還条件 |
| 負担、分担 | 負担金、分担金 | 協定、規約、法令、負担割合 |
| 委託、業務 | 委託契約、役務提供 | 契約書、仕様書、成果物、検収 |
| 指定管理、管理運営 | 指定管理 | 指定議案、協定、事業計画、モニタリング |
| 工事、修繕、購入 | 調達契約、工事請負、物品購入 | 入札結果、契約結果、仕様書、検査調書 |
| 実行委員会、協議会 | 補助金、負担金、委託、その他 | 団体規約、収支報告、契約・交付決定の有無 |

## 7. 支出先による初動判定

支出先は、支出類型を断定する根拠ではなく、追加確認の優先順位を決める材料である。

- 外郭団体、第三セクター、社会福祉協議会、観光協会、商工会、協議会、実行委員会への支出は、補助金、委託料、指定管理料、負担金、交付金が混在しやすい。
- 民間事業者への支出は、委託契約、調達契約、工事請負、物品購入、役務提供を確認する。
- 個人への支出は、扶助費、補償金、報償費、旅費、交際費等を確認し、個人情報を混入させない。
- 国、都道府県、市町村、広域連合、一部事務組合への支出は、負担金、分担金、償還金、交付金等を確認する。

## 8. 支出根拠による初動判定

以下の根拠を確認し、支出の法的・事務的性質を分ける。

- 根拠法令、条例、規則、要綱、要領
- 契約書、仕様書、見積書、請書
- 交付決定通知、変更交付決定、確定通知
- 協定書、基本協定、年度協定、指定議案
- 入札公告、プロポーザル実施要領、選定結果
- 実績報告書、事業報告書、収支報告書
- 成果物、納品物、検収調書、完了届
- 精算書、返還命令、返還条件

## 9. 補助金に分類する条件

以下が中心であれば、補助金として分類する。

- 行政が公益上必要と認める事業に対し、相手方の事業費の一部または全部を助成している。
- 交付決定、変更交付決定、額の確定、実績報告、精算、返還条件がある。
- 補助対象経費、補助率、上限額、自己負担、対象外経費が定められている。
- 支出先の収入側で補助金、助成金、県補助金、市町村補助金、国庫補助金等として処理されている。
- 行政が成果物を直接購入するのではなく、相手方の事業実施を財政的に支援している。

国庫補助金、県補助金、市町村補助金は、財源または交付主体を示す場合がある。地方公共団体から最終支出先への支出類型が補助金かどうかは、交付決定や補助要綱で確認する。

## 10. 委託契約に分類する条件

以下が中心であれば、委託契約として分類する。

- 行政が実施すべき業務を相手方に委ね、契約に基づき対価を支払っている。
- 契約書、請書、仕様書、業務計画、成果物、完了報告、検収、支払がある。
- 支払は反対給付の対価であり、成果物または役務提供の確認がある。
- 再委託、変更契約、履行期間、契約保証等の契約管理事項がある。
- 支出先の収入側で受託料、委託料、業務収入等として処理されている。

補助金と委託契約が同一団体に支出されている場合は、交付決定に基づく支出と契約に基づく支出を分けて子case_id化する。

## 11. 指定管理に分類する条件

以下が中心であれば、指定管理として分類する。

- 対象が公の施設であり、指定管理者制度に基づく指定がある。
- 指定議案、指定管理者一覧、基本協定、年度協定がある。
- 指定管理料、利用料金制、利用料金併用制、管理運営業務の範囲が定められている。
- 事業計画、事業報告、収支報告、モニタリング、評価結果がある。
- 協定により施設管理、利用許可、料金収受、修繕負担、リスク分担が定められている。

公の施設の管理運営であっても、指定議案や協定が確認できない場合は、通常の委託契約またはその他支出の可能性を残してgap_idを発行する。

## 12. 調達手続に分類する条件

以下が中心であれば、調達手続として分類する。

- 行政が物品、工事、役務、印刷、修繕、備品等を購入または発注している。
- 入札、随意契約、プロポーザル、見積合わせ、1者応札、落札、契約締結がある。
- 予定価格、落札率、契約額、見積額、選定理由、随意契約理由が確認対象である。
- 契約結果一覧、入札結果、公告、仕様書、契約書、検査調書がある。
- 工事請負、物品購入、役務提供、備品購入、需用費による購入が含まれる。

委託契約は調達手続を経て締結されることがある。問題意識が契約履行や成果物なら委託契約Runbookへ、選定過程や契約方式なら調達手続Runbookへ接続する。

## 13. 負担金・交付金・分担金に分類する条件

以下が中心であれば、負担金、交付金、分担金として分類する。

- 法令、条例、規約、協定、広域事務、共同事業等に基づく負担割合がある。
- 国庫負担金、県負担金、市町村負担金等、費用負担の義務または割合が定められている。
- 交付金として、使途、算定式、配分基準、対象事業が定められている。
- 分担金として、受益、区域、構成団体、事業費按分等の根拠がある。
- 相手方が行政機関、広域連合、一部事務組合、協議会、共同事業体である。

地方交付税、地方債、一般財源は、個別支出の支出類型ではなく財源または歳入・資金調達の性質を持つため、国庫補助金、国庫負担金、国庫交付金と混同しない。

## 14. 扶助費・補償金・報償費に分類する条件

扶助費、補償金、報償費は以下により分類する。

- 扶助費: 法令、条例、要綱等に基づき、対象者要件を満たす個人または世帯へ給付する支出。個人情報を扱うため、公開資料への転記は最小限にする。
- 補償金: 行政行為、用地取得、損失補償、休業補償、移転補償等の算定根拠に基づく支出。
- 報償費: 講師謝金、委員謝礼、協力者謝礼、記念品等、役務や協力への謝礼性を持つ支出。

これらに契約書や仕様書がある場合でも、実質が給付、補償、謝礼のどれかを確認し、個人情報や非公開情報を混入させない。

## 15. 直営事業費に分類する条件

以下が中心であれば、直営事業費として分類する。

- 行政が自ら事業を実施し、職員、施設、消耗品、印刷、旅費、備品、使用料等を組み合わせて支出している。
- 支出先ごとの契約や購入は存在しても、事業全体は補助金、委託、指定管理ではない。
- 需用費、旅費、役務費、使用料及び賃借料、備品購入費等が複数組み合わさっている。
- 支出先ではなく、行政内部の事業目的、実施内容、成果、予算執行が主な確認対象である。

直営事業内に個別の委託、物品購入、工事請負が含まれる場合は、必要に応じて子case_idへ分ける。

## 16. 複合支出・混在支出の扱い

同一事業、同一団体、同一年度に複数類型が混在する場合は、以下の手順で扱う。

1. 事業全体を親case_idにする。
2. 支出類型ごとに子case_idを作成する。
3. 補助金、委託契約、指定管理、調達手続、負担金、交付金、直営事業費を分ける。
4. 各子case_idにsource_id、支出額、年度、支出主体、支出先を紐づける。
5. 混在していること自体は事実として記録し、不正、癒着、違法、利益供与とは断定しない。

| 中心となる確認事項 | 接続先 |
| --- | --- |
| 補助金、交付決定、補助率、対象経費、実績報告、精算、返還条件 | `runbook_補助金支出確認_v0.1.md` |
| 契約書、仕様書、成果物、検収、支払、再委託 | `runbook_委託契約確認_v0.1.md` |

| 類型不明、資料不足、名称と実態の不一致 | gap_idまたは情報公開請求候補 |

複数の性質が混在する場合は、親case_idを作り、支出類型ごとに子case_idへ分離する。

## 18. source_idとの接続

source_idには、少なくとも以下を記録する。

- 資料名
- 作成主体または公開主体
- 年度
- 公開URLまたは取得経路
- 確認日
- 該当ページ、表、項目名
- 支出主体、支出先、支出名、事業名
- 予算科目、歳出節、金額
- 根拠法令、条例、規則、要綱、要領、仕様書

要約資料しかない場合は、要約資料であることを明示し、重要断定には使わない。

## 19. case_idとの接続

case_idには、以下を記録する。

- 親case_idまたは子case_idの別
- 支出年度
- 支出主体
- 支出先
- 支出名、事業名
- 初動分類
- 確認済み資料
- 未確認資料
- 接続先Runbook
- gap_id、warning_id、stop_idの有無
- 情報公開請求候補の有無

類型が確定できない場合は、case_idに「未分類」または「類型未確認」と記録し、推定分類と確定分類を分ける。

## 20. gap_id / warning_id / stop_id の発行条件

### gap_id

gap_idは、資料不足により分類または接続先判断ができない場合に発行する。

- 契約の有無が不明
- 交付決定の有無が不明
- 協定の有無が不明
- 指定管理の有無が不明
- 成果物または反対給付の有無が不明
- 精算、返還条件の有無が不明
- 団体収入側の科目が不明
- 国費、県費、市町村費、自己財源の混在有無が不明

### warning_id

warning_idは、誤分類や過度な推定の危険がある場合に発行する。

- 予算科目と支出実態が一致しない可能性がある。
- 支出名と契約・交付・協定の実態が一致しない可能性がある。
- 外郭団体、協議会、実行委員会経由で複数財源が混在している。
- 国庫補助金、国庫負担金、国庫交付金、県費、市町村費、自己財源が混在している。
- 同一団体に補助金、委託料、指定管理料、負担金が併存している。

### stop_id

stop_idは、作業を停止し、分類や評価を確定しない条件である。

- 支出類型が未確認のまま不正・癒着・違法・利益供与と断定しようとしている。
- 補助金、委託契約、指定管理、調達手続を混同したまま重要評価を行う。
- 予算科目、契約書、交付決定、協定、仕様書、事業報告が未確認のまま分類を確定しようとしている。
- 国費、県費、市町村費、自己財源の区分が未確認のまま財源評価を断定する。
- 個人情報、黒塗り前資料、非公開資料、secretの混入疑いがある。
- PDF原本や画像をGitHubへ追加しようとしている。
- 要約資料だけで重要断定を行おうとしている。
- 旧本文と新本文が混在している。
- 変更対象外ファイルが差分に入っている。
- README.md、archive、04_templatesが目的外に変更されている。

## 21. 情報公開請求候補への接続

以下が未確認の場合は、情報公開請求候補に接続する。

- 契約書、仕様書、請書、変更契約書
- 交付決定通知、変更交付決定通知、額の確定通知
- 指定管理の基本協定、年度協定、モニタリング資料
- 入札公告、入札結果、予定価格、落札率、見積書
- 随意契約理由書、プロポーザル審査資料、選定委員会資料
- 実績報告書、事業報告書、収支報告書、精算書
- 団体決算書、団体収入側の科目明細
- 国費、県費、市町村費、自己財源の財源内訳

請求候補には、請求したい文書名、対象年度、支出主体、支出先、事業名、必要理由を記録する。

## 22. 公開前チェックへの引き渡し

公開前チェックに引き渡す前に、以下を確認する。

- 事実、推定、評価、未確認事項、情報公開請求候補が分離されている。
- 類型分類の根拠となるsource_idが記録されている。
- 不正、癒着、違法、利益供与等の評価語が、資料未確認のまま使われていない。
- 個人情報、黒塗り前資料、非公開資料、secret、APIキーが含まれていない。
- PDF原本、画像、不要な添付資料がGitHubに追加されていない。
- 複合支出は親case_idと子case_idに分離されている。

## 23. 禁止事項

- 予算科目だけで最終分類すること。
- 支出名だけで最終分類すること。
- 支出先の属性だけで補助金、委託、癒着、不正と断定すること。
- 補助金と委託料を混同すること。
- 指定管理料と通常の委託料を混同すること。
- 国庫補助金、国庫負担金、国庫交付金、地方交付税、地方債、一般財源を混同すること。
- 類型不明のまま重要評価を確定すること。
- 個人情報、secret、APIキー、非公開資料を追加すること。
- PDF原本、画像をGitHubへ追加すること。
- 既存Runbook本文をコピーして混入させること。
- 過去の壊れ差分を再利用すること。
- 旧本文と新本文を併記すること。
- README.md、archive、04_templates、他の05_toolsファイルを目的外に変更すること。
