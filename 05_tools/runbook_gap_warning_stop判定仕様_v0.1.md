# gap_id / warning_id / stop_id 判定仕様 Runbook v0.1

## 1. 位置づけ

このRunbookは、後方支援軍の作業において発生する未確認事項、注意事項、停止条件を `gap_id` / `warning_id` / `stop_id` として判定・記録・引き渡すための標準仕様である。

対象は、資料確認、成果物作成、公開前チェック、情報公開請求候補の整理、法務官引渡前の確認である。個別案件の判断を属人的にせず、どの案件でも同じ基準で記録できるようにする。

このRunbookは判定基準を定めるものであり、既存Runbook本文、PDF原本、画像、個人情報、secret、APIキー、非公開資料を取り込まない。

## 2. gap_id / warning_id / stop_id の定義

| id種別 | 定義 | 基本的な扱い |
|---|---|---|
| `gap_id` | 根拠不足、資料欠落、年度不明、数値差異、原資料未取得など、追加確認が必要な未確認事項。 | 成果物への反映前に追加確認する。影響が限定的な場合は、未確認箇所を明示して作業を継続できる。 |
| `warning_id` | 使用は可能だが、要約資料のみ、更新可能性あり、引用注意、数値の制約など、成果物に注意書きが必要な事項。 | 注意書き、出典注記、前提条件、確認日を付けて使用できる。 |
| `stop_id` | 公開、PR、PDF化、外部発信、情報公開請求、法務官引渡を一時停止すべき重大条件。 | 解除条件を満たすまで対象作業を停止する。 |

## 3. gap_idを発行する条件

次のいずれかに該当する場合は、`gap_id` を発行する。

1. 根拠資料が不足しており、断定に必要な一次資料または確認済み資料がそろっていない。
2. 参照した資料の年度、版、発行日、更新日、対象期間が不明である。
3. 同一項目について資料間の数値、名称、日付、制度名、担当部署名が一致しない。
4. 原資料が未取得で、要約、二次資料、メモ、転記のみを参照している。
5. source_idが未付与、重複、または参照先不明である。
6. case_idとの関連付けが未確定である。
7. 引用予定箇所のページ、行、表番号、URL、取得日が不足している。
8. 情報公開請求で確認すべき資料名、所管、期間、請求対象が未整理である。
9. 公開前チェックで確認すべき未解決事項が残っている。

## 4. warning_idを発行する条件

次のいずれかに該当し、使用自体は可能だが成果物に注意書きが必要な場合は、`warning_id` を発行する。

1. 参照資料が要約資料、概要資料、説明資料、議事要旨、プレスリリースなどであり、原資料ではない。
2. 資料が今後更新される可能性がある。
3. 数値が概算、速報値、見込み、暫定値、推計値、四捨五入値である。
4. 引用文の範囲が短く、文脈説明が必要である。
5. 制度名、組織名、役職名、様式名が時点により変わる可能性がある。
6. source_idは確認済みだが、参照条件、取得日、対象年度を明示しないと誤解を招く。
7. case_idに接続できるが、他案件にも影響する可能性がある。
8. 公開前チェックで注意書き付き使用可と判断された。

## 5. stop_idを発行する条件

次のいずれかに該当する場合は、`stop_id` を発行し、公開、PR、PDF化、外部発信、情報公開請求、法務官引渡のうち影響する作業を停止する。

1. 本文未確認のまま重要断定が必要である。
2. 数値根拠が確認できない。
3. secret、APIキー、個人情報、黒塗り前情報の混入疑いがある。
4. PDF原本や非公開資料をGitHubへ追加しようとしている。
5. 引用文と原文の一致が確認できない。
6. 旧本文と新本文が混在している。
7. 変更対象外ファイルが差分に入っている。
8. archiveやREADME.mdが目的外に変更されている。
9. 法的評価、外部発信、名誉毀損リスクがあるのに根拠確認が未了である。
10. 公開前チェックで停止判断が出ている。
11. 情報公開請求候補として扱う前に、請求対象、根拠、公開可否の確認が不足している。
12. 法務官引渡に必要な根拠、引用、差分、出典、時点情報が確認できない。

## 6. gap / warning / stop の判定表

| 状況 | gap_id | warning_id | stop_id | 判断の目安 |
|---|---:|---:|---:|---|
| 原資料が未取得で、要約資料のみ確認済み | 要 | 要 | 不要 | 断定せず、要約資料であることを明記する。 |
| 年度が不明 | 要 | 場合により要 | 場合により要 | 年度不明が結論に影響するなら停止する。 |
| 数値差異がある | 要 | 場合により要 | 場合により要 | 差異が小さく注記可能ならwarning、結論を左右するならstop。 |
| 引用文と原文の一致未確認 | 要 | 不要 | 要 | 引用を含む公開・PR・PDF化を停止する。 |
| secretまたは個人情報の混入疑い | 不要 | 不要 | 要 | 直ちに差分確認と除去を行う。 |
| source_id未付与 | 要 | 不要 | 場合により要 | 重要根拠ならstop、補助情報ならgap。 |
| case_id未接続 | 要 | 不要 | 場合により要 | 引渡や公開判断に影響する場合はstop。 |
| 更新可能性がある公開資料 | 不要 | 要 | 不要 | 取得日、版、確認日を明記する。 |
| README.mdまたはarchiveの目的外変更 | 不要 | 不要 | 要 | PR前に差分から除外する。 |

## 7. source_idとの接続

`source_id` は、根拠資料を追跡するための識別子である。`gap_id` / `warning_id` / `stop_id` は、可能な限り対象の `source_id` と接続する。

接続時は、次を確認する。

1. `source_id` が一意である。
2. 資料名、発行主体、発行日または確認日、URLまたは保管場所、対象年度が記録されている。
3. 引用箇所がある場合は、ページ、章、表番号、行、項目名などを記録している。
4. 原資料未取得の場合は `gap_id` を発行する。
5. 要約資料のみを使用する場合は `warning_id` を発行し、成果物に注意書きを付ける。
6. 根拠資料の真正性、公開可否、引用一致が確認できない場合は `stop_id` を発行する。

## 8. case_idとの接続

`case_id` は、作業対象、案件、論点、請求候補、成果物単位を追跡するための識別子である。各idは、影響する `case_id` に接続する。

接続時は、次を確認する。

1. 対象案件の `case_id` が明確である。
2. 複数案件に影響する場合は、すべての `case_id` を記録する。
3. `case_id` が未定の場合は、仮の管理名を置き、`gap_id` として未確定理由を記録する。
4. `stop_id` が発行された場合は、停止対象の `case_id` と停止範囲を明記する。
5. 解除済みになった場合も、解除日、解除理由、確認者を `case_id` 側で追跡できるようにする。

## 9. 公開前チェックとの接続

公開前チェックでは、すべての `gap_id` / `warning_id` / `stop_id` を確認する。

公開前チェックへ引き渡す条件は次のとおりである。

1. 未対応、確認中、追加資料待ち、情報公開請求候補、停止中、公開前再確認必要の状態が残っている。
2. 成果物に注意書きが必要な `warning_id` がある。
3. 引用、数値、年度、source_id、case_idのいずれかに未確認事項がある。
4. 外部発信、PDF化、法務官引渡、情報公開請求に接続する可能性がある。
5. README.md、archive、変更対象外ファイル、PDF、画像、secret、個人情報が差分に入っていないことを確認する必要がある。

公開前チェックの結果は、台帳の状態、再確認条件、解除条件、公開前チェック引渡要否に反映する。

## 10. 情報公開請求候補との接続

情報公開請求候補は、未取得資料や確認不能事項を解消するための候補である。`gap_id` または `warning_id` から情報公開請求候補へ接続できる。

接続する場合は、次を記録する。

1. 請求候補にした理由。
2. 必要な資料名、対象年度、対象部署、対象期間。
3. 既存資料で確認できた範囲と確認できない範囲。
4. 請求しない場合の影響。
5. 請求前に公開資料で再確認すべき事項。
6. 請求文案に含めてはならない個人情報、secret、非公開情報の有無。

請求対象や根拠が未整理のまま外部提出に進む場合は、`stop_id` を発行する。

## 11. 再確認・解除・再開条件

| 対象 | 再確認条件 | 解除条件 | 再開条件 |
|---|---|---|---|
| `gap_id` | 原資料、年度、数値、source_id、case_idの不足が解消されたか確認する。 | 不足事項が解消し、判定理由と確認結果が台帳に記録された。 | 影響範囲の成果物へ反映し、必要なら公開前チェックへ引き渡す。 |
| `warning_id` | 注意書き、取得日、対象期間、制約条件が成果物に反映されているか確認する。 | 注意付き使用可の条件が満たされ、読者が制約を理解できる記述になった。 | 注意書きを残したまま成果物利用を再開する。 |
| `stop_id` | 停止理由、混入疑い、引用一致、数値根拠、変更範囲、法的リスクを再確認する。 | 停止理由が解消し、必要な除去、修正、根拠確認、レビューが完了した。 | 解除記録を残し、停止対象だった公開、PR、PDF化、外部発信、情報公開請求、法務官引渡を再開する。 |

解除時は、解除した理由だけでなく、何を確認して解除したかを記録する。解除条件が不明確な場合は、解除済みにせず、公開前再確認必要または停止中を維持する。

## 12. 台帳の最小項目

台帳には、少なくとも次の項目を置く。

| 項目 | 内容 |
|---|---|
| id | `gap_id`、`warning_id`、`stop_id` の識別子。 |
| 種別 | gap、warning、stopの別。 |
| case_id | 関連する案件、論点、成果物、請求候補の識別子。 |
| source_id | 関連する根拠資料の識別子。未確定の場合は未確定と記録する。 |
| 発生日 | 判定を発行した日。 |
| 発生箇所 | ファイル、章、表、行、資料箇所、作業工程など。 |
| 内容 | 発生した未確認事項、注意事項、停止条件の概要。 |
| 判定理由 | なぜgap、warning、stopにしたか。 |
| 影響範囲 | 影響する成果物、PR、PDF、外部発信、請求候補、法務官引渡など。 |
| 必要対応 | 確認、修正、注記、資料取得、差分除外、レビューなど。 |
| 担当 | 対応担当者または担当チーム。 |
| 状態 | 現在の状態分類。 |
| 再確認条件 | いつ、何を、どの資料で再確認するか。 |
| 解除条件 | どの条件を満たせば解除できるか。 |
| 公開前チェック引渡要否 | 公開前チェックへ渡す必要があるか。 |
| 備考 | 補足、関連PR、確認履歴、判断保留理由など。 |

状態分類は、少なくとも次を使用する。

| 状態 | 説明 |
|---|---|
| 未対応 | 発行後、まだ対応に着手していない。 |
| 確認中 | 資料、差分、引用、数値、公開可否などを確認している。 |
| 追加資料待ち | 追加資料、原資料、確認回答を待っている。 |
| 情報公開請求候補 | 情報公開請求で確認する候補になっている。 |
| 注意付き使用可 | 注意書き、制約、取得日などを付ければ使用できる。 |
| 停止中 | stop_idにより対象作業を停止している。 |
| 解除済み | 解除条件を満たし、解除記録が残っている。 |
| 取下げ | 誤判定、対象外、重複などにより取り下げた。 |
| 公開前再確認必要 | 公開前チェックで再確認が必要である。 |

## 13. 判定例

| 例 | 判定 | 理由 | 必要対応 |
|---|---|---|---|
| 2024年度の数値として記載したが、資料の対象年度が確認できない。 | `gap_id` | 年度不明であり、結論への影響確認が必要。 | 原資料または発行主体の資料で対象年度を確認する。 |
| 公開資料の概要版だけを根拠に制度説明を作成する。 | `warning_id` | 使用は可能だが、概要版であることの注記が必要。 | 取得日、資料種別、原資料未確認である旨を記録する。 |
| 引用予定文と原文の一致を確認できない。 | `stop_id` | 誤引用のまま公開されるおそれがある。 | 原文照合が完了するまで公開、PR、PDF化を停止する。 |
| 差分にREADME.mdの目的外変更が含まれている。 | `stop_id` | 変更対象外ファイルが混入している。 | 差分から除外し、PR前に再確認する。 |
| PDF原本をGitHubへ追加しようとしている。 | `stop_id` | 原本や非公開資料の混入リスクがある。 | 追加を中止し、必要なら公開URLまたはsource_idのみ記録する。 |
| 数値は確認済みだが速報値である。 | `warning_id` | 数値の制約を明示すれば使用できる。 | 速報値、確認日、更新可能性を注記する。 |
| source_idが未付与の根拠を補助的に参照している。 | `gap_id` | 根拠追跡ができない。 | source_idを付与し、資料情報を台帳へ記録する。 |

## 14. 禁止事項

1. secret、APIキー、個人情報、黒塗り前情報を台帳、Runbook、PR、成果物へ記載しない。
2. PDF原本、画像、非公開資料をGitHubへ追加しない。
3. 既存Runbook本文をコピーして混入させない。
4. README.mdを目的外に変更しない。
5. archiveを目的外に変更しない。
6. 変更対象外ファイルを差分に含めない。
7. 本文未確認、引用未確認、数値根拠未確認のまま重要断定をしない。
8. 法的評価、外部発信、名誉毀損リスクがある事項を、根拠確認未了のまま公開しない。
9. `stop_id` が停止中のまま、公開、PR、PDF化、外部発信、情報公開請求、法務官引渡を進めない。
10. 章番号の重複や壊れたMarkdown表を残したままPRを作成しない。

## 15. PR前確認条件

PR前に、次を確認する。

1. 変更ファイルは `05_tools/runbook_gap_warning_stop判定仕様_v0.1.md` の1個のみである。
2. 新規追加ファイルであり、差分は追加行のみである。
3. README.mdを変更していない。
4. archiveを変更していない。
5. PDFや画像を追加していない。
6. secretや個人情報を追加していない。
7. 章番号の重複がない。
8. Markdown表が壊れていない。
9. `gap_id` / `warning_id` / `stop_id` の定義、発行条件、判定表、台帳項目、source_id、case_id、公開前チェック、情報公開請求候補、再確認・解除・再開条件が含まれている。
10. `stop_id` に該当する条件が残っている場合は、PR、公開、PDF化、外部発信、情報公開請求、法務官引渡を進めない。
