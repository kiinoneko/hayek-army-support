# 補助金・委託契約確認Runbook v0.1

## 1. Runbookの位置づけ

- 補助金・委託契約確認Runbookは、公金支出の性質を分類し、根拠資料に基づき整理するための作業手順書である。
- 補助金、委託料、負担金、交付金、指定管理料は性質が異なるため、同じ欄で混同しない。
- 本Runbookは、決算カード作成Runbook、情報公開請求作成Runbook、案件台帳テンプレート、source_id台帳、gap_id台帳、公開前チェックリストと接続する。
- 公金支出の確認は、違法、不正、癒着の断定ではなく、資料、金額、契約、成果物、責任線を整理する作業である。
- source_idなしに金額を正式化しない。
- 契約書、仕様書、交付要綱、実績報告書、支払実績を混同しない。
- gap_idが残る場合は断定しない。
- stop_idがある場合は評価断定又は外部発信へ進めない。

## 4. 公金支出区分

| 区分 | 性質 | 主な根拠資料 | 注意 |
|---|---|---|---|
| 補助金 | 団体又は事業への財政支援 | 交付要綱、交付決定通知、申請書、実績報告書 | 成果物や返還条件を確認する |
| 委託料 | 自治体業務を外部へ委託する対価 | 契約書、仕様書、設計書、検査調書、支払実績 | 契約額と支払額を分ける |
| 負担金 | 団体、協議会、広域事業等への負担 | 負担金一覧、協定、規約、決算書 | 任意負担か制度上の負担か分ける |
| 交付金 | 特定目的で交付される財源 | 交付要綱、交付決定、実績報告 | 補助金との違いを確認する |
| 指定管理料 | 公の施設管理の対価 | 指定管理協定、仕様書、収支報告、モニタリング | 利用料収入との関係を確認する |
| 助成金 | 特定活動への支援 | 募集要項、交付決定、実績報告 | 補助金と同様に根拠確認する |
| 随意契約 | 競争入札によらない契約 | 随意契約理由書、契約書、見積書 | 理由と相手方選定根拠を確認する |

## 6. 公開資料確認手順

| 手順 | 内容 |
|---:|---|
| 1 | 自治体公式サイトで予算書、決算書、主要施策成果報告書を確認する |
| 2 | 補助金一覧、負担金一覧、契約一覧、随意契約一覧を確認する |
| 3 | 議会資料、委員会資料、議事録、会議録を確認する |
| 4 | 指定管理者制度資料、協定書、モニタリング結果を確認する |
| 5 | 社協・外郭団体の場合はWAM、事業報告、計算書類を確認する |
| 6 | 資料名、年度、公開元、URL、取得日を記録する |
| 7 | 本文確認できた資料のみsource_id正式化する |
| 8 | 不足資料をgap_id化する |

## 7. 補助金確認手順

| 確認事項 | 確認資料 |
|---|---|
| 補助金名 | 予算書、決算書、補助金一覧 |
| 交付先 | 交付決定通知、補助金一覧 |
| 交付額 | 交付決定通知、決算書、精算書 |
| 予算額 | 予算書 |
| 決算額 | 決算書 |
| 交付根拠 | 補助金交付要綱、条例、規則 |
| 申請内容 | 交付申請書、事業計画書 |
| 実績 | 実績報告書、収支報告書 |
| 精算 | 精算書、返還資料 |
| 成果 | 成果報告書、KPI、事業報告 |

- 補助金は契約ではない。
- 補助金額、交付決定額、精算額、決算額を混同しない。
- 補助金は交付目的、対象経費、実績報告、返還条件を確認する。
- 補助金だけで癒着、不正、無駄と断定しない。

## 8. 委託契約確認手順

| 確認事項 | 確認資料 |
|---|---|
| 委託業務名 | 契約書、仕様書、契約一覧 |
| 契約相手方 | 契約書、契約一覧 |
| 契約額 | 契約書、契約一覧 |
| 支払額 | 支出命令書、支払実績、決算資料 |
| 契約期間 | 契約書 |
| 業務内容 | 仕様書、設計書 |
| 選定方法 | 入札結果、随意契約理由書 |
| 成果物 | 成果報告書、納品物 |
| 検査 | 検査調書、完了確認書 |
| 再委託 | 契約書、仕様書、承認資料 |

- 委託料は自治体業務の対価であり、補助金と性質が異なる。
- 契約額、予定価格、支払実績、決算額を混同しない。
- 随意契約の場合は理由書と相手方選定根拠を確認する。
- 成果物、検査、完了確認が未確認の場合はgap_id化する。

## 9. 負担金・交付金・指定管理料確認手順

| 区分 | 確認事項 | 確認資料 |
|---|---|---|
| 負担金 | 負担先、負担根拠、負担額、算定根拠 | 負担金一覧、規約、協定、決算書 |
| 交付金 | 交付目的、対象事業、交付額、実績 | 交付要綱、交付決定、実績報告 |
| 指定管理料 | 指定管理者、施設名、協定額、収支、利用料 | 基本協定、年度協定、収支報告、モニタリング |
| 助成金 | 助成対象、助成額、実績 | 募集要項、交付決定、実績報告 |

- 指定管理料は施設管理の対価であり、補助金とは区別する。
- 利用料金制の場合は利用料収入と指定管理料を分ける。
- 負担金は制度的負担か任意負担か確認する。
- 交付金は制度根拠と使途を確認する。

## 10. 金額照合手順

| 照合項目 | 確認内容 |
|---|---|
| 予算額と決算額 | 差額、増減理由 |
| 交付決定額と精算額 | 返還、減額、未執行 |
| 契約額と支払額 | 変更契約、出来高、未払い |
| 契約一覧と決算書 | 対象科目、年度一致 |
| 補助金一覧と決算書 | 款項目、事業名一致 |
| 自治体資料と団体資料 | 収入側・支出側の一致 |
| 単位 | 円、千円、百万円の混在 |
| 年度 | 予算年度、契約年度、支払年度の一致 |

- 差額がある場合は、誤りと断定せず理由分類を行う。
- 変更契約、精算、繰越、未執行、返還、会計区分差を確認する。
- 金額の分母・分子・年度・単位を明記する。

## 11. 成果物・実績報告確認手順

| 確認事項 | 文書候補 |
|---|---|
| 事業実施内容 | 実績報告書、事業報告書 |
| 成果物 | 成果報告書、納品物、報告書 |
| 完了確認 | 検査調書、完了届、完了報告 |
| KPI | 事業評価書、成果指標、主要施策成果報告書 |
| 収支 | 収支報告書、精算書 |
| 返還 | 返還命令、精算通知 |

- 成果物が未確認の場合、成果なしと断定しない。
- 実績報告書が未確認の場合はgap_id化する。
- 成果指標が不明な場合はpolicy_idの未確認事項として管理する。

## 12. actor_id / policy_id との接続

| ID | 接続対象 |
|---|---|
| `actor_id` | 首長、担当部署長、議員、監査委員、審議会委員、同意人事、団体代表者 |
| `policy_id` | 補助金、委託事業、指定管理、負担金、交付金、関連事業 |
| `quote_id` | 議会発言、答弁、SNS、会見、説明資料中の発言 |
| `source_id` | 根拠資料 |
| `case_id` | 案件全体 |

- 政治行政アクターの個人名は、公的職務に関係する範囲で明記する。
- actor_idは責任線を追跡するための索引であり、個人責任や違法性の断定根拠ではない。
- policy_idは政策・事業を接続する索引であり、効果、失敗、不正を単独で断定する根拠ではない。
- 議会同意人事、監査委員、審議会委員等は、公的影響を持つ場合に記録対象とする。

## 13. gap_id / warning_id / stop_id 判定

| 状態 | 発行ID | 対応 |
|---|---|---|
| 補助金額が未確認 | `GAP_` | 補助金一覧、交付決定、決算資料を確認 |
| 委託契約額が未確認 | `GAP_` | 契約書、契約一覧、支払実績を確認 |
| 成果物未確認 | `GAP_` | 実績報告書、成果報告書を確認 |
| 補助金と委託料が混在 | `WARN_` | 区分して表示 |
| 予算額と決算額が混在 | `WARN_` | 年度と金額種別を分ける |
| 契約額と支払額が混在 | `WARN_` | 契約額、支払額、決算額を分ける |
| source_idなし | `STOP_` | 正式化停止 |
| 本文未確認 | `STOP_` | 本文確認まで停止 |
| 法的断定が必要 | `STOP_` | 法務官確認 |
| 外部発信予定 | `DEC_` | USER_DECISION |

## 14. 情報公開請求への接続

| gap内容 | request_id候補 |
|---|---|
| 補助金交付決定額が不明 | 交付決定通知、交付申請書、実績報告書 |
| 補助金精算額が不明 | 精算書、返還資料 |
| 委託契約額が不明 | 委託契約書、契約一覧 |
| 仕様書が不明 | 業務仕様書、設計書 |
| 成果物が不明 | 成果報告書、納品物、検査調書 |
| 随意契約理由が不明 | 随意契約理由書、選定理由書 |
| 支払実績が不明 | 支出命令書、支払調書 |
| 指定管理の収支が不明 | 収支報告書、年度協定、モニタリング |

- request_id化は提出済みを意味しない。
- 情報公開請求は公開資料確認後の補助手段である。
- 提出判断はdecision_idを必要とする。
- 請求文案作成前に請求対象文書名を具体化する。

## 16. 公開前チェック

- 公開前チェックリストを使用する。
- PDF原本をGitHubに置かない。
- 個人情報を含めない。
- 黒塗り前資料を含めない。
- secretを含めない。
- source_idを明記する。
- gap_idを隠さない。
- warning_idを反映する。
- stop_idがある場合は公開しない。
- decision_idが必要な場合は取得する。
- 違法、不正、癒着、汚職を根拠なく断定しない。

## 17. 禁止事項

- 補助金と委託料を混同しない。
- 契約額、支払額、決算額を混同しない。
- 交付決定額、精算額、返還額を混同しない。
- 予算額と決算額を混同しない。
- source_idなしに金額を正式化しない。
- PDF存在確認だけで本文確認済みにしない。
- 成果物未確認のまま成果なしと断定しない。
- 随意契約というだけで不正と断定しない。
- 補助金交付というだけで癒着と断定しない。
- actor_idだけで個人責任、故意、違法、不正を断定しない。
- policy_idだけで政策の失敗、不正、癒着を断定しない。
- gap_idを隠して評価断定しない。
- warning_idを無視して外部発信しない。
- stop_idがある状態で公開しない。
- decision_idなしに情報公開請求、外部発信、法務官引渡を進めない。
- PDF原本、個人情報、黒塗り前資料、APIキーをGitHubに追加しない。
