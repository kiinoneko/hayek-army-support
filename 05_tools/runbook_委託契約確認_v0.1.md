# Runbook: 委託契約確認 v0.1

## 1. 位置づけ

このRunbookは、後方支援軍が国、都道府県、市町村、外郭団体、社会福祉協議会、観光協会、商工会、第三セクター等に関係する委託契約を確認するための標準手順である。

確認対象は、契約目的、仕様書、契約金額、支払額、契約期間、契約方法、競争性、随意契約理由、成果物、検収、再委託、実施報告、費用対効果、直営・民間・廃止との比較に限定する。

委託料は行政事務、公共サービス、調査、広報、施設管理、相談業務等の対価であり、補助金、指定管理、入札制度全体の詳細確認とは分離する。

## 2. 対象範囲

対象に含めるものは次のとおりである。

- 国の委託契約
- 都道府県の委託契約
- 市町村の委託契約
- 外郭団体・第三セクターに関係する委託契約
- 社会福祉協議会、観光協会、商工会等への委託契約
- 調査委託
- 事業運営委託
- 広報・PR委託
- システム・DX委託
- 施設管理関連委託
- イベント運営委託
- 相談業務委託
- 再委託を含む委託契約

対象外とするものは次のとおりである。

- 補助金の適否、補助率、補助対象経費の詳細確認
- 指定管理制度の選定、協定、利用料金、モニタリングの詳細確認
- 入札制度全体、調達制度全体、例規体系全体の詳細分析
- 個別職員、個別事業者、個人への根拠なき非難
- 契約資料未確認の状態での不正、癒着、違法、利益供与の断定

## 3. 委託契約の基本区分

委託契約は、少なくとも次の区分で整理する。

| 区分 | 確認内容 | 注意点 |
| --- | --- | --- |
| 調査委託 | 調査目的、調査手法、報告書、データ、分析結果 | 成果物の有無だけでなく、仕様との対応を確認する。 |
| 事業運営委託 | 運営内容、対象者、実施回数、運営体制、報告方法 | 委託先の自主事業や補助事業と混同しない。 |
| 広報・PR委託 | 媒体、掲載期間、制作物、配信実績、効果測定 | 露出量、成果指標、納品物を分けて確認する。 |
| システム・DX委託 | 要件、設計、構築、保守、ライセンス、運用 | 初期構築費、保守費、追加変更費を分ける。 |
| 施設管理関連委託 | 清掃、警備、設備管理、保守点検、受付 | 指定管理や工事請負と混同しない。 |
| イベント運営委託 | 企画、会場、出演、広報、安全管理、実績報告 | 来場者数、支出、再委託の範囲を確認する。 |
| 相談業務委託 | 相談員体制、対応件数、記録、報告、守秘 | 個人情報をGitHubへ追加しない。 |
| 再委託を含む委託 | 再委託範囲、承認、責任分担、成果物 | 再委託があるだけで丸投げと断定しない。 |

## 4. 確認する資料

確認する資料は、原則として資料名、作成者、作成日、対象年度、公開URLまたは情報公開請求候補、source_idを記録する。

| 資料 | 主な確認点 |
| --- | --- |
| 契約書 | 契約相手方、契約金額、契約期間、業務内容、支払条件、再委託条項 |
| 仕様書 | 業務範囲、成果物、実施体制、納期、報告義務、検収条件 |
| 設計書 | 積算根拠、数量、単価、工数、仕様との対応 |
| 予定価格資料 | 予定価格、算定根拠、見積比較、設計額との関係 |
| 見積書 | 見積者、見積額、内訳、提出日、仕様との対応 |
| 契約結果一覧 | 契約名、契約相手方、契約金額、契約方法、公表日 |
| 入札結果または選定結果 | 参加者、入札額、落札者、評価結果、不調・辞退の有無 |
| 随意契約理由書 | 根拠条項、理由、唯一性、緊急性、継続性、価格妥当性 |
| プロポーザル実施要領 | 参加資格、提案条件、審査手順、スケジュール |
| 評価基準 | 配点、評価項目、価格点、技術点、失格条件 |
| 評価結果 | 評価者、点数、順位、選定理由、講評 |
| 業務計画書 | 実施体制、工程、役割分担、リスク対応 |
| 業務報告書 | 実施内容、実施日、件数、課題、対応状況 |
| 成果報告書 | 成果、指標、達成状況、分析、今後の課題 |
| 納品物一覧 | 納品物名、形式、数量、納品日、仕様との対応 |
| 完了届 | 完了日、対象業務、提出者、契約期間との整合 |
| 検査調書 | 検査日、検査者、合否、指摘事項、是正状況 |
| 支払実績 | 支払日、支払額、支払先、支払根拠、分割払いの有無 |
| 変更契約資料 | 変更理由、変更額、変更期間、変更仕様、決裁資料 |
| 再委託承認資料 | 再委託先、再委託範囲、承認日、責任分担 |
| 予算書 | 予算額、款項目、事業名、財源、当初・補正の区分 |
| 決算書 | 決算額、執行額、不用額、翌年度繰越の有無 |
| 主要施策成果報告書 | 事業目的、実績、成果指標、決算説明 |
| 議会資料 | 議案、委員会資料、質疑、説明資料、議決日 |

## 5. 契約目的の確認観点

- 契約目的が予算書、仕様書、契約書、主要施策成果報告書で一貫しているか。
- 委託により達成する行政目的、対象者、対象区域、対象期間が明記されているか。
- 目的が抽象的すぎて、成果物、実施内容、効果測定と接続できない状態になっていないか。
- 既存事業の継続、緊急対応、新規政策、法令対応のいずれに近いかを区分する。
- 委託先の団体目的や自主事業の目的と、発注者の契約目的を混同しない。

## 6. 仕様書の確認観点

- 業務範囲、数量、実施回数、成果物、納期、報告義務、検収条件が明記されているか。
- 仕様が契約金額、見積内訳、業務計画書、成果報告書と対応しているか。
- 成果物の形式、提出方法、著作権、データ利用、保存期間が確認できるか。
- 仕様変更がある場合、変更契約資料、協議記録、変更後仕様書が存在するか。
- 仕様が曖昧な場合は、直ちに不正とせず、gap_idまたは情報公開請求候補として扱う。

## 7. 契約金額・支払額・変更契約の確認観点

- 契約額、予定価格、見積額、変更契約額、支払額、決算額を分けて記録する。
- 契約金額と支払額が一致しない場合、前払、概算払、精算、減額、変更契約、不履行部分の有無を確認する。
- 変更契約がある場合、変更理由、変更時期、変更額、変更後仕様、決裁資料を確認する。
- 決算額は事業全体の金額である場合があるため、個別契約の支払額と混同しない。
- 金額の大小だけで不適切と断定せず、仕様、成果物、期間、競争性、積算根拠と合わせて評価する。

## 8. 契約方法・競争性の確認観点

- 一般競争入札、指名競争入札、随意契約、プロポーザル、企画提案、見積合わせ等の契約方法を確認する。
- 参加者数、辞退者数、無効入札、1者応札、1者見積、不調、再公告の有無を確認する。
- 参加資格、地域要件、実績要件、仕様条件が過度に限定的でないかを確認する。
- プロポーザルの場合、実施要領、評価基準、評価結果、審査体制を確認する。
- 1者応札・1者見積だけで直ちに不正と断定せず、公告期間、仕様、参加資格、見積依頼先、過去実績を確認する。

## 9. 随意契約理由の確認観点

- 随意契約理由書、根拠条項、決裁資料、見積書、価格妥当性資料を確認する。
- 唯一性、緊急性、継続性、専門性、少額、契約不調後の扱いなど、理由の種類を区分する。
- 理由が仕様、契約目的、過去実績、成果物、契約期間と整合しているか確認する。
- 随意契約というだけで不正、癒着、違法、利益供与と断定しない。
- 理由書が未確認の場合は、重要評価を保留し、情報公開請求候補に接続する。

## 10. 成果物・納品物の確認観点

- 成果物、納品物一覧、成果報告書、業務報告書、完了届、検査調書を相互に照合する。
- 仕様書に定められた成果物が、形式、数量、提出日、内容の面で納品されているか確認する。
- 公開可能な成果物と、個人情報、機密情報、非公開情報を含む成果物を分ける。
- 成果物未確認のまま成果なしと断定しない。
- 成果物の質に関する評価は、事実、推定、評価、未確認事項を分離して記録する。

## 11. 検収・完了確認・支払確認の確認観点

- 完了届、検査調書、検収記録、支払命令、支払実績を確認する。
- 検査日、検査者、指摘事項、是正対応、支払日が契約期間と整合するか確認する。
- 検収前支払、概算払、前払がある場合は、根拠と精算状況を確認する。
- 完了確認が形式的に見える場合でも、資料未確認のまま不正と断定しない。
- 検収資料が未確認の場合は、warning_idまたは情報公開請求候補に接続する。

## 12. 再委託の確認観点

- 契約書、仕様書、再委託承認資料、業務計画書、成果報告書で再委託の有無を確認する。
- 再委託先、再委託範囲、金額、承認日、責任分担、成果物の帰属を確認する。
- 主要部分の再委託が禁止または制限されている場合、その範囲と実態を確認する。
- 再委託があるだけで丸投げと断定しない。
- 承認資料がない場合は、契約条項、仕様、実施体制との照合を行い、必要に応じてgap_idを発行する。

## 13. 実施報告・成果報告の確認観点

- 業務計画書、業務報告書、成果報告書、主要施策成果報告書、議会資料を照合する。
- 実施日、実施回数、参加者数、相談件数、処理件数、配信数、閲覧数等の実績を確認する。
- 実績値と成果指標、契約目的、仕様書の関係を確認する。
- 委託先の自己評価、発注者の評価、議会説明、公開資料の記述を分ける。
- 要約資料だけで重要断定を行わず、一次資料または情報公開請求候補に接続する。

## 14. 費用対効果の確認観点

- 契約金額、支払額、成果物、実績値、対象者数、期間、類似契約との比較を確認する。
- 単価、件数、対象者1人当たり費用、成果物1件当たり費用など、可能な範囲で補助指標を作る。
- 効果が定量化しにくい業務では、仕様達成、必要性、代替手段、継続理由を分けて記録する。
- 費用対効果が低い可能性は評価として扱い、事実と推定を混同しない。
- 成果物、検収、実施報告が未確認の場合は、費用対効果の重要評価を保留する。

## 15. 直営・民間・廃止との比較観点

- 直営で実施した場合の人員、専門性、設備、責任、継続性を確認する。
- 民間サービス、既存制度、共同調達、広域連携で代替可能かを確認する。
- 廃止、縮小、統合、内製化、仕様見直しの選択肢を分けて記録する。
- 委託が妥当かどうかは、契約目的、仕様、成果物、費用、リスク、行政責任を合わせて評価する。
- 比較資料がない場合は、断定せず情報公開請求候補または今後の確認事項に接続する。

## 16. 予算・決算・契約資料との照合

- 予算書、補正予算、決算書、主要施策成果報告書、契約結果一覧、支払実績を照合する。
- 予算額、契約額、変更契約額、支払額、決算額、不用額を別項目で記録する。
- 事業名、款項目、契約名、支出科目が一致しない場合は、対応関係を確認する。
- 複数契約が同一事業に含まれる場合、個別契約と事業全体を分ける。
- 金額差異は直ちに不正とせず、予算残、入札差金、変更契約、精算、翌年度繰越を確認する。

## 17. source_idとの接続

source_idは、資料単位で発行する。契約書、仕様書、契約結果一覧、随意契約理由書、成果報告書、検査調書、支払実績などを同一source_idにまとめない。

記録項目は次のとおりである。

- source_id
- 資料名
- 作成者または公表者
- 作成日または公表日
- 対象年度
- URLまたは取得方法
- 公開資料、情報公開請求資料、未取得資料の区分
- 個人情報、黒塗り、非公開部分、secret混入疑いの有無

## 18. case_idとの接続

case_idは、確認対象となる契約または契約群単位で発行する。

- 単一契約を確認する場合は、契約名、年度、発注者、受託者をcase_idに接続する。
- 同一事業の複数契約を確認する場合は、親case_idと子case_idに分ける。
- 補助金、指定管理、入札制度全体の詳細分析が必要になった場合は、別Runbookまたは別case_idに分離する。
- 事実、推定、評価、未確認事項、情報公開請求候補をcase_id内で分離する。

## 19. gap_id / warning_id / stop_id の発行条件

### 19.1 gap_id

次の場合はgap_idを発行する。

- 契約書、仕様書、成果物、検収資料のいずれかが不足している。
- 契約額、支払額、変更契約額、決算額の対応関係が未整理である。
- 随意契約理由書、評価結果、再委託承認資料が未取得である。
- 仕様書と成果報告書の対応が確認できない。
- 予算書、決算書、契約結果一覧の事業名または金額が一致しない。

### 19.2 warning_id

次の場合はwarning_idを発行する。

- 1者応札、1者見積、同一受託者の継続契約など、競争性の追加確認が必要である。
- 随意契約理由が抽象的で、唯一性、緊急性、継続性、価格妥当性の確認が必要である。
- 成果物、実施報告、検収資料に不足または不整合がある。
- 再委託の範囲、承認、責任分担が不明確である。
- 要約資料と一次資料の内容に差異がある。

### 19.3 stop_id

次の場合はstop_idを発行し、公開、断定、PR前の確定評価を停止する。

- 契約書、仕様書、成果物、検収資料が未確認のまま不正・癒着・違法と断定しようとしている。
- 契約額、支払額、変更契約額、決算額の区分が未確認のまま重要評価を行う。
- 随意契約理由が未確認のまま重要評価を行う。
- 成果物、納品物、検収記録が未確認のまま成果なしと断定しようとしている。
- 個人情報、黒塗り前資料、非公開資料、secretの混入疑いがある。
- PDF原本や画像をGitHubへ追加しようとしている。
- 要約資料だけで重要断定を行おうとしている。
- 旧本文と新本文が混在している。
- 変更対象外ファイルが差分に入っている。
- README.md、archive、04_templatesが目的外に変更されている。

## 20. 情報公開請求候補への接続

未取得資料がある場合は、次のように情報公開請求候補へ接続する。

| 未取得資料 | 請求候補の例 |
| --- | --- |
| 契約書 | 対象年度、契約名、受託者を指定して契約書一式を請求する。 |
| 仕様書 | 契約書に添付または別紙となる仕様書、特記仕様書を請求する。 |
| 随意契約理由書 | 根拠条項、理由書、決裁資料、価格妥当性資料を請求する。 |
| 評価結果 | プロポーザル評価基準、評価表、審査結果、講評を請求する。 |
| 成果物 | 納品物一覧、成果報告書、公開可能な成果物を請求する。 |
| 検収資料 | 完了届、検査調書、検収記録、是正記録を請求する。 |
| 支払実績 | 支払日、支払額、支払命令、精算資料を請求する。 |
| 再委託承認資料 | 再委託申請、承認書、再委託範囲、再委託先を請求する。 |

情報公開請求候補には、個人情報、法人秘密、非公開情報が含まれる可能性を注記し、GitHubへ原本PDF、画像、黒塗り前資料、secretを追加しない。

## 21. 公開前チェックへの引き渡し

公開前チェックへ引き渡す前に、次を確認する。

- 事実、推定、評価、未確認事項、情報公開請求候補が分離されている。
- 不正、癒着、違法、利益供与などの強い表現に、十分な資料根拠がある。
- 契約額、支払額、変更契約額、決算額が混同されていない。
- 随意契約、1者応札、再委託だけを根拠に断定していない。
- 個人情報、PDF原本、画像、secret、APIキー、非公開資料が含まれていない。
- source_id、case_id、gap_id、warning_id、stop_idの接続が確認できる。

## 22. 禁止事項

- 既存Runbook本文をコピーして混入させない。
- 過去の壊れ差分を再利用しない。
- 旧本文と新本文を併記しない。
- README.md、archive、04_templates、他の05_toolsファイルを変更しない。
- PDF原本、画像、個人情報、secret、APIキーを追加しない。
- 補助金、指定管理、入札制度全体の詳細確認をこのRunbookで扱わない。
- 随意契約というだけで不正、癒着、違法、利益供与と断定しない。
- 1者応札・1者見積だけで直ちに不正と断定しない。
- 成果物未確認のまま成果なしと断定しない。
- 再委託があるだけで丸投げと断定しない。
- 要約資料だけで重要断定を行わない。
