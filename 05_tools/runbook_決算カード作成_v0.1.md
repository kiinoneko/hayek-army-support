# 決算カード作成Runbook v0.1

## 1. Runbookの位置づけ

決算カード作成Runbookは、後方支援軍において決算カードを実際に作るための作業手順書である。

- 決算カード仕様は「カードに載せる項目」を定める。
- 本Runbookは「資料確認、数値確認、未確認整理、カード化、公開前確認」の手順を定める。
- 決算カードは評価断定ではなく、根拠資料に基づく可視化資料である。
- `source_id`なしに数値を正式化しない。
- `gap_id`が残る場合は断定しない。
- `warning_id`がある場合はカード内に注意書きを反映する。
- `stop_id`がある場合はカード化又は公開を停止する。

## 2. 対象

本Runbookは、次の資料を用いる決算カード作成を対象とする。

- 市町村決算カード
- 財政状況資料
- 予算書
- 決算書
- 主要施策成果報告書
- 健全化判断比率資料
- 将来負担比率資料
- 社会福祉協議会決算書
- WAM現況報告書
- 事業報告書
- 補助金交付資料
- 委託契約資料
- 指定管理資料
- 外郭団体決算資料

## 3. 使用する主なID

| ID | 用途 |
|---|---|
| `case_id` | 決算カード作成案件を管理する |
| `source_id` | 根拠資料を管理する |
| `gap_id` | 未確認事項を管理する |
| `warning_id` | 注意事項を管理する |
| `stop_id` | 自動進行停止を管理する |
| `policy_id` | 関連政策・事業を管理する |
| `request_id` | 情報公開請求候補を管理する |
| `decision_id` | ユーザー判断を管理する |
| `change_id` | カード修正履歴を管理する |
| `incident_id` | 誤公開・誤記等を管理する |

| 2 | 議会資料、主要施策成果報告書、決算カードを確認する |

## 6. source_id化手順

| 手順 | 内容 |
|---|---|
| 1 | 資料名を確認する |
| 2 | 公開元を確認する |
| 3 | 年度・対象期間を確認する |
| 4 | URL又は保存先を確認する |
| 5 | 取得日を記録する |
| 6 | 該当ページ・該当欄を確認する |
| 7 | 信頼度と内容確度を記録する |

- PDF存在確認と本文確認を混同しない。
- 本文未確認資料は内容確度C以下とする。
- 内容確度C以下の資料で数値を正式化しない。

## 7. 数値抽出手順

|---|---|
| 歳入総額 | 決算書、決算カード |
| 歳出総額 | 決算書、決算カード |
| 人件費 | 決算書、性質別歳出 |
| 扶助費 | 決算書、性質別歳出 |
| 公債費 | 決算書、性質別歳出 |
| 普通建設事業費 | 決算書、性質別歳出 |
| 地方債残高 | 決算カード、財政状況資料 |
| 基金残高 | 決算書、財政状況資料 |
| 経常収支比率 | 決算カード |
| 実質公債費比率 | 健全化判断比率 |
| 将来負担比率 | 健全化判断比率 |
| 社協サービス活動収益 | WAM計算書類 |
| 社協人件費 | WAM計算書類 |
| 補助金収益 | WAM計算書類、自治体決算資料 |
| 受託金収益 | WAM計算書類、委託契約資料 |

差額がある場合は、単位差、会計範囲差、年度差、予算・決算差、契約額・支払額差、資料改定差、転記誤り疑いに分類し、必要に応じて`gap_id`又は`warning_id`を発行する。

## 9. 指標算出手順

| 指標 | 算式 | 注意 |
|---|---|---|
| 人件費率 | 人件費 ÷ 歳出総額又はサービス活動収益 | 分母を明記する |
| 補助金比率 | 補助金額 ÷ 対象収益又は歳出 | 町単独か公的財源全体か区別する |
| 委託料比率 | 委託料 ÷ 対象収益又は歳出 | 契約額と支払額を混同しない |
| 補助金＋委託料比率 | 補助金＋委託料 ÷ 対象収益 | 合算理由を明記する |
| 基金比率 | 基金残高 ÷ 標準財政規模等 | 分母を明記する |
| 地方債残高比率 | 地方債残高 ÷ 歳入等 | 比較条件を明記する |
| 経常収支比率 | 公式値を使用 | 独自算出しない |
| 実質公債費比率 | 公式値を使用 | 独自算出しない |
| 将来負担比率 | 公式値を使用 | 独自算出しない |

指標を算出する場合は、分母、分子、単位、年度、会計範囲、算式、使用した`source_id`を記録する。公式値を優先すべき指標は独自算出しない。

## 10. gap_id / warning_id / stop_id 判定

| 状態 | 発行ID | 対応 |
|---|---|---|
| 資料が見つからない | `GAP_` | 資料探索又はrequest_id候補 |
| 本文未確認 | `GAP_` 又は `STOP_` | 正式化停止 |
| 数値の根拠がない | `STOP_` | カード化停止 |
| 分母・分子が不明 | `WARN_` 又は `STOP_` | 指標算出停止又は注意書き |
| 補助金と委託料が混在 | `WARN_` | 区分して表示 |
| 予算と決算が混在 | `WARN_` | 表示を分ける |
| 法的断定が必要 | `STOP_` | 法務官確認 |
| 外部公開予定 | `DEC_` | USER_DECISION |

- `gap_id`は、未確認事項を見える化するために発行する。
- `warning_id`は、カード化は可能だが読み手に注意喚起すべき事項に発行する。
- `stop_id`は、カード化又は公開を停止すべき重大な欠落、未確認、法務確認事項に発行する。
- 外部公開予定がある場合は、公開可否について`decision_id`を取得する。

## 11. QRコード・原文URLの扱い

- 決算カードに原文URL又はQRコードを添えることは有効である。
- QRコードは公開可能URLに限定する。
- secret入りURL、private共有URL、署名付きURL、有効期限付きURLはQRコード化しない。
- QRコードは根拠への導線であり、根拠そのものではない。
- QRコード先が差し替わる可能性がある場合は取得日を明記する。
- QRコードには対応する`source_id`を併記する。

## 13. 社協・外郭団体向け追加確認

- WAM現況報告書を確認する。
- 計算書類を確認する。
- 事業活動計算書と資金収支計算書を混同しない。
- サービス活動収益と事業活動収入を混同しない。
- 人件費率の分母を明記する。
- 常勤、非常勤、FTE、常勤換算を混同しない。
- 補助金収益、受託金収益、介護保険事業収益、障害福祉サービス等事業収益を区分する。
- 自治体側の補助金・委託料資料と照合する。
- 町依存度を出す場合は、町補助金、町委託料、国県補助金、制度収入を分ける。

## 14. 自治体決算カード向け追加確認

- 決算カードの公式値を優先する。
- 普通会計と一般会計を混同しない。
- 歳入歳出総額、標準財政規模、地方債残高、基金残高を確認する。
- 経常収支比率、実質公債費比率、将来負担比率は公式値を使う。
- 一時的収入、臨時財源、大規模事業の影響を`warning_id`化する。
- 他自治体比較では、人口規模、会計範囲、年度、制度差を確認する。

## 15. 情報公開請求への接続

| gap内容 | request_id候補 |
|---|---|
| 補助金交付決定額が不明 | 補助金交付決定通知、実績報告 |
| 委託契約額が不明 | 委託契約書、仕様書、支払実績 |
| 成果物が不明 | 事業報告書、実績報告書 |
| 職員数・FTEが不明 | 現況報告書、職員配置資料 |
| 町負担額が不明 | 補助金一覧、決算資料、契約一覧 |
| 指標の算出根拠が不明 | 算出資料、説明資料 |

- 情報公開請求は情報収集の開始点ではない。
- 先に公開資料を確認する。
- `request_id`化にはUSER_DECISIONが必要な場合がある。
- 提出前に公開前チェックリストを通す。

## 16. 公開前チェック

- 公開前チェックリストを使用する。
- PDF原本をGitHubに置かない。
- 個人情報を含めない。
- 黒塗り前資料を含めない。
- secretを含めない。
- `source_id`を明記する。
- `gap_id`を隠さない。
- `warning_id`を反映する。
- `stop_id`がある場合は公開しない。
- `decision_id`が必要な場合は取得する。

## 18. 禁止事項

- `source_id`なしに数値を正式化しない。
- PDF存在確認だけで本文確認済みにしない。
- 予算額、決算額、契約額、支払実績を混同しない。
- 補助金、委託料、負担金、交付金、制度収入を混同しない。
- 人件費率の分母を省略しない。
- FTE未確認で職員1人当たり指標を出さない。
- `gap_id`を隠して断定しない。
- `warning_id`を無視してカード化しない。
- `stop_id`がある状態で公開しない。
- 原文未確認の発言を`quote_id`化しない。
- QRコードにsecret入りURLを使わない。
- PDF原本、個人情報、黒塗り前資料、APIキーをGitHubに追加しない。
- AI判断だけで外部公開しない。
