# ハイエク軍 後方支援軍

このリポジトリは、公開資料等を収集し、検証可能な台帳・作業資料・上申書へ変換して、ハイエク軍およびユーザーの判断を支援する後方支援用の作業領域です。Codexは資料整理、台帳生成、比較、論点整理、上申書作成の補助に限定し、最終判断は行いません。

## フォルダの役割

| フォルダ | 役割 | 主な受け渡し先 |
|---|---|---|
| [`00_index`](00_index/README.md) | 案件台帳、資料索引、進捗、成果物間の対応関係を管理する | 全フォルダおよびハイエク軍 |
| [`01_sources`](01_sources/README.md) | 原資料と取得記録を、内容を改変せず保存する | `02_processed` |
| [`02_processed`](02_processed/README.md) | 原資料から抽出・整形した検証可能な作業資料を保存する | `03_reports` |
| [`03_reports`](03_reports/README.md) | 判断材料となる報告書・上申書の草案と確定版を保存する | ハイエク軍およびユーザー |
| [`04_templates`](04_templates/README.md) | 台帳、分析、報告、上申に使う標準様式を管理する | `00_index`、`02_processed`、`03_reports` |
| [`05_tools`](05_tools/README.md) | 収集、抽出、整形、検証を補助する再現可能なツールを管理する | `01_sources`、`02_processed`、`00_index` |
| [`06_doctrine`](06_doctrine/README.md) | 後方支援軍の情報加工線、責任境界、品質基準、運用原則を管理する | 全フォルダおよびハイエク軍 |

## 正式運用文書

- 後方支援軍の正式指示プロンプトは、[`00_index/後方支援軍_指示プロンプト.md`](00_index/後方支援軍_指示プロンプト.md)に保存する。
- 後方支援軍の基本ドクトリンは、[`06_doctrine/後方支援軍_基本ドクトリン.md`](06_doctrine/後方支援軍_基本ドクトリン.md)に保存する。
- 後方支援軍は、原資料を整理し、台帳化し、資料化し、ハイエク軍へ上申可能な状態へ整える情報兵站軍である。
- 後方支援軍は最終判断を行わず、最終判断はハイエク軍およびユーザーに留保する。

## 標準作業フロー

1. `00_index`で案件ID、対象、担当、状態を登録する。
2. `01_sources`に原資料と取得元、取得日、対象期間を保存する。
3. `02_processed`で事実、引用、出典、推定、評価、未確認事項、情報公開請求候補を分離して整理する。
4. `04_templates`の様式を使い、`03_reports`に報告書または上申書を作成する。
5. `00_index`から原資料、作業資料、報告書を追跡できる状態でハイエク軍へ接続する。

## 共通運用原則

- 補助金、助成、給付、減免、委託、指定管理、外郭団体支援、公費投入、施設維持費、行政サービス拡充は、名称や会計区分にかかわらず、すべて納税者負担として整理する。
- 事実、引用、出典、推定、評価、未確認事項、情報公開請求候補は、見出し、列、ファイル等で明確に分離する。
- 推定は推定と明記し、根拠、計算方法、仮定、不確実性を添えて断定しない。
- Codexは最終判断をせず、資料整理、台帳生成、比較、論点提示、上申書作成の補助だけを行う。
- 違法性、妥当性、政治的評価、制度設計、案件の採否に関する最終判断は、ハイエク軍およびユーザーに留保する。
- 成果物は必ず原資料と出典へ遡れるようにし、不明点を推測で補完しない。

各フォルダ固有の保存対象と作業範囲は、それぞれのREADMEに従ってください。
