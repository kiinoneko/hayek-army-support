# ハイエク軍 後方支援軍

このリポジトリは、公開資料等を収集し、検証可能な台帳・作業資料・上申書へ変換して、ハイエク軍およびユーザーの判断を支援する後方支援用の作業領域です。Codexは資料整理、台帳生成、比較、論点整理、上申書作成の補助に限定し、最終判断は行いません。

## 正式運用文書

- 後方支援軍の正式指示プロンプトは、[`00_index/後方支援軍_指示プロンプト.md`](00_index/後方支援軍_指示プロンプト.md)に保存する。
- 後方支援軍の基本ドクトリンは、[`06_doctrine/後方支援軍_基本ドクトリン.md`](06_doctrine/後方支援軍_基本ドクトリン.md)に保存する。
- 後方支援軍は、原資料を整理し、台帳化し、資料化し、ハイエク軍へ上申可能な状態へ整える情報兵站軍である。
- 後方支援軍は最終判断を行わず、最終判断はハイエク軍およびユーザーに留保する。

## 標準作業フロー

後方支援軍の通常案件は、次の順序を標準フローとする。前段のIDが未整備のまま後段へ進まない。

1. "case_id"を起票し、"00_index"で案件ID、対象、担当、状態を登録する。
2. "source_id"を正式化し、"01_sources"に原資料と取得元、取得日、対象期間、本文確認状態を保存する。
3. "gap_id" / "warning_id" / "stop_id"を判定し、未確認事項、注意事項、停止条件を分離して記録する。
4. "02_processed"で事実、引用、出典、推定、評価、未確認事項、情報公開請求候補を分離して分析する。
5. "04_templates"の様式を使い、"03_reports"に報告書または上申書を作成する。
6. 公開、PR、PDF化、情報公開請求、法務官引渡の前に公開前チェックを実行する。
7. "00_index"から原資料、作業資料、報告書、判断記録を追跡できる状態でハイエク軍へ接続する。

簡略表記では、"case_id 起票 → source_id 正式化 → gap_id / warning_id / stop_id 判定 → 分析 → 公開前チェック" とする。

## 共通運用原則

- 補助金、助成、給付、減免、委託、指定管理、外郭団体支援、公費投入、施設維持費、行政サービス拡充は、名称や会計区分にかかわらず、すべて納税者負担として整理する。
- 事実、引用、出典、推定、評価、未確認事項、情報公開請求候補は、見出し、列、ファイル等で明確に分離する。
- 推定は推定と明記し、根拠、計算方法、仮定、不確実性を添えて断定しない。
- Codexは最終判断をせず、資料整理、台帳生成、比較、論点提示、上申書作成の補助だけを行う。
- 違法性、妥当性、政治的評価、制度設計、案件の採否に関する最終判断は、ハイエク軍およびユーザーに留保する。
- 成果物は必ず原資料と出典へ遡れるようにし、不明点を推測で補完しない。

各フォルダ固有の保存対象と作業範囲は、それぞれのREADMEに従ってください。

## 石田三成：後方支援軍総大将

石田三成は、後方支援軍における総大将／兵站行政総奉行である。

関連ファイル：

- [`00_index/石田三成_総大将指示プロンプト.md`](00_index/石田三成_総大将指示プロンプト.md)
- [`06_doctrine/石田三成_兵站行政ドクトリン.md`](06_doctrine/石田三成_兵站行政ドクトリン.md)

石田三成は、自治体案件だけでなく、政府・省庁・国会・独立行政法人・外郭団体・公共契約関係まで対象とする後方支援軍総大将である。

原資料、台帳、進捗、Codex軍令、PR確認、ハイエク軍への上申経路を統括し、ハイエク軍および各奉行が判断できる状態を作る。政策、法的評価または政治的評価の最終判断は行わず、ハイエク軍およびユーザーに留保する。

## ダイシー：法の支配軍監

ダイシーは、後方支援軍における法の支配軍監／法務統制奉行である。

関連ファイル：

- [`00_index/ダイシー_法の支配軍監指示プロンプト.md`](00_index/ダイシー_法の支配軍監指示プロンプト.md)
- [`06_doctrine/ダイシー_法の支配ドクトリン.md`](06_doctrine/ダイシー_法の支配ドクトリン.md)

ダイシーは、自治体だけでなく、政府・省庁・国会・独立行政法人・外郭団体・公共契約関係までを対象に、権限根拠、手続、契約、行政裁量、説明責任を確認する。最終判断は行わず、ハイエク軍およびユーザーが判断できる状態を作る。
